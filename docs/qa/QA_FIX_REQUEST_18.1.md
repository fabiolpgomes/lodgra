### 2. [MEDIUM] Sem CHECK constraint nas colunas *_fee_type

**Issue ID:** FIX-18.1-02

**Location:** `supabase/migrations/20260506_02_...sql` (mesma migração do issue 01)

**Problem:**
`cleaning_fee_type` e `pet_fee_type` são colunas `text` sem restrição. O TypeScript valida com `FeeType = 'per_stay' | 'per_night'` mas o DB aceita qualquer string — o que permite dados inválidos por inserções directas ou seeds futuras.

**Expected:**
Adicionar à mesma migração do issue 01:

```sql
ALTER TABLE properties
  ADD CONSTRAINT cleaning_fee_type_check
    CHECK (cleaning_fee_type IS NULL OR cleaning_fee_type IN ('per_stay', 'per_night')),
  ADD CONSTRAINT pet_fee_type_check
    CHECK (pet_fee_type IS NULL OR pet_fee_type IN ('per_stay', 'per_night'));
```

> Nota: usar `IS NULL OR` porque as colunas são opcionais (nullable).

**Verification:**
- [ ] Constraints adicionadas na migração `20260506_02_...`
- [ ] Constraints permitem NULL (colunas opcionais)
- [ ] Constraints rejeitam valores fora de `('per_stay', 'per_night')`

**Status:** [x] Fixed

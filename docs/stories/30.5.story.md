# Story 30.5 — Post-Checkout Cleaning Trigger: Notificar Cleaner

**Epic:** 30 — WhatsApp/SMS Automated Operations  
**Status:** Ready  
**Prioridade:** P2  
**Tipo:** Backend (Automation)  
**Agente:** @dev  
**Depende de:** 30.1, 29.1 (Cleaner Portal schema)  
**Complexity:** LOW (3 story points)

---

## Story

**Como** gestor,  
**Quero** que o cleaner receba automaticamente uma notificação WhatsApp quando um checkout é confirmado,  
**Para que** a limpeza comece sem necessidade de comunicação manual.

---

## Contexto de Negócio

O fluxo operacional ideal: hóspede faz checkout → sistema detecta → notifica cleaner automaticamente → cleaner vai à propriedade. Elimina o passo manual de "ligar ao cleaner para avisar que podem ir limpar".

---

## Implementation Notes

- Endpoint reusa `sendTemplate()` de Story 30.1
- Recupera `default_cleaner_id` de `properties` table
- Valida se cleaner tem telefone antes de envio
- Se Epic 29 enabled: cria `cleaning_task` com status `pending`
- Task link é condicional (só inclui se Epic 29 ativo)
- Campo `cleaner_notified_at` rastreia envio
- Manual trigger: gestor chama quando checkout é confirmado

---

## Epic 29 Integration

**Se `organizations.cleaner_portal_enabled = true`:**
- Cria automaticamente `cleaning_task` com `status=pending`
- WhatsApp inclui link direto para tarefa no portal
- Cleaner pode confirmar chegada via portal

**Se `organizations.cleaner_portal_enabled = false`:**
- Envia apenas mensagem WhatsApp
- Sem criação de tarefa
- Sem link para portal

## Story

**Como** prospect que visita www.homestay.pt pela primeira vez,
**Quero** ver uma landing page moderna, premium e internacionalmente confiável,
**Para que** me sinta confiante em subscrever e perceba claramente o valor da plataforma.

---

## Contexto de Negócio

A `LandingPage.tsx` actual usa cores Tailwind genéricas (`blue-600`, `green-50`, `gray-900`) sem identidade de marca. É funcional mas visualmente inconsistente com o novo design system (Story 14.1) e com a experiência que os hóspedes terão em `/p/[slug]` (Story 14.2).

Objectivos do redesign:
1. **Consistência** — mesma paleta ocean blue + coral em todos os pontos de contacto
2. **Conversão** — CTAs corais destacam-se no contexto Apple-like (branco + espaço)
3. **Internacional** — suporte visual ao mercado US além de BR/PT
4. **Premium** — hero mais imersivo, secções com mais whitespace, tipografia hierárquica

### Princípios Apple aplicados à landing

- 1 CTA principal por secção (nunca dois a competir)
- Muito whitespace vertical (80–120px entre secções)
- Texto grande e bold nos heroes, leve no corpo
- Imagens/ícones como protagonistas, não como decoração

### Tokens CSS a usar (disponíveis após 14.1)

```css
/* Disponíveis como CSS variables: */
--hs-brand-50    /* hero badge bg */
--hs-brand-100   /* feature icon bg */
--hs-brand-500   /* primary brand (steps, borders) */
--hs-brand-600   /* links, secondary brand */
--hs-brand-700   /* text on light brand bg */
--hs-brand-900   /* stats section bg */
--hs-neutral-50  /* hero background (warm white) */
--hs-neutral-500 /* text secondary */
--hs-neutral-900 /* text primary (warm black) */
--hs-cta-bg      /* coral CTA button */
--hs-cta-bg-hover/* coral CTA hover */
--hs-cta-text    /* white */
```

### Padrão de uso Tailwind com CSS vars (Tailwind v4)

Tokens registados em `@theme inline` de `globals.css` como `--color-hs-*`.
Usar directamente as classes geradas pelo Tailwind:

```
bg-hs-cta-bg hover:bg-hs-cta-bg-hover text-white
text-hs-neutral-900
bg-hs-brand-50 text-hs-brand-700
```

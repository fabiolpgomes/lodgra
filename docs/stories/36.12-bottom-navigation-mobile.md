# Story 36.12: Bottom Navigation (Mobile)

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.12  
**Status:** Draft  
**Story Points:** 3  
**Priority:** MEDIUM  

---

## User Story

As a **Host on Mobile**, I want **to navigate between main sections via a bottom bar** so that **I can easily access Calendários, Anúncios, Mensagens, etc.**.

---

## Description

Build mobile bottom navigation bar with 5 tabs (Hoje | Calendário | Anúncios | Mensagens | Menu). Mobile-only feature. Reuses existing navigation infrastructure.

**Why:** Bottom nav is mobile-native UX pattern (thumb-friendly navigation).

---

## Acceptance Criteria

### Bottom Navigation Bar

- [ ] Fixed position at bottom of screen
- [ ] 5 tabs: Hoje | Calendário | Anúncios | Mensagens | Menu
- [ ] Each tab has icon + label

### Tab Icons

- [ ] Hoje: Home icon (🏠)
- [ ] Calendário: Calendar icon (🗓️)
- [ ] Anúncios: Announcement icon (📣)
- [ ] Mensagens: Message icon (💬)
- [ ] Menu: Hamburger icon (≡)

### Active State

- [ ] Currently active tab highlighted (color, bold)
- [ ] Calendários tab active (since we're on calendar page)
- [ ] Active state styling: Icon highlighted (red/blue), label bold

### Tap Interaction

- [ ] Tap tab → Navigate to corresponding page
- [ ] Update active state on navigation
- [ ] Smooth transition

### Safe Area

- [ ] Padding for notch devices (iPhone X, etc.)
- [ ] `padding-bottom: max(safe-area-inset-bottom, 16px)`

### Responsive

- [ ] Mobile only (≤768px)
- [ ] Hidden on web (>768px)

---

## Definition of Done

- [ ] All 5 tabs display correctly
- [ ] Icons visible and clear
- [ ] Navigation works
- [ ] Active state updates
- [ ] Safe area padding applied
- [ ] Mobile-only (CSS media query)

## Risks

| Risk | Mitigation |
|------|-----------|
| Content hidden behind nav bar | Add `padding-bottom` to main content |
| Icons not clear on small screens | Use bold, recognizable icon set |

---

## Technical Notes

- Use existing icon set (Feather, Heroicons, etc.)
- Fixed position: `position: fixed, bottom: 0`
- Touch target: ≥44px height
- Safe area: Use CSS `env(safe-area-inset-bottom)`

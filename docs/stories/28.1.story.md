# Story 28.1 — Fix Next.js Static Prerender: Add force-dynamic to Route Layouts

**Epic:** Build Infrastructure

## Story

**As a** developer,  
**I want** Next.js to skip static prerendering for all pages that require Supabase,  
**So that** the production build succeeds and the 3 local commits can be pushed.

---

## Background

During `npm run build`, Next.js attempts to statically prerender pages. Pages that call
`createClient()` or `createServerClient()` fail because `NEXT_PUBLIC_SUPABASE_URL` and
`NEXT_PUBLIC_SUPABASE_ANON_KEY` are not available at build time.

**Build error:**
```
Error: @supabase/ssr: Your project's URL and API key are required to create a Supabase client!
Export encountered an error on /auth/change-password/page: /auth/change-password
Export encountered an error on /admin/google-distribution/page: /admin/google-distribution
```

**Root cause:** 41 pages across multiple route groups call Supabase directly.  
**Fix strategy:** Add `export const dynamic = 'force-dynamic'` to route group layouts
so all child pages inherit it — minimal change, maximum coverage.

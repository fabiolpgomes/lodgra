## Story

**Como** proprietário de imóveis para short-term rental,
**Quero** ver um Demonstrativo de Resultado (P&L) que separa receita bruta, taxas de plataforma, despesas operacionais e impostos,
**Para que** eu possa saber a minha margem real e onde estou a perder dinheiro.

---

## Contexto de Negócio

Actualmente o sistema calcula `Lucro = receita_bruta - despesas`. Isto está **errado** para gestão real porque:
1. `platform_fee` (comissão Airbnb/Booking) é debitado antes do recebimento
2. Impostos têm tratamento fiscal diferente de custos operacionais
3. O utilizador precisa de ver `receita_bruta → receita_liquida → lucro_operacional → lucro_fiscal`

Os campos `platform_fee` e `net_amount` já existem na tabela `reservations` mas **não são utilizados nos relatórios actuais**.

### Separação de despesas por categoria

```typescript
// Separar impostos das despesas operacionais
const operationalExpenses = expenses?.filter(e => e.category !== 'taxes') || []
const taxExpenses = expenses?.filter(e => e.category === 'taxes') || []

const operationalByCurrency = groupByCurrency(
  operationalExpenses.map(e => ({
    currency: e.currency || 'EUR',
    amount: Number(e.amount)
  }))
)

const taxByCurrency = groupByCurrency(
  taxExpenses.map(e => ({
    currency: e.currency || 'EUR',
    amount: Number(e.amount)
  }))
)
```

### Cálculo de platform fees

```typescript
// Usar platform_fee se disponível, senão 0
const platformFeesByCurrency = groupByCurrency(
  reservations?.map(r => ({
    currency: r.currency || 'EUR',
    amount: r.platform_fee ? Number(r.platform_fee) : 0
  })) || []
)

// net_amount = total_amount - platform_fee (calcular se net_amount for null)
const netRevenueByCurrency = groupByCurrency(
  reservations?.map(r => ({
    currency: r.currency || 'EUR',
    amount: r.net_amount
      ? Number(r.net_amount)
      : Number(r.total_amount) - (r.platform_fee ? Number(r.platform_fee) : 0)
  })) || []
)
```

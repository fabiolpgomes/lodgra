## Contexto de Negócio

O portal de cleaners requer novos tipos de dados: tarefas de limpeza, templates de checklist, itens de checklist, fotos de prova, e tokens de acesso para cleaners. Todos os dados devem ser multi-tenant (`organization_id`) e ter RLS configurado.

### O que já existe
- Tabela `user_profiles` com campo `guest_type` (values: 'staff', 'owner')
- Supabase Storage configurado (usado em despesas e propriedades)
- RLS padrão por `organization_id` em todas as tabelas principais

### O que falta
- Novo `guest_type` value: `'cleaner'`
- Tabela `cleaning_tasks` — tarefa de limpeza por reserva/propriedade
- Tabela `cleaning_checklist_templates` — templates criados pelo gestor
- Tabela `cleaning_checklist_items` — itens do template
- Tabela `cleaning_checklist_responses` — respostas do cleaner por tarefa
- Tabela `cleaning_photos` — fotos de prova por tarefa
- Tabela `cleaner_access_tokens` — tokens de autenticação sem password
- Storage bucket `cleaning-photos`

## Schema de Referência

```sql
-- Tarefa de limpeza
CREATE TABLE cleaning_tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID NOT NULL REFERENCES organizations(id),
  property_id UUID NOT NULL REFERENCES properties(id),
  reservation_id UUID REFERENCES reservations(id),
  cleaner_id UUID REFERENCES user_profiles(id),
  checklist_template_id UUID REFERENCES cleaning_checklist_templates(id),
  status TEXT NOT NULL DEFAULT 'pending', -- pending | in_progress | done | issue
  scheduled_date DATE NOT NULL,
  scheduled_time TIME,
  notes TEXT,
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Templates de checklist
CREATE TABLE cleaning_checklist_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID NOT NULL REFERENCES organizations(id),
  property_id UUID REFERENCES properties(id), -- null = global da org
  name TEXT NOT NULL,
  description TEXT,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Itens do template
CREATE TABLE cleaning_checklist_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  template_id UUID NOT NULL REFERENCES cleaning_checklist_templates(id) ON DELETE CASCADE,
  label TEXT NOT NULL,
  category TEXT, -- ex: 'quarto', 'banheiro', 'cozinha', 'geral'
  is_required BOOLEAN DEFAULT false,
  order_index INTEGER NOT NULL DEFAULT 0
);

-- Respostas do cleaner
CREATE TABLE cleaning_checklist_responses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID NOT NULL REFERENCES cleaning_tasks(id) ON DELETE CASCADE,
  item_id UUID NOT NULL REFERENCES cleaning_checklist_items(id),
  is_checked BOOLEAN DEFAULT false,
  notes TEXT,
  checked_at TIMESTAMPTZ,
  UNIQUE(task_id, item_id)
);

-- Fotos de prova
CREATE TABLE cleaning_photos (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID NOT NULL REFERENCES cleaning_tasks(id) ON DELETE CASCADE,
  storage_path TEXT NOT NULL,
  caption TEXT,
  uploaded_at TIMESTAMPTZ DEFAULT now()
);

-- Tokens de acesso para cleaners (sem password)
CREATE TABLE cleaner_access_tokens (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  cleaner_id UUID NOT NULL REFERENCES user_profiles(id),
  token TEXT NOT NULL UNIQUE,
  expires_at TIMESTAMPTZ NOT NULL,
  used_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## Riscos & Mitigação

**RLS mal configurado** — Cleaner pode ver dados de outra organização. **Mitigação:** Testar RLS com usuário cleaner de organização diferente; incluir teste de isolamento.

**Storage bucket público** — Fotos de limpeza expostas publicamente. **Mitigação:** Bucket privado com signed URLs (TTL 1h) para visualização.

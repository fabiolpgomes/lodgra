## Story

**Como** proprietário de imóvel,
**Quero** que cada página de propriedade tenha uma imagem Open Graph personalizada e dinâmica,
**Para que** quando a página for partilhada em redes sociais (Facebook, LinkedIn, WhatsApp), apareça com uma preview visual atraente e específica da propriedade.

---

## Contexto de Negócio

A landing page (2.1) tem uma imagem OG genérica estática. As páginas de propriedade pública (`/p/[slug]`) ainda não geram imagens OG dinâmicas.

Imagens OG dinâmicas:
- Aumentam CTR em redes sociais (estudos mostram +20-30%)
- Melhoram percepção de profissionalismo
- Diferencial competitivo vs concorrentes
- Fácil implementação com `next/og` (ImageResponse)

**Tecnologia:** Next.js 15 `ImageResponse` nativa — sem dependências externas.

## Scope

**IN:**
- Páginas `/p/[slug]` geram imagens OG dinâmicas (1200×630px)
- Caching de 24h via CDN para performance
- Fallback visual elegante para propriedades incompletas
- i18n suporte: `/pt/p/[slug]`, `/es/p/[slug]` (agnóstico a idioma)
- Testes unit + integration

**OUT:**
- OG images para outras páginas (/features, /pricing, /docs) — future stories
- I18n com texto traduzido na imagem — out of scope (usar estrutura agnóstica)
- CDN configuração (assumir Vercel/Next.js default)
- Analytics de social shares (future)

## Riscos & Mitigações

| Risco | Probabilidade | Impacto | Mitigação |
|-------|---------------|--------|-----------|
| **Performance:** Imagem demora >500ms (AC7) | Média | Alto | Usar caching 24h, testar com load real, otimizar queries de propriedade |

## Dev Notes

### Estrutura de Dados Necessária

```typescript
// Do Supabase: property_listings + properties
interface PropertyForOG {
  id: string
  name: string
  location: string // City, Country
  photo_url?: string
  rating?: number
  price_per_night: number
  currency: string
}
```

### Implementação: `src/app/p/[slug]/opengraph-image.tsx`

```typescript
import { ImageResponse } from 'next/og'
import { getPropertyBySlug } from '@/lib/supabase/properties'

export const runtime = 'nodejs'
export const revalidate = 86400 // Cache 24h

export async function GET(
  request: Request,
  { params }: { params: { slug: string } }
) {
  try {
    const property = await getPropertyBySlug(params.slug)
    
    if (!property) {
      return new ImageResponse(
        <FallbackImage />,
        { width: 1200, height: 630 }
      )
    }

    return new ImageResponse(
      <PropertyOGImage property={property} />,
      { width: 1200, height: 630 }
    )
  } catch (error) {
    return new ImageResponse(
      <ErrorImage />,
      { width: 1200, height: 630 }
    )
  }
}
```

### Componente `PropertyOGImage`

Estrutura visual sugerida:
- Foto da propriedade como background (esquerda, 70%)
- Overlay gradient semi-transparente
- Texto à direita (30%): nome, localização, rating, preço
- Logo/marca Lodgra no canto inferior

### Metadata em `src/app/p/[slug]/page.tsx`

```typescript
export async function generateMetadata({ params }) {
  const property = await getPropertyBySlug(params.slug)
  
  return {
    title: property.name,
    description: property.description,
    openGraph: {
      title: property.name,
      description: `${property.location} • ${property.price_per_night}${property.currency}/noite`,
      images: [
        {
          url: `/p/${params.slug}/opengraph-image`,
          width: 1200,
          height: 630,
          alt: property.name,
        },
      ],
    },
  }
}
```

### i18n Handling

Para suportar `/pt/p/[slug]`, `/es/p/[slug]`, etc., a imagem OG pode ser:
1. Agnóstica a idioma (recomendado — imagem é visual)
2. Ou incluir texto traduzido (complexo — evitar)

Recomendação: Manter agnóstica, apenas imagem.

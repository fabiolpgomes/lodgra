## 📋 Story

**As a** property manager  
**I want** to sync booking metrics from Airbnb, Booking.com, VRBO, and Flatio  
**So that** the multi-platform dashboard shows real data from all channels (not just Google)

---

## 🎯 Acceptance Criteria

### AC1: Airbnb Metrics Sync
- [ ] OAuth 2.0 connection to Airbnb API
- [ ] Fetch daily metrics: impressions, clicks, bookings, revenue
- [ ] Real-time or hourly sync (Airbnb rate limits: high)
- [ ] Store in `airbnb_booking_metrics` table
- [ ] **Success Metric:** <30min data latency

### AC2: Booking.com Metrics Sync
- [ ] API v2 connection to Booking.com
- [ ] Fetch daily metrics: impressions, clicks, bookings, revenue
- [ ] Hourly sync schedule (Booking updates frequently)
- [ ] Store in `booking_booking_metrics` table
- [ ] **Success Metric:** <1h data latency

### AC3: VRBO Metrics Sync
- [ ] ExpediaGroup API connection for VRBO
- [ ] Fetch daily metrics: impressions, clicks, bookings, revenue
- [ ] Daily sync (24h update cycle typical)
- [ ] Store in `vrbo_booking_metrics` table
- [ ] **Success Metric:** <24h data latency

### AC4: Flatio Metrics Sync
- [ ] Flatio API v3 integration
- [ ] Fetch daily metrics: impressions, clicks, bookings, revenue
- [ ] Daily sync schedule
- [ ] Store in `flatio_booking_metrics` table
- [ ] **Success Metric:** <24h data latency

### AC5: Error Handling & Retries
- [ ] Exponential backoff for failed syncs
- [ ] Alert if no successful sync for >48h
- [ ] Cache last successful data (fallback)
- [ ] Log all sync attempts with status

### AC6: Data Normalization
- [ ] Normalize metric names across platforms
- [ ] Convert currencies to base currency (USD/EUR)
- [ ] Reconcile timezone differences
- [ ] Calculate CTR, conversion rate consistently

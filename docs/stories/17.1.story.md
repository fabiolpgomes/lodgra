# Story 17.1: Despesas — upload e gestão de documentos

**Epic:** 17 — Anexos de Documentos
**Status:** Ready for Dev
**Prioridade:** P1
**Tipo:** Full Stack
**Agente:** @dev

---

## Story

**Como** gestor de propriedades,
**Quero** anexar documentos (recibos, facturas) a uma despesa,
**Para que** tenha o comprovativo associado directamente à despesa no sistema.

---

## Acceptance Criteria

- [ ] **AC1** — Upload de ficheiros: PDF, JPEG/JPG, .doc/.docx, .xls/.xlsx
- [ ] **AC2** — Máximo 20MB por ficheiro, máximo 5 ficheiros por despesa
- [ ] **AC3** — Ficheiros listados na página de detalhe da despesa com nome e tipo
- [ ] **AC4** — Download/visualização do ficheiro (link directo ou preview)
- [ ] **AC5** — Remoção individual de ficheiro com confirmação
- [ ] **AC6** — Ficheiros armazenados em Supabase Storage: `expenses/{expense_id}/{filename}`
- [ ] **AC7** — Validação de tipo MIME no servidor

---

## Dev Notes

### Storage path

```
expenses/{expense_id}/{uuid}-{original_filename}
```

### Nova tabela necessária

```sql
CREATE TABLE expense_documents (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  expense_id uuid NOT NULL REFERENCES expenses(id) ON DELETE CASCADE,
  file_name text NOT NULL,
  file_path text NOT NULL,
  file_size integer,
  mime_type text,
  created_at timestamptz DEFAULT now()
);
```

### Componente reutilizável

Criar `src/components/common/ui/FileUpload.tsx` — componente genérico de upload para reutilizar em Story 17.2 e futuras.

### Tipos MIME permitidos

```typescript
const ALLOWED_TYPES = [
  'application/pdf',
  'image/jpeg',
  'image/jpg',
  'application/msword',
  'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
  'application/vnd.ms-excel',
  'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
]
```

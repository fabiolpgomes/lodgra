## Contexto de Negócio

WhatsApp Cloud API (Meta) permite enviar mensagens HSM (Highly Structured Messages) aprovadas previamente. A integração requer: access token, phone number ID, webhook de verificação, e templates aprovados. Esta story cria o serviço central de envio reutilizado por todas as outras stories.

### Pré-requisitos externos
- `WHATSAPP_ACCESS_TOKEN` — token de acesso Meta Business
- `WHATSAPP_PHONE_NUMBER_ID` — ID do número de telefone registado
- `WHATSAPP_WEBHOOK_VERIFY_TOKEN` — token personalizado para verificação do webhook

## Schema — Tabela de Logs

```sql
CREATE TABLE whatsapp_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID REFERENCES organizations(id),
  to_phone TEXT NOT NULL,
  template_name TEXT,
  message_text TEXT,
  wa_message_id TEXT, -- ID retornado pela Meta API
  status TEXT DEFAULT 'sent', -- sent | delivered | read | failed
  error_message TEXT,
  sent_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

## Riscos & Mitigação

**Template não aprovado** — Meta rejeita templates com conteúdo promocional ou erros de formatação. **Mitigação:** Iniciar processo de aprovação com antecedência; ter templates de fallback simples.

**Número de telefone inválido** — Erro na API se número não existe no WhatsApp. **Mitigação:** Validar formato E.164 antes do envio; tratar erro 400 graciosamente.

**Rate limit 24h** — Mensagens fora de janela de 24h requerem templates HSM. **Mitigação:** Usar sempre templates aprovados para mensagens operacionais.

## Dev Notes

- API endpoint: `https://graph.facebook.com/v19.0/{phone_number_id}/messages`
- Autenticação: `Authorization: Bearer {WHATSAPP_ACCESS_TOKEN}`
- Usar `fetch` nativo (Next.js) ou `axios` — sem SDK oficial necessário
- Webhook verify: `GET ?hub.mode=subscribe&hub.verify_token=TOKEN&hub.challenge=CHALLENGE`

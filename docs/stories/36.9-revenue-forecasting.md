## User Story

As a **Host**, I want **to forecast my property's revenue for the next 30, 60, and 90 days** so that **I can plan my finances and optimize pricing strategy**.

---

## Description

Build a revenue forecasting feature that analyzes historical booking patterns and predicts future revenue. Includes time-series analysis with seasonal adjustments, confidence scoring, and PDF export. Mobile-friendly visualization of forecast trends.

**Why:** Hosts need predictive insights into revenue to plan investments and optimize pricing without guesswork.

---

## Acceptance Criteria

### Forecast Data Analysis

- [x] Calculate average revenue per night from last 90 days of bookings
- [x] Detect seasonal patterns (peaks/valleys by month/day-of-week)
- [x] Detect day-of-week trends (weekends vs weekdays)
- [x] Account for upcoming holidays and local events
- [x] Generate 30-day, 60-day, and 90-day revenue projections
- [x] Calculate occupancy rate forecast based on current bookings
- [x] Apply seasonal adjustment factor to base projection

### Forecast Visualization (Mobile + Web)

- [ ] Display three forecast cards (30-day, 60-day, 90-day)
- [ ] Each card shows: Projected revenue (EUR), % change vs last period, confidence score
- [ ] Interactive chart (line graph with date range selector)
- [ ] Chart displays: Projected revenue line + confidence interval bands (±15%)
- [ ] Tooltip on hover: Date, projected amount, confidence level
- [ ] Legend showing "Forecast" vs "Historical data"
- [ ] Mobile: Full-width cards, stacked chart
- [ ] Web: Side-by-side layout, larger chart

### Accuracy & Confidence

- [x] Calculate confidence score (0-100%) based on data variance
- [x] Display confidence level: "High (95%)", "Medium (70%)", "Low (50%)"
- [x] Show confidence reasoning: "Based on 30 bookings in similar season"
- [x] Display data quality warning if insufficient historical data (<10 bookings)

### Forecast Details Modal

- [ ] Tap/click forecast card → opens detailed breakdown
- [ ] Show daily breakdown for selected period
- [ ] Display assumptions used: seasonal factor, occupancy rate, base price
- [ ] Show forecast vs historical comparison chart
- [ ] Display list of major factors influencing forecast (holidays, events, etc.)

### PDF Export

- [ ] Export button on forecast view
- [ ] PDF includes: Summary (3 forecasts), detailed chart, assumptions, methodology
- [ ] Branded with property name and date generated
- [ ] Email PDF to host (optional: store in property documents)

### Responsive Design

- [ ] Mobile (≤768px): Cards stack vertically, chart is touch-friendly
- [ ] Web (>768px): Cards in row, larger chart, detailed sidebar info
- [ ] All numbers format correctly (EUR, percentages)

## Technical Notes

- Use Recharts library for charts (already in project)
- Time-series algorithm: Simple Moving Average (SMA) + seasonal decomposition
- Confidence calculation: variance of historical data normalized to 0-100
- PDF generation: use `jsPDF` + `html2canvas`
- API endpoint returns pre-calculated forecast data (calculations on backend for performance)
- Debounce chart interactions to avoid re-rendering
- Cache forecast calculations (valid for 24 hours)
- Format all currency as EUR with 2 decimals

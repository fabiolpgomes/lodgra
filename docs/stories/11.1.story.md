## Story

**Como** utilizador da plataforma,
**Quero** poder controlar que dados são recolhidos sobre mim (cookies analíticos, tracking),
**Para que** o meu consentimento seja registado conforme RGPD (PT) e LGPD (BR).

---

## Contexto de Negócio

Já existe um `CookieBanner.tsx` que guarda consentimento em `localStorage`. Isto **não é suficiente** para compliance porque:
1. Consentimento deve ser registado server-side (auditável)
2. Google Analytics deve respeitar a escolha (não carregar se recusado)
3. Consentimento deve ser revogável a qualquer momento
4. LGPD exige base legal explícita para cada finalidade

### O que já existe
- `src/components/ui/CookieBanner.tsx` — banner básico accept/decline
- `src/components/analytics/GoogleAnalytics.tsx` — GA tracking
- `localStorage` key `cookie_consent`

### O que falta
- Registo server-side do consentimento
- GA condicionado ao consentimento
- UI para revogar consentimento (Settings)
- Granularidade por tipo de cookie (analytics vs essential)
- Timestamp de consentimento para auditoria

## Dev Notes

### Migração

```sql
CREATE TABLE IF NOT EXISTS consent_records (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE SET NULL,
  consent_type TEXT NOT NULL CHECK (consent_type IN ('analytics', 'marketing', 'essential')),
  consent_value BOOLEAN NOT NULL,
  ip_address TEXT,
  user_agent TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_consent_records_user ON consent_records(user_id);
CREATE INDEX idx_consent_records_type ON consent_records(consent_type, created_at DESC);
```

### Google Analytics condicional

```tsx
// GoogleAnalytics.tsx
const consent = getConsent()
if (consent !== 'accepted') return null
```

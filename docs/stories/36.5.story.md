# Story 36.5: Calendar Bulk Operations

**Epic:** EPIC 36 (Pricing Management System)  
**Story Points:** 8  
**Status:** InReview  
**Priority:** High  
**Dependencies:** Stories 36.1-36.4 (complete)

---

## Story

Implement bulk date range operations for efficient pricing management. Users can select multiple calendar dates, apply pricing changes to all selected dates simultaneously, and undo operations if needed.

---

## Acceptance Criteria

- [x] Select date range by holding + click start date → click end date
- [x] Apply single price to all dates in selected range
- [x] Apply discount percentage to all dates in selected range
- [x] Copy price from source date to entire range
- [x] Bulk delete daily price overrides from range
- [x] Confirmation dialog displays affected dates before applying changes
- [x] Undo last operation restores previous prices from history
- [x] Validate range (end date >= start date)
- [x] Visual feedback highlights selected date range
- [x] Mobile support (tap-and-hold to start, drag to select range)
- [x] Operation history stores last 5 operations
- [x] Optimistic updates with rollback on error

---

## Dev Notes

### Component Architecture

**CalendarMonth Enhancement:**
- Add range selection state (startDate, endDate, isSelecting)
- Add visual highlight for selected range (CSS/Tailwind classes)
- Implement click + modifier key detection
- Mobile: Implement touch event handlers for tap-hold-drag

**Hooks:**
- `useBulkPricingOperation` - Manages bulk operations:
  - Batch API calls for price updates
  - Operation history tracking (last 5)
  - Undo mechanism (restore previous state)
  - Optimistic updates with error rollback

**Components:**
- `BulkOperationModal` - Confirmation dialog:
  - List affected dates
  - Show preview of changes
  - Confirm/Cancel actions
  - Display operation type (apply price, discount, copy, delete)

**State Management:**
- Operation history stored in component state or context
- Previous prices captured before each operation
- Optimistic updates to UI immediately, revert on API failure

### Implementation Details

- **Range Selection:** Multiple click with modifier key (Cmd/Ctrl) or dedicated button mode
- **Mobile Interaction:** Long-press to initiate, drag to expand range
- **API Integration:** Batch endpoint for bulk price updates or individual calls in parallel
- **Validation:** Ensure end >= start, validate all dates are within property's available dates
- **Error Handling:** Show error toast if bulk operation fails, restore previous prices

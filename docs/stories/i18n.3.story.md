## Implementation Notes

### Architecture
- **Forex Client** (`src/lib/forex/client.ts`): Fetches rates from exchangerate-api.com (free, no auth)
- **Rates Manager** (`src/lib/forex/rates.ts`): Handles caching (Redis + in-memory fallback), fallback rates hardcoded
- **Currency Config** (`src/lib/currency/config.ts`): Locale↔Currency mapping, symbols
- **Converter** (`src/lib/currency/convert.ts`): Base currency is EUR, converts to BRL/USD, 2 decimals
- **Pricing Converter** (`src/lib/pricing/convertPricing.ts`): Adapts converter for pricing API responses
- **Cron Endpoint** (`src/app/api/cron/update-forex/route.ts`): Secured with CRON_SECRET, updates cache hourly

### Key Design Decisions
1. **EUR as base**: All DB prices in EUR, conversion happens at API layer (no dual-write risk)
2. **Cache strategy**: Redis (Upstash) primary, memory fallback if unavailable
3. **Fallback rates**: Hardcoded (1 EUR = 5.5 BRL, 1.1 USD) if API fails
4. **Rounding**: Standard Math.round to 2 decimals (financial accuracy)
5. **Free API**: exchangerate-api.com chosen (no auth, 1500 req/month free tier)

### Testing Coverage
- **38 unit tests**: Config mapping, conversion math, cache behavior, fallback handling
- **All passing**: `npm test -- --testPathPattern="currency|forex"`
- **No breaking changes**: Existing currency.ts utilities unchanged

### Deployment
1. Add `CRON_SECRET` to .env (if not present)
2. Optionally set `UPSTASH_REDIS_REST_URL` & `UPSTASH_REDIS_REST_TOKEN` for distributed caching
3. Schedule cron: POST /api/cron/update-forex with `Authorization: Bearer $CRON_SECRET`
4. Query pricing: GET /api/public/properties/[slug]/pricing?checkin=...&checkout=...&currency=BRL

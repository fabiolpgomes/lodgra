# Story 36.2: Refatorar Sistema para Usar Platform Booking IDs Como Primary Reference

## 🎯 Objetivo

Refatorar o sistema de importação iCal para usar **identificadores únicos da plataforma** (Booking ID, Airbnb Listing ID, etc.) como primary reference em vez de gerar emails fake (`imported-{timestamp}@lodgra.local`) e UIDs genéricos.

**Benefício**: Elimina duplicação, facilita rastreamento de origem, prepara para integração bidirecional futura (enviar cancelamentos de volta ao Booking).

---

## 🔴 Problema Atual

### Arquitetura Atual

```typescript
// Ao importar reserva do iCal:
{
  external_id: "booking_6816972454",
  guest_email: "imported-{timestamp}-{random}@lodgra.local",  // ← PROBLEMA!
  guest_name: "Hóspede Importado",                           // ← Genérico
  booking_source: "ical_auto_sync"
}
```

### Limitações

| Limitação | Impacto |
|-----------|---------|
| Email fake | Não pode enviar confirmações reais ao hóspede |
| Sem guest_name | Usuário deve preencher manualmente |
| Sem booking_reference | Difícil rastreamento de qual plataforma |
| Sem booking_source estruturado | Não sabe qual plataforma específica |
| Sem link bidirecional | Não pode enviar dados de volta ao Booking |

### Cenário Problemático

1. Booking envia reserva: guest = "Laurence BAREILLE"
2. Lodgra importa como: guest = "Hóspede Importado", email = "imported-xxx@lodgra.local"
3. Proprietário atualiza manualmente
4. Próxima sincronização: dados voltam ao padrão? Conflito?

---

## ✅ Solução Proposta

### Arquitetura Alvo

```typescript
// Estrutura refatorada:
{
  external_id: "booking_6816972454",
  booking_reference: "6816972454",        // ← Platform ID armazenado
  booking_source: "booking",              // ← Plataforma específica
  
  // Se plataforma enviar dados:
  guest: {
    first_name: "Laurence",              // ← Real do Booking
    last_name: "BAREILLE",
    email: null,                          // ← Ainda pode ser vazio
    phone: "+33 6 12 34 56 78"           // ← Se disponível
  },
  
  // Rastreabilidade:
  platform_sync_metadata: {
    platform: "booking",
    booking_id: "6816972454",
    synced_at: "2026-07-09T15:05:51Z",
    platform_url: "https://www.booking.com/booking?..."  // ← Link direto
  }
}
```

### Benefícios

| Benefício | Descrição |
|-----------|-----------|
| **Dados Reais** | Guest name/email vem da plataforma, não genérico |
| **Rastreabilidade** | Saber exatamente qual plataforma e qual ID |
| **Link Bidirecional** | Futuramente: enviar cancelamentos de volta |
| **Integração** | Facilita webhooks e APIs das plataformas |
| **Auditoria** | Histórico de sincronização com timestamps |

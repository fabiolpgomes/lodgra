## User Story

As a **Booking System**, I need **to calculate final price with discount applied** so that **guests see the correct price when booking**.

---

## Description

Implement backend pricing calculation engine that takes check-in/check-out dates and returns final price with discount applied. This is the core logic for pricing optimization.

**Logic:**
1. Calculate total nights between check-in and check-out
2. Determine applicable discount (weekly ≥7 nights, monthly ≥28 nights)
3. Apply discount to daily prices (respecting daily overrides)
4. Return breakdown with base price, discount %, and final total

**Why:** Without this, discounts aren't applied in booking flow.

---

## Acceptance Criteria

### Core Function

- [ ] Create function `calculatePrice(propertyId, checkInDate, checkOutDate)`
- [ ] Input validation:
  - checkInDate must be before checkOutDate
  - Both dates must be valid ISO format (YYYY-MM-DD)
  - Property must exist

### Discount Precedence

- [ ] If total nights ≥ 28 → Apply monthly discount (if configured)
- [ ] Else if total nights ≥ 7 → Apply weekly discount (if configured)
- [ ] Else → No discount (return base price)

**Example:**
- 15 nights → Uses weekly discount (7+ rule applies, 28+ does not)
- 6 nights → No discount
- 28 nights → Uses monthly discount

### Price Calculation

- [ ] For each night in stay:
  - Fetch price (daily override OR base_price OR weekend_price)
  - Sum daily prices → baseTotal
- [ ] Apply discount:
  - If discount exists: `finalPrice = baseTotal × (1 - discountPercent/100)`
  - Else: `finalPrice = baseTotal`

### Response Format

```json
{
  "baseTotal": 1192.00,
  "discountApplied": true,
  "discountType": "weekly",
  "discountPercentage": 21,
  "discountAmount": 251.32,
  "finalTotal": 940.68,
  "breakdown": [
    { "date": "2026-07-05", "price": 149.00 },
    { "date": "2026-07-06", "price": 149.00 },
    ...
  ]
}
```

### Edge Cases

- [ ] Mixed prices (some days base, some weekend, some custom) → Sum correctly
- [ ] No discount configured → Return base price
- [ ] Discount = 0% → Return base price
- [ ] Rounding: Always round to 2 decimals (EUR)

### API Endpoint

- [ ] `POST /api/properties/:id/calculate-price`
- [ ] Body: `{ checkInDate: "2026-07-05", checkOutDate: "2026-07-12" }`
- [ ] Returns: Pricing breakdown (above format)

## Technical Notes

- Use JavaScript `Date` carefully (timezone issues)
- For date ranges, use `date-fns` library
- Store discounts as integer percentages (0-100), not decimals
- Rounding: Use `Math.round(value * 100) / 100` or library function
- Performance: Cache discount rules in memory (optional)

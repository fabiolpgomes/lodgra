# Story 29.3 — Cleaner Dashboard: Vista Mobile de Tarefas

**Epic:** 29 — Cleaner Operations Portal  
**Status:** Draft  
**Prioridade:** P1  
**Tipo:** Frontend (Mobile-first)  
**Agente:** @dev  
**Depende de:** 29.1, 29.2  
**Complexity:** MEDIUM (5 story points)

---

## Story

**Como** cleaner,  
**Quero** ver as minhas tarefas de limpeza do dia num painel mobile simples,  
**Para que** saiba o que tenho de fazer, em que propriedade, e a que horas.

---

## Contexto de Negócio

O cleaner acede ao portal via WhatsApp link num smartphone. A interface deve ser ultra-simples: lista de tarefas ordenadas por hora, com status visual, e acção de iniciar limpeza com um toque. Não há menus, não há configurações — apenas tarefas.

---

## Critérios de Aceitação

- [ ] **AC1** — `/cleaner/dashboard` lista as tarefas do dia ordenadas por `scheduled_time`
- [ ] **AC2** — Cada tarefa mostra: propriedade, hora, status (badge colorido), nome da reserva
- [ ] **AC3** — Tarefa com status `pending` tem botão "Iniciar Limpeza" → muda para `in_progress`
- [ ] **AC4** — Tarefa `in_progress` tem botão "Concluir" → abre fluxo de checklist + fotos (story 29.4/29.5)
- [ ] **AC5** — Tarefa `done` mostra hora de conclusão e badge verde
- [ ] **AC6** — Tarefa `issue` mostra badge vermelho com notas do problema
- [ ] **AC7** — Vista "Próximas" mostra tarefas dos próximos 7 dias (tab separado)
- [ ] **AC8** — UI 100% mobile-first (touch targets ≥ 44px, sem hover states necessários)
- [ ] **AC9** — Realtime: status actualiza sem refresh quando gestor modifica tarefa
- [ ] **AC10** — Testes: render de lista, mudança de status, realtime subscription (4+ testes)

## Riscos & Mitigação

**Realtime na rede móvel** — Supabase Realtime pode desconectar em redes instáveis. **Mitigação:** Implementar reconnect automático; fallback para polling a cada 30s.

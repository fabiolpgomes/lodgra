# Story 6.1: Management Commission Reports

## Story

As a property manager using the HomeStay platform, I want to track commission earnings from bookings so that I can understand my revenue breakdown and validate platform fee calculations.

## Acceptance Criteria

- [x] Commission rates are configurable per subscription plan
- [x] Each booking records the commission amount deducted
- [x] Commission dashboard shows:
  - Total commissions earned (current month, year-to-date, all-time)
  - Commission breakdown by property
  - Commission trend chart (daily/weekly/monthly)
  - Commission rate for current plan
- [x] Commission report exports to CSV with booking-level detail
- [x] Commission calculations are transparent and auditable
- [x] RLS policies ensure users can only see their own organization's commissions
- [x] Dashboard reflects real-time commission data from bookings

## Story Details

### Context

Commission is the platform fee deducted from each booking's gross revenue. The SaaS model uses tiered pricing with commission rates:

- **Starter** (€19/month): 20% commission
- **Professional** (€49/month): 15% commission
- **Business** (€99/month): 10% commission

Currently, commission is calculated but not tracked or reported. Users have no visibility into:
- How much commission they've paid
- Commission breakdown by property
- Commission rate for their plan
- Historical commission trends

This story adds transparent commission reporting to build trust and provide financial visibility.

### Technical Approach

#### Database Changes
1. Add commission tracking columns to `bookings` table:
   - `commission_amount` (decimal) - Amount deducted as commission
   - `commission_rate` (decimal) - Rate applied (e.g., 0.20 for 20%)
   - `commission_calculated_at` (timestamp) - When commission was calculated

2. Update booking creation flow to:
   - Fetch current org plan via `organizations.plan`
   - Look up commission rate from `plans.commission_rate`
   - Calculate: `commission_amount = gross_revenue * commission_rate`
   - Store commission details in booking record

3. Create indexed view: `commission_summary` for dashboard queries
   - GROUP BY organization_id, property_id
   - SUM commission_amount
   - For each: count bookings, avg commission, date range

#### API Changes
1. `GET /api/commissions/dashboard` - Dashboard summary
   ```typescript
   {
     currentMonth: { total, count, avgPerBooking },
     yearToDate: { total, count, avgPerBooking },
     allTime: { total, count, avgPerBooking },
     currentRate: 0.15,  // from plan
     byProperty: [{ id, name, total, count }, ...]
   }
   ```

2. `GET /api/commissions/history` - Paginated commission history
   ```typescript
   {
     data: [
       {
         bookingId, propertyId, guestName, checkIn, checkOut,
         grossRevenue, commissionRate, commissionAmount,
         calculatedAt
       }
     ],
     pagination: { page, pageSize, total }
   }
   ```

3. `GET /api/commissions/export?format=csv` - CSV export

#### UI Components
1. `CommissionDashboard.tsx` - Main dashboard with metrics cards
2. `CommissionChart.tsx` - Trend visualization (Chart.js or similar)
3. `CommissionHistory.tsx` - Paginated booking-level detail table
4. `CommissionExport.tsx` - CSV download button
5. Commission section in existing `/dashboard/reports` page

#### Feature Gating
- Commission reports available on all plans (transparency for all)
- Advanced analytics (trends, forecasting) gated to Professional+ plans

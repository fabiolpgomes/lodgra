## Objective

Create a Google Vacation Rentals Feed API endpoint that generates a valid XML feed containing all property listings with schema data from Story 27.1. This feed enables bulk submission to Google's Vacation Rentals partner program for automated property synchronization and rich search results.

- [x] **AC1:** `/api/feeds/google-vacation-rentals` endpoint created (GET method, public access)
- [x] **AC2:** XML feed includes all properties with required fields (name, description, address, price, images, availability)
- [x] **AC3:** Feed validates against Google's XSD schema specification
- [x] **AC4:** Supports pagination/filtering (limit, offset, updated_since filters)

## Implementation Details

### Location
- **Primary file:** `/src/app/api/feeds/google-vacation-rentals/route.ts` (new endpoint)
- **Utilities:** `/src/lib/feeds/google-feed-generator.ts` (feed generation logic)
- **Schema:** `/src/lib/feeds/google-feed-schema.ts` (XML structure definitions)
- **Tests:** `/src/__tests__/feeds/google-feed.test.ts` (unit + integration)

### Data Source
- Property data: `properties` table (via Story 27.1 schema generator)
- Images: `property_media` table (photos, prioritize WebP)
- Availability: `property_pricing` table (pricing, min_stay)
- Reviews: `property_reviews` table (aggregate ratings from Story 27.1)
- Updates: Use `updated_at` timestamp for delta feeds

### XML Feed Structure (Google Format)
```xml
<?xml version="1.0" encoding="UTF-8"?>
<feed xmlns="http://www.google.com/feeds/property">
  <title>Lodgra Property Feed</title>
  <link href="https://lodgra.app" rel="alternate"/>
  <updated>2026-05-15T10:30:00Z</updated>
  <entry>
    <id>property-uuid-123</id>
    <title>Cozy Beach Villa</title>
    <content type="xhtml">
      <div xmlns="http://www.w3.org/1999/xhtml">
        <p>Stunning beachfront property with sea views...</p>
      </div>
    </content>
    <link href="https://lodgra.app/properties/villa-slug" rel="alternate"/>
    <link href="https://cdn.lodgra.app/properties/uuid/photo1.webp" rel="image"/>
    <link href="https://cdn.lodgra.app/properties/uuid/photo2.webp" rel="image"/>
    <author>
      <name>Property Owner</name>
    </author>
    <gd:rating average="4.7" min="1" max="5"/>
    <gd:money amount="150.00" currencyCode="EUR"/>
    <georss:point>-8.7353 39.7412</georss:point>
    <property:address>
      <property:city>Lagos</property:city>
      <property:country>PT</property:country>
      <property:postalCode>8600-001</property:postalCode>
    </property:address>
    <property:checkInTime>14:00</property:checkInTime>
    <property:checkOutTime>11:00</property:checkOutTime>
  </entry>
</feed>
```

### API Endpoint Spec
```
GET /api/feeds/google-vacation-rentals
Query Parameters:
  - limit: number (default 100, max 1000)
  - offset: number (default 0, pagination)
  - updated_since: ISO8601 timestamp (delta feed, returns only updated properties)
  - currency: string (default EUR, ISO 4217 code)

Response Headers:
  - Content-Type: application/atom+xml; charset=utf-8
  - ETag: hash of feed content (for client caching)
  - Last-Modified: ISO8601 timestamp (for conditional GET)
  - Cache-Control: public, max-age=3600 (1 hour cache)
  - X-Feed-Count: total number of entries

Response: 200 OK
  - Valid XML feed with all properties
  - Empty <feed/> if no properties match criteria
```

### Implementation Approach

1. **Create feed generator utility** (`/src/lib/feeds/google-feed-generator.ts`):
   - Function `generateGoogleVacationRentalsFeed(options: FeedOptions)` returns XML string
   - Reuses schema data from Story 27.1's generateLodgingBusinessSchema
   - Handles pagination, filtering, currency conversion
   - Implements caching layer (Redis/in-memory)

2. **Create XML schema definitions** (`/src/lib/feeds/google-feed-schema.ts`):
   - Define Atom feed structure (namespace declarations)
   - Map property data to feed entry fields
   - Validate against Google's XSD before returning

3. **Integrate into API route** (`/src/app/api/feeds/google-vacation-rentals/route.ts`):
   - Parse query parameters
   - Call generateGoogleVacationRentalsFeed
   - Set HTTP caching headers
   - Return XML response

4. **Performance & Caching:**
   - Cache full feed for 1 hour (Cache-Control: max-age=3600)
   - Invalidate cache on property updates (via Supabase hooks or cron)
   - Support conditional requests (ETag, Last-Modified)
   - Lazy-load images (only include top 5 per property)

5. **Security & Validation:**
   - XML injection prevention (escape all user data)
   - No PII exposure (only public property data)
   - Rate limiting (max 10 requests/min per IP)
   - Feed validation against Google's schema before return

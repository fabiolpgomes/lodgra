## Story

**Como** hóspede,  
**Quero** receber o código de acesso da propriedade via WhatsApp antes do check-in,  
**Para que** possa fazer check-in de forma autónoma sem precisar de contactar o gestor.

**Como** gestor,  
**Quero** que este envio seja automático e configurável,  
**Para que** poupe tempo em operações repetitivas.

---

## Contexto de Negócio

O código de check-in (código da fechadura, caixa de chaves, ou PIN do portão) é a informação mais crítica de uma reserva. Actualmente enviado manualmente por email ou WhatsApp pessoal. A automação elimina este trabalho manual e reduz erros.

### Campo de dados
O código de acesso já pode existir em `reservations` ou `properties`. Se não existir, criar campo `checkin_code` em `reservations`.

## Critérios de Aceitação

- [ ] **AC1** — Campo `checkin_code` em `reservations` (texto livre, ex: "Caixa cinza, código 4521")
- [ ] **AC0** — Todas as mensagens incluem obrigatoriamente o título (`property_name`) e endereço completo (`property_address`) da propriedade
- [ ] **AC2** — Campo `checkin_instructions` em `reservations` ou `properties` (texto completo)
- [ ] **AC3** — Envio automático: **24h antes** do check-in via Vercel Cron
- [ ] **AC4** — Gestor pode configurar: enviar 24h antes, 48h antes, ou no dia (por organização)
- [ ] **AC5** — Mensagem inclui obrigatoriamente: título da propriedade, endereço completo, nome do hóspede, data, código de acesso, instruções
- [ ] **AC6** — Log de envio registado em `whatsapp_logs`
- [ ] **AC7** — Gestor pode reenviar manualmente via botão na reserva
- [ ] **AC8** — Se hóspede não tem número WhatsApp registado: fallback para email


---

## Riscos & Mitigação

**Reserva sem telefone do hóspede** — Impossível enviar WhatsApp. **Mitigação:** Verificar campo `guest_phone` no momento do cron; se ausente, usar email fallback; notificar gestor se ambos ausentes.

**Código de acesso vazio** — Mensagem enviada sem código é inútil. **Mitigação:** Verificar `checkin_code` antes de enviar; notificar gestor se campo vazio 48h antes.

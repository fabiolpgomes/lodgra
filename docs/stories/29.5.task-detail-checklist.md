# Story 29.5 — Task Detail: Checklist Management

**Epic:** 29 — Cleaner Operations Portal  
**Status:** Ready  
**Prioridade:** P1 — Core feature (execução)  
**Tipo:** Frontend + API  
**Agente Executor:** @dev (Dex)

## Story

**Como** cleaner,  
**Quero** marcar itens do checklist como completo,  
**Para que** possa registrar o progresso da limpeza.

---

## Acceptance Criteria

### AC1: Page `/[locale]/cleaning/tasks/[taskId]` mostra detalhes
- [ ] **GIVEN** taskId válido
- **WHEN** acessa página
- **THEN** carrega em < 2s
- [ ] **AND** mostra: property info, reservation info, checklist, status, notes

### AC2: Property info (name, address, check-in/out times)
- [ ] **WHEN** página renderiza
- **THEN** exibe: "Vila Paraíso", "Rua X, Porto", "In: 15h", "Out: 11h"

### AC3: Reservation info (guest name, dates) se aplicável
- [ ] **WHEN** task tem reservation_id
- **THEN** mostra: "Guest: João Silva", "Check-in: 25 Maio 15:00"

### AC4: Checklist items com checkboxes + labels
- [ ] **WHEN** checklist renderiza
- **THEN** mostra 10+ items com checkboxes
- [ ] **AND** cada item tem label (ex: "Trocar roupa de cama")

### AC5: Marca item como done (click checkbox)
- [ ] **WHEN** clica checkbox
- **THEN** item fica marcado visualmente (strike-through)
- [ ] **AND** POST `/api/cleaning/[id]/checklist-items/[itemId]` com `is_done=true`
- [ ] **AND** `done_at = now()` salvo em DB

### AC6: Pode desmarcar (reverse action)
- [ ] **WHEN** clica checkbox marcado
- **THEN** desmarca
- [ ] **AND** `is_done=false`, `done_at=null`

### AC7: Progresso visual (X/Y itens completos)
- [ ] **WHEN** 5/10 itens completos
- **THEN** mostra "5/10" em progresso bar
- [ ] **AND** percentage bar preenchida 50%

### AC8: Status buttons (Start, In Progress, Mark Complete)
- [ ] **WHEN** task status = pending
- **THEN** botão "Começar" fica visível
- [ ] **WHEN** clica "Começar"
- **THEN** status muda para in_progress, botão muda para "Marcar Completo"
- [ ] **WHEN** clica "Marcar Completo"
- **THEN** status = done

### AC9: Manager notes (read-only para cleaner)
- [ ] **WHEN** página renderiza
- **THEN** mostra campo "Notas do Gestor" (read-only)
- [ ] **AND** conteúdo: notes do cleaning_tasks.notes

### AC10: Integração com photos (inline gallery)
- [ ] **WHEN** scroll down após checklist
- **THEN** vê seção "Fotos" com upload button
- [ ] **AND** fotos já uploaded aparecem em preview

## Integration Verification

### IV1: API integration (`/api/cleaning/[id]`)
- [ ] **WHEN** page loads
- **THEN** GET `/api/cleaning/[taskId]` retorna task completo com items

### IV2: RLS valida cleaner é assigned_to
- [ ] **GIVEN** cleaner-A tenta acessar task de cleaner-B
- **WHEN** GET `/api/cleaning/[otherTaskId]`
- **THEN** 403 Forbidden

### IV3: Update status (`PATCH /api/cleaning/[id]`)
- [ ] **WHEN** muda status button
- **THEN** PATCH to `/api/cleaning/[id]` com status
- [ ] **AND** updated_at em DB

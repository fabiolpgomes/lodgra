# Story 36.6: Responsive Layout Foundation

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.6  
**Status:** Draft  
**Story Points:** 5  
**Priority:** HIGH  

---

## User Story

As a **Mobile-First User**, I need **a responsive calendar layout** so that **I see the right layout for my device (mobile card list vs web 3-column)**.

---

## Description

Implement responsive layout structure that adapts to mobile (≤768px) vs web (>768px) viewports. This is the foundation for all calendar UI work (Stories 36.7-36.12 build on top).

**Why:** 90% mobile users need optimized mobile UX; web needs different layout to use space.

---

## Acceptance Criteria

### Mobile Layout (≤768px)

- [ ] Header: "Calendários" + Search icon (🔍)
- [ ] Main view: Card list (vertical scroll, each card is a property)
- [ ] Each card structure:
  - Thumbnail photo (rounded, 4:3 aspect)
  - Property name
  - Location
  - Mini calendar dots (visual occupancy)
- [ ] Bottom navigation: Hoje | Calendário | Anúncios | Mensagens | Menu
- [ ] Tap card → Navigate to detailed calendar view

### Web Layout (>768px)

- [ ] Header: Hamburger (≡) | "Calendários" | Search
- [ ] 3-column layout:
  - Left (25%): Sidebar with property list
  - Center (50%): Detailed calendar grid
  - Right (25%): Config panel (settings)
- [ ] Hamburger button: Top-left, fixed position
- [ ] No bottom navigation (not needed on web)

### Breakpoint Logic

- [ ] Media query: `@media (max-width: 768px)` = mobile
- [ ] Media query: `@media (min-width: 769px)` = web
- [ ] Layout transitions smoothly at breakpoint
- [ ] No horizontal scroll on mobile

### Components

- [ ] Shared: Header, Search
- [ ] Mobile-only: CardList, BottomNav
- [ ] Web-only: Sidebar, ConfigPanel, Hamburger
- [ ] Shared: DetailedCalendar (used in both layouts)

---

## Definition of Done

- [ ] Layout renders correctly on mobile (iPhone 12)
- [ ] Layout renders correctly on web (desktop, tablet)
- [ ] No layout shift/flicker at 768px breakpoint
- [ ] All navigation works in both layouts
- [ ] Responsive images (photos scale properly)
- [ ] No horizontal scroll
- [ ] No TypeScript errors

## Risks

| Risk | Mitigation |
|------|-----------|
| Layout shift on breakpoint | Use CSS grid/flex properly, test at boundary |
| Mobile keyboard covers inputs | Use bottom sheet or scrolling containers |
| Images not responsive | Use `max-width: 100%`, aspect-ratio CSS |

---

## Technical Notes

- Use Tailwind or CSS media queries for responsive design
- Flexbox for flexible column sizing
- CSS Grid for calendar grid layout
- Store breakpoint value in CSS variable for consistency
- Test on real devices (not just browser dev tools)

# Story 36.6 — Advanced Pricing Features

**Epic:** EPIC 36 (Pricing Management System)

## Story Summary

Implement advanced pricing constraint features including minimum/maximum nightly price enforcement, seasonal pricing rules with date ranges, and price floor/ceiling overrides. Enable property managers to create sophisticated pricing strategies with validation, warnings, and a seasonal pricing calendar view.

---

## Acceptance Criteria

- [ ] Set minimum nightly price with validation (prevents accidentally low prices)
- [ ] Set maximum nightly price with validation (prevents overpricing)
- [ ] Create seasonal pricing rules (date ranges with custom rates)
- [ ] Apply price floor/ceiling to override daily prices
- [ ] Validate prices against min/max range before saving
- [ ] Display price range warnings in UI when price falls outside constraints
- [ ] List all active seasonal pricing rules
- [ ] Edit existing seasonal pricing rules
- [ ] Delete seasonal pricing rules
- [ ] Seasonal pricing calendar view (highlight seasonal dates)
- [ ] Bulk enforce min/max pricing on all dates
- [ ] Mobile-responsive constraint management interface

## Dev Notes

### Architecture

**Pricing Calculator Enhancement:**
- Extend `pricing-calculator.ts` with floor/ceiling enforcement logic
- Add validation functions for min/max constraints
- Integrate seasonal rule lookup into price calculation pipeline

**React Hooks:**
- Create `usePricingConstraints()` hook for managing min/max and seasonal rules
- Handle constraint state, validation, and updates
- Integrate with existing pricing state management

**Database Schema:**
- New table: `seasonal_pricing_rules` (property_id, start_date, end_date, rate, type)
- New columns in `property_pricing`: `min_nightly_price`, `max_nightly_price`
- Migration: add constraints table and pricing columns

**API Endpoints:**
- `GET /api/properties/{id}/pricing/constraints` — fetch min/max settings
- `GET /api/properties/{id}/pricing/seasonal-rules` — list seasonal rules
- `POST /api/properties/{id}/pricing/constraints` — update min/max
- `POST /api/properties/{id}/pricing/seasonal-rules` — create seasonal rule
- `PATCH /api/properties/{id}/pricing/seasonal-rules/{ruleId}` — update rule
- `DELETE /api/properties/{id}/pricing/seasonal-rules/{ruleId}` — delete rule
- `POST /api/properties/{id}/pricing/bulk-enforce` — apply min/max to date range

**UI Components:**
- `PricingConstraints` — min/max input fields with live validation
- `SeasonalRuleEditor` — modal for creating/editing seasonal rules
- `SeasonalPricingCalendar` — calendar view highlighting seasonal dates
- `PricingWarnings` — visual alerts when price outside constraints
- Integration into Property Pricing Settings panel

**Mobile Considerations:**
- Responsive constraint input (stack on small screens)
- Touch-friendly date pickers for seasonal rules
- Calendar view optimized for mobile (swipe navigation, week view option)
- Inline validation feedback with minimal modal dialogs

### Key Implementation Details

1. **Constraint Validation:** Min/max constraints are soft (warning) by default, configurable to hard (blocking)
2. **Seasonal Override Priority:** Seasonal rates take precedence over base rate but respect hard constraints
3. **Bulk Operations:** Enforce min/max across date range, respecting existing overrides
4. **Conflict Detection:** Warn if seasonal rules overlap
5. **Date Boundaries:** Seasonal rules include full day (00:00-23:59)

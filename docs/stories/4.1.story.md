## Story

**Como** proprietário de múltiplas propriedades,
**Quero** visualizar despesas filtrando por propriedade específica e período,
**Para que** eu possa analisar custos operacionais por ativo de forma rápida e eficiente.

---

## Contexto de Negócio

Actualmente o módulo de Despesas mostra todas as despesas sem filtros contextuais. Ao gerir múltiplas propriedades, o proprietário precisa de:
1. Ver despesas apenas de uma propriedade (ou todas)
2. Filtrar por período (data inicial/final)
3. Visualizar categorias numa ordem lógica
4. Saber como foi pago (coluna renomeada)

Isto melhora a análise de custos por propriedade e facilita a comparação de performance operacional.

### Queries Supabase

Manter lógica RLS existente. Apenas filtrar frontend:

```typescript
// Query base (sem mudanças)
let expensesQuery = supabase
  .from('expenses')
  .select(`id, description, amount, currency, category, created_at, property_listings(name)`)
  .order('created_at', { ascending: false })

// Filtros aplicados no frontend após fetch
const filtered = expenses.filter(e => {
  const matchesProperty = !selectedProperty || e.property_listings?.name === selectedProperty
  const matchesPeriod = e.created_at >= startDate && e.created_at <= endDate
  return matchesProperty && matchesPeriod
})
```

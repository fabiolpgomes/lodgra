## User Story

As a **Desktop-First Host**, I want **to access my calendar and settings through a responsive web layout with sidebar properties and hamburger menu** so that **I can manage multiple properties and pricing efficiently on larger screens without losing mobile responsiveness**.

---

## Description

Extend the calendar redesign from Story 36.14 (mobile-first) to web/desktop layouts:
- **Web (>768px):** Sidebar with property list + central calendar + config panel
- **Hamburger Menu (Mobile fallback):** Overlay menu for property selection on smaller tablets
- **Responsive behavior:** Smooth transition from mobile card layout to web sidebar layout
- **Settings Panel (Web):** Right sidebar with PreûÏos | Descontos | Disponibilidade tabs
- **Property Sidebar (Web):** Left sidebar with filterable property list + mini calendar preview

**Why:** Users need efficient multi-property management on desktop. Story 36.14 solved mobile, 36.15 completes web experience.

## Technical Notes

### Component Reuse Strategy
- **Reuse from 36.14:**
  - `DetailedCalendarMobile` ã Show same calendar in center (rename to `DetailedCalendar`)
  - `SettingsTabs` ã Right sidebar (same component)
  - `PriceCard`, `DiscountCard`, `AvailabilityCard` ã Reuse in settings
  - `mobile-calendar.css` ã Extend with desktop breakpoints (768px, 1024px)

- **New Components:**
  - `PropertySidebar` ã Left sidebar with property list + mini calendar
  - `DesktopCalendarLayout` ã Main layout container (grid: sidebar | calendar | sidebar)
  - `HamburgerMenu` ã Overlay for property selection (tablet mode)
  - `SettingsSidebar` ã Right sidebar wrapper for SettingsTabs

### CSS Architecture
```css
/* Existing mobile-calendar.css */
/* + new desktop breakpoints */

@media (min-width: 768px) {
  .desktop-layout {
    display: grid;
    grid-template-columns: 280px 1fr; /* Sidebar + Calendar */
  }
}

@media (min-width: 1024px) {
  .desktop-layout {
    display: grid;
    grid-template-columns: 280px 1fr 300px; /* Sidebar + Calendar + Settings */
  }
}

/* Hamburger menu: 768px < width < 1024px */
@media (min-width: 768px) and (max-width: 1023px) {
  .sidebar-left { display: none; }
  .hamburger-menu { display: flex; }
}
```

### Data Flow
1. User selects property from sidebar/hamburger
2. `PropertySidebar` or `HamburgerMenu` emits `onPropertySelect`
3. Parent component updates state: `selectedPropertyId`
4. `DetailedCalendar` fetches pricing/reservations for selected property
5. `SettingsSidebar` shows settings for selected property

## Story

**Como** gestor,  
**Quero** editar os templates de mensagens WhatsApp no painel de definições,  
**Para que** as comunicações automáticas reflitam o tom e informações da minha empresa.

---

## Contexto de Negócio

Os gestores querem personalizar as mensagens — adicionar o nome da propriedade, instruções específicas de check-in, contactos de emergência. O editor deve mostrar preview da mensagem final com as variáveis substituídas.

### Variáveis disponíveis por template

> **OBRIGATÓRIO:** `{{property_name}}` e `{{property_address}}` devem estar presentes em TODOS os templates. O editor deve mostrar aviso de erro se estiverem ausentes.

| Variável | Obrigatório | Descrição |
|----------|-------------|-----------|
| `{{property_name}}` | ✅ SIM | Título da propriedade |
| `{{property_address}}` | ✅ SIM | Endereço completo (rua, cidade, código postal) |
| `{{guest_name}}` | — | Nome do hóspede |
| `{{checkin_date}}` | — | Data de check-in |
| `{{checkout_date}}` | — | Data de checkout |
| `{{checkin_code}}` | — | Código de acesso |
| `{{cleaner_name}}` | — | Nome do cleaner |
| `{{task_date}}` | — | Data da tarefa |
| `{{task_time}}` | — | Hora da tarefa |
| `{{manager_phone}}` | — | Telefone do gestor |

A variável `{{property_address}}` é composta automaticamente: `{address}, {city} {postal_code}`

## Schema

```sql
CREATE TABLE whatsapp_message_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID NOT NULL REFERENCES organizations(id),
  template_key TEXT NOT NULL, -- ex: 'checkin_code', 'checkout_reminder'
  language TEXT NOT NULL DEFAULT 'pt-BR',
  body TEXT NOT NULL,
  is_custom BOOLEAN DEFAULT false, -- false = usando template padrão
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(organization_id, template_key, language)
);
```

# Story 36.8: Sidebar Properties (Web)

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.8  
**Status:** Draft  
**Story Points:** 3  
**Priority:** HIGH  

---

## User Story

As a **Host on Web**, I want **to select a property from a sidebar list** so that **I can view and manage that property's calendar and pricing**.

---

## Description

Build left sidebar (web-only) showing property list with selection state. Click property → highlights it and updates center calendar + right config panel to show that property's data.

**Why:** Web layout needs property selector; sidebar keeps all properties visible while focused on one.

---

## Acceptance Criteria

### Sidebar Structure

- [ ] Fixed width: ~25% of viewport (adjust for different screen sizes)
- [ ] Scrollable vertically if 8+ properties
- [ ] Each property item:
  - Thumbnail (small square, ~60x60px)
  - Property name (truncate if long)
  - Checkbox or radio button (single select)

### Selection State

- [ ] On click property:
  - Set as "active" (highlight: dark background or border)
  - Center calendar updates to show this property's calendar
  - Right config panel updates to show this property's settings
- [ ] Active state styling: Bold name, colored background or border

### Responsive Web

- [ ] Hidden on mobile (≤768px)
- [ ] Visible and scrollable on web (>768px)
- [ ] Padding/spacing consistent with design

### Data Binding

- [ ] Props: List of properties (from parent)
- [ ] State: activePropertyId (shared with parent or Context)
- [ ] On selection: Trigger parent callback to update calendar + config

---

## Definition of Done

- [ ] All properties listed in sidebar
- [ ] Selection works
- [ ] Calendar updates on selection
- [ ] Config panel updates on selection
- [ ] Sidebar scrollable
- [ ] Mobile-hidden (CSS media query)
- [ ] No TypeScript errors

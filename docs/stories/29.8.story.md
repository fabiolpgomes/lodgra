# Story 29.8 — Premium Gate: Feature Flag + Add-on Billing

**Epic:** 29 — Cleaner Operations Portal  
**Status:** Draft  
**Prioridade:** P2  
**Tipo:** Backend (Billing + Feature Flags)  
**Agente:** @dev  
**Depende de:** 29.1–29.7, Epic 12 (Stripe)  
**Complexity:** MEDIUM (3 story points)

---

## Story

**Como** administrador do produto,  
**Quero** bloquear o Cleaner Portal para planos Essential e disponibilizá-lo como add-on,  
**Para que** apenas clientes Expansão/Premium ou assinantes do add-on tenham acesso.

---

## Critérios de Aceitação

- [ ] **AC1** — Feature `cleaner_portal` activada para planos: `expansao`, `premium`
- [ ] **AC2** — Plano `essential`: acesso bloqueado com paywall mostrando upgrade ou add-on
- [ ] **AC3** — Add-on `cleaner_portal_addon` disponível para compra separada (Stripe product)
- [ ] **AC4** — `organizations` tem campo `cleaner_portal_enabled: boolean` (default false)
- [ ] **AC5** — Ao fazer upgrade para Expansão/Premium, campo activado automaticamente via webhook
- [ ] **AC6** — Ao assinar add-on, campo activado via webhook Stripe
- [ ] **AC7** — Middleware verifica acesso antes de renderizar qualquer rota `/cleaner/*` e `/operations/cleaning`
- [ ] **AC8** — UI de paywall: card descritivo do Cleaner Portal com CTA de upgrade
- [ ] **AC9** — Testes: feature flag, paywall render, webhook activation (4+ testes)

# Story 22.1: Booking Widgets — Limitar Hóspedes ao max_guests da Propriedade

**Epic:** 22 — Booking UX Fixes
**Status:** Done
**Prioridade:** P2 — Correctness: evitar hóspedes em excesso no formulário
**Tipo:** Frontend
**Agente:** @dev

---

## Story

**Como** gestor de propriedade,
**Quero** que o widget de reserva só permita seleccionar até ao máximo de hóspedes configurado,
**Para que** os hóspedes não consigam reservar para mais pessoas do que a propriedade suporta.

---

## Context

Os widgets `BookingWidgetDesktop` e `BookingWidgetMobile` tinham o selector de hóspedes fixo
em 1-10 independentemente do `max_guests` da propriedade. Um imóvel com capacidade para 4
pessoas mostrava 10 opções no dropdown. O `max_guests` já estava visível nas estatísticas
rápidas da página pública mas não era enforçado no widget.

---

## Acceptance Criteria

- [x] **AC1** — `BookingWidgetDesktop` aceita prop `maxGuests` e limita o selector ao máximo
- [x] **AC2** — `BookingWidgetMobile` aceita prop `maxGuests` e limita o selector ao máximo
- [x] **AC3** — `initialGuests` é clampado a `maxGuests` (previne URL manipulation)
- [x] **AC4** — `PropertyPageV2` passa `property.max_guests` aos dois widgets
- [x] **AC5** — Default de 10 quando `max_guests` é null (backward compatible)
- [x] **AC6** — Build PASS + Lint PASS

---

## Dev Notes

### Mudança nos widgets (Desktop e Mobile)
```ts
// Prop adicionada
maxGuests?: number  // default 10

// Estado inicial com clamp
const [guests, setGuests] = useState(Math.min(initialGuests, Math.max(1, maxGuests)))

// Selector gerado dinamicamente
Array.from({ length: Math.max(1, maxGuests) }, (_, i) => ...)
```

### PropertyPageV2
```tsx
maxGuests={property.max_guests ?? undefined}  // undefined → default 10
```

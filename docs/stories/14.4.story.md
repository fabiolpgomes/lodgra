# Story 14.4: PWA — Brand Tokens no Manifest + Splash Screen

**Epic:** 14 — Property Public Experience
**Status:** 🔵 Ready for Dev
**Prioridade:** P1 — Quick win, impacto imediato na identidade da marca instalada
**Tipo:** Config / Assets
**Agente:** @dev
**Esforço:** XS (~1h)
**Depende de:** Story 14.1 (Design System ✅)

---

## Story

**Como** utilizador que instala o Home Stay como PWA no telemóvel,
**Quero** ver a identidade visual correcta da marca (ocean blue, logo) na barra do browser e no splash screen,
**Para que** a experiência instalada reflicta a marca Home Stay e não pareça uma app genérica.

---

## Contexto

O `manifest.json` actual usa `theme_color: #111827` (cinza escuro genérico) e `background_color: #ffffff` (branco puro). Quando o utilizador instala o PWA:
- A barra de endereço/status bar do browser mostra `#111827` — escuro e sem identidade
- O splash screen (Android/iOS) mostra fundo branco sem contexto de marca

Com o design system definido (Story 14.1), temos os valores correctos: ocean blue como cor primária e off-white quente como background.

### Layout — meta tags

Localizar `src/app/layout.tsx` e verificar/adicionar:

```tsx
// No <head> ou via Next.js metadata:
export const metadata: Metadata = {
  // ...existing...
  themeColor: '#1567A8',
  appleWebApp: {
    capable: true,
    statusBarStyle: 'default',
    title: 'Home Stay',
  },
}
```

Se o layout usar `<meta>` directas:
```html
<meta name="theme-color" content="#1567A8" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
```

### Verificar ícones

```bash
ls public/icons/
# Deve existir: icon-192x192.png, icon-384x384.png, icon-512x512.png
```

Se os ícones forem genéricos (sem branding), anotar como dívida técnica — não bloqueia esta story.

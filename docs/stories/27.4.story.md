## Objective

Build an admin dashboard for property owners to monitor Google Vacation Rentals indexing status, view feed generation metrics, and manually trigger feed refreshes. This is a premium tier feature that provides visibility into distribution health and performance.

---

## Acceptance Criteria

- [ ] **AC1:** Dashboard page at `/admin/google-distribution` (protected, admin/premium tier only) — Route only accessible if authenticated AND user's properties.tier='premium'
- [ ] **AC2:** Display metrics: aggregated counts from google_feed_logs — (a) total properties with latest status='success', (b) pending count (status='queued' or 'in_progress'), (c) rejected count (status='failed' with age >30d), (d) error count (latest status='failed' within 30d)
- [ ] **AC3:** Show status per property table: property name, derived status ("indexed"/"pending"/"error"/"rejected"), submitted date, last updated timestamp (from google_feed_logs.timestamp), view details action
- [ ] **AC4:** Action button "Refresh Feed" → POST `/api/admin/google-feed/refresh` (optional propertyIds array, optional force flag) — returns 202 Accepted with jobId
- [ ] **AC5:** Feed generation logs section: display latest 20 entries from google_feed_logs with columns: timestamp, action (auto/manual), status, properties_count, duration_ms, error_message (if any)
- [ ] **AC6:** Premium tier gating: page redirects to pricing page if user's properties.tier !== 'premium' — RLS policy enforces SELECT access only for owned properties
- [ ] **AC7:** No breaking changes, all existing tests pass (regression: 27.2, 27.3 feed generation unaffected)

### API Endpoint

**POST /api/admin/google-feed/refresh**
```
Headers:
  Authorization: Bearer {jwt}

Body: (optional)
  {
    "propertyIds": ["uuid1", "uuid2"],  // if not provided, refresh all
    "force": true|false  // force refresh even if recently updated
  }

Response: 202 Accepted
  {
    "jobId": "job-uuid-123",
    "status": "queued",
    "timestamp": "2026-05-16T14:35:00Z",
    "propertiesCount": 258
  }

Error: 403 Forbidden
  - User not admin
  - Property not premium tier
```

- **Data Source**: Feed generation logs stored in `google_feed_logs` table. Schema:
  ```sql
  CREATE TABLE google_feed_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    property_id UUID NOT NULL REFERENCES properties(id),
    timestamp TIMESTAMP NOT NULL DEFAULT NOW(),
    action TEXT NOT NULL CHECK (action IN ('auto', 'manual')),
    status TEXT NOT NULL CHECK (status IN ('success', 'failed', 'queued')),
    duration_ms INTEGER,
    error_message TEXT,
    properties_count INTEGER,
    created_at TIMESTAMP DEFAULT NOW()
  );
  ```
- **Status Determination**: Status per property is derived from feed generation logs + last feed timestamp. Statuses: "indexed" (recent success), "pending" (queued or in progress), "error" (last attempt failed), "rejected" (never succeeded)
- **Premium Gating**: Implement using Supabase RLS policy `tier = 'premium'` — same pattern used in existing admin pages
- **Async Refresh**: Use Bull.js (npm package `bull@^4.0.0`) for async feed refresh queue. Configuration: concurrency=1, max retries=2, job timeout=60s
- **UI Components**: Reuse existing admin layout components: `AdminLayout` from `/src/layouts/AdminLayout.tsx`, `Table` from `/src/components/ui/Table.tsx`, `Card` from `/src/components/ui/Card.tsx`
- **Error Handling**: Feed generation errors should be logged to Sentry and displayed with clear messages (timeout, validation error, etc.)

## Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| **Async Job Queueing Complexity** — Bull.js configuration (concurrency=1, max retries=2, timeout=60s) impacts throughput under load | MEDIUM | Load test with 500+ property refresh; @dev to monitor job queue depth; fallback to sync if queue backs up |
| **RLS Policy Correctness** — Premium tier check must be enforced across all queries (dashboard read, refresh write) | HIGH | @data-engineer to review and validate RLS policy before merge; @qa to test unauthorized access (non-premium users should 403) |
| **Large Dashboard Data** — Metrics aggregation and status table for 500+ properties may timeout or slow render | MEDIUM | Implement pagination for status table (50 per page); cache metrics for 5 min; @dev to profile dashboard render time with 500+ properties |
| **Feed Log Table Growth** — google_feed_logs will grow unbounded without retention policy | LOW | Implement archival: retain logs for 90 days, archive older; monitor table size quarterly |
| **Property Status Derivation Logic** — Status determination from feed logs is complex ("indexed" vs "pending" vs "error" vs "rejected") | MEDIUM | Unit tests for status calculation function (4 test cases min); @qa to verify correctness with sample data |

# Story 36.7: Card List (Mobile)

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.7  
**Status:** Draft  
**Story Points:** 5  
**Priority:** HIGH  

---

## User Story

As a **Host on Mobile**, I want **to see all my properties in a card list** so that **I can quickly select one to view its calendar**.

---

## Description

Build mobile card list showing all properties with mini calendar visual (occupied/available days). Each card is a property thumbnail. Tap card → navigate to detailed calendar view.

**Why:** Card layout is mobile-native (Instagram/TikTok style), easier to scan than lists.

---

## Acceptance Criteria

### Card Structure

- [ ] Thumbnail photo (rounded corners, 4:3 aspect ratio, ~300x225px)
- [ ] Property name (truncate if long)
- [ ] Location (city/area)
- [ ] Mini calendar visual:
  - 4 rows × 7 columns (28 dots)
  - Each dot represents 1 day
  - ● (black/filled) = occupied/booked
  - • (light) = available
  - Red dot = today

### Card List Behavior

- [ ] Vertical scroll (one column of cards)
- [ ] All properties loaded at once (no pagination)
- [ ] Pull-to-refresh: Reload property list + reservations
- [ ] Smooth scroll animation

### Tap Interaction

- [ ] Tap card → Navigate to detailed calendar (Story 36.9)
- [ ] Pass property ID as route param
- [ ] Card loading: Show skeleton while fetching data

### Mobile Responsiveness

- [ ] Full-width cards (with padding)
- [ ] Touch target: ≥44px height
- [ ] Safe area padding (for notch devices)

---

## Definition of Done

- [ ] All properties display as cards
- [ ] Photos load quickly (lazy loading or cached)
- [ ] Mini calendar dots accurate
- [ ] Tap navigation works
- [ ] Pull-to-refresh works

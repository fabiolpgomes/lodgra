## Context

Auditoria pós-Epic 19.4 revelou 2 bugs e 1 melhoria no fluxo de checkout directo:

1. **BUG CRÍTICO** — `blockedRanges` query em `page.tsx` usa colunas erradas (`start_date`/`end_date`) e filtro errado (`property_id`) para a tabela `reservations`. As reservas directas (Stripe) nunca aparecem como datas bloqueadas no date picker público porque a query devolve sempre `[]`.
2. **BUG menor** — `BookingSummary` e `booking-confirmed` hardcodam `€` — quebrado para propriedades BRL/USD.
3. **INFO** — Webhook `/api/stripe/booking-webhook/route.ts` + emails `bookingConfirmationGuest.ts` já existem e estão implementados.

## Dev Notes

### Fix 1 — blockedRanges query (page.tsx)

```ts
// Two-step: listing IDs first, then reservations
const { data: propertyListingsData } = await adminClient
  .from('property_listings')
  .select('id')
  .eq('property_id', property.id)

const listingIds = (propertyListingsData ?? []).map((l: { id: string }) => l.id)

let blockedRanges: { start: string; end: string }[] = []
if (listingIds.length > 0) {
  const { data: blockedReservationsRaw } = await adminClient
    .from('reservations')
    .select('check_in, check_out')
    .in('property_listing_id', listingIds)
    .in('status', ['confirmed', 'pending_payment', 'pending'])
    .gte('check_out', today)
    .lte('check_in', futureDate)

  blockedRanges = (blockedReservationsRaw ?? []).map(
    (r: { check_in: string; check_out: string }) => ({ start: r.check_in, end: r.check_out })
  )
}
```

## Story

**Como** gestor de propriedades de short-term rental,
**Quero** ver a taxa de ocupação e o RevPAR (Revenue Per Available Room) nos relatórios,
**Para que** possa avaliar a eficiência real das minhas propriedades — não apenas a receita bruta, mas quanto estou a extrair do potencial disponível.

---

## Contexto de Negócio

O ADR (Average Daily Rate) já existe no dashboard. Mas sem RevPAR e ocupação, o utilizador não sabe se está a ganhar €150/noite porque tem 30% de ocupação ou 80%. São métricas distintas e complementares:

- **Taxa de Ocupação** = noites reservadas ÷ noites disponíveis no período × 100
- **RevPAR** = receita total ÷ noites disponíveis no período (ou ADR × Ocupação%)

## Dev Notes

### Cálculo de Noites Disponíveis

O cálculo de `noites_disponíveis` deve ser feito assim:

```typescript
// dias no período × número de propriedades activas com escopo do utilizador
const periodDays = Math.ceil(
  (new Date(endDate).getTime() - new Date(startDate).getTime()) / (1000 * 60 * 60 * 24)
)
const totalAvailableNights = periodDays * numberOfProperties

// Se filtrado por propriedade específica:
const totalAvailableNights = periodDays * 1
```

### Fórmulas

```typescript
// RevPAR (por moeda)
const revpar = totalAvailableNights > 0
  ? revenue / totalAvailableNights
  : 0

// Taxa de ocupação
const occupancyRate = totalAvailableNights > 0
  ? (totalNightsBooked / totalAvailableNights) * 100
  : 0
```

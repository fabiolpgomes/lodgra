# Story 15.2: Webhook Listener Booking.com

## Status
Ready for Review

## Story

**As a** gestor de propriedades,  
**I want** que novas reservas do Booking.com apareçam automaticamente no dashboard com dados completos,  
**so that** não precise de introduzir manualmente nome do hóspede, valor ou datas.

## Acceptance Criteria

1. Endpoint `POST /api/webhooks/booking/reservation` processa eventos de nova reserva, modificação e cancelamento
2. Validação de assinatura HMAC-SHA256 obrigatória — rejeita pedidos sem assinatura válida (401)
3. Idempotência: mesmo evento enviado 2x não duplica reserva (retorna 200 sem criar duplicado)
4. Cria ou actualiza `guest` (upsert por `guest_email` dentro da org) e `reservation` com todos os campos disponíveis
5. Guarda `raw_data` (jsonb) com payload completo do Booking.com
6. Responde em < 3 segundos (Booking.com tem timeout curto — processamento pesado delegado)
7. Logs de erro estruturados via Sentry (`captureException` com contexto)
8. Funciona em modo sandbox com credenciais de teste do Booking.com Connectivity Program

### Padrão de API Route (Next.js App Router)
```typescript
import { NextRequest, NextResponse } from 'next/server'
import { createAdminClient } from '@/lib/supabase/admin'

export async function POST(request: NextRequest) {
  // 1. Ler raw body para HMAC
  const rawBody = await request.text()
  const signature = request.headers.get('x-booking-signature') ?? ''
  
  // 2. Verificar assinatura
  if (!verifyBookingSignature(rawBody, signature, process.env.BOOKING_WEBHOOK_SECRET!)) {
    return NextResponse.json({ error: 'Invalid signature' }, { status: 401 })
  }
  
  // 3. Parsear e processar
  const event = JSON.parse(rawBody)
  // ...
  return NextResponse.json({ ok: true })
}
```

### CRÍTICO: Raw Body para HMAC
O Next.js App Router lê o body como stream. Para HMAC:
```typescript
const rawBody = await request.text() // antes de qualquer .json()
```

### Determinar `organization_id` no webhook
O webhook não tem sessão de utilizador. Estratégia:
```typescript
// Via channel_listings → property_listing → property → organization_id
const { data: listing } = await adminClient
  .from('channel_listings')
  .select('property_listing:property_listings(property:properties(organization_id))')
  .eq('external_id', event.listing_id)
  .single()
const org_id = listing?.property_listing?.property?.organization_id
```

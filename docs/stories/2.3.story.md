## Contexto de Negócio

O site tem uma estrutura de conteúdo público plana:
- `/` (home)
- `/features` (funcionalidades)
- `/pricing` (planos)
- `/docs` (documentação)
- `/blog` (blog — futura)
- `/p/[slug]` (propriedades públicas)

Faltam links internos estratégicos:
- Home não linkam para Features/Pricing
- Features/Pricing não linkam de volta para Home
- Propriedades públicas não estão linkadas da home/features
- Blog (quando existir) não terá links cruzados

**Benefícios:**
- Distribuição de PageRank
- Melhor rastreamento de Google
- Melhora de UX (navegação intuitiva)
- Oportunidades de anchor text otimizado

### 3. Property Cards — Similar Properties

Em `src/app/p/[slug]/page.tsx`, adicionar secção de propriedades similares:

```typescript
export default async function PropertyPage({ params }: { params: { slug: string } }) {
  const property = await getPropertyBySlug(params.slug)
  const similarProperties = await getSimilarProperties(params.slug, {
    location: property.location,
    limit: 5,
    exclude: property.id
  })

  return (
    <>
      {/* Property content */}
      
      <section className="similar-properties">
        <h2>Propriedades Similares</h2>
        <div className="grid">
          {similarProperties.map((p) => (
            <Link key={p.id} href={`/p/${p.slug}`}>
              <PropertyCard property={p} />
            </Link>
          ))}
        </div>
      </section>
    </>
  )
}

async function getSimilarProperties(
  excludeSlug: string,
  options: { location: string; limit: number; exclude: string }
) {
  // Query: propriedades na mesma localização, excluindo current
  return supabase
    .from('property_listings')
    .select('*')
    .ilike('location', `%${options.location}%`)
    .neq('id', options.exclude)
    .limit(options.limit)
    .order('rating', { ascending: false })
}
```

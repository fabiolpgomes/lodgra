## Story

**Como** potencial hóspede,
**Quero** ver um calendário com as datas disponíveis e bloqueadas da propriedade,
**Para que** possa escolher as datas do meu estadia antes de reservar.

---

## Contexto de Negócio

O calendário público é o ponto de entrada do fluxo de reserva directa. Deve mostrar em tempo real quais as datas disponíveis (livres), bloqueadas (reservadas via Airbnb/Booking/iCal) e o preço por noite (Story 9.3 fornece o preço, mas se não existir mostrar preço base).

O calendário já existe no dashboard interno (`FullCalendar`). O calendário público é mais simples — apenas selecção de datas para check-in/check-out.

---

## Critérios de Aceitação

- [ ] **AC1** — Endpoint público `GET /api/public/properties/[slug]/availability?year=YYYY&month=MM` retorna array de datas bloqueadas e disponíveis (sem autenticação)
- [ ] **AC2** — Calendário visual mostra datas bloqueadas (cinzento/riscado) e disponíveis (branco/verde)
- [ ] **AC3** — Hóspede pode seleccionar check-in e check-out; validação: check-out > check-in, mínimo de noites (via pricing rules se existirem, senão 1 noite)
- [ ] **AC4** — Ao seleccionar datas válidas, mostra resumo de preço (noites × preço/noite + total estimado)
- [ ] **AC5** — Botão "Reservar" aparece após selecção válida e leva ao fluxo de checkout (Story 9.4)
- [ ] **AC6** — Calendário é mobile-friendly (swipe para mudar mês, touch-friendly)
- [ ] **AC7** — Datas passadas são sempre bloqueadas
- [ ] **AC8** — Rate limiting no endpoint público (50 req/min por IP)

## Dev Notes

### Endpoint público de disponibilidade
```
src/app/api/public/properties/[slug]/availability/route.ts
```

```typescript
// GET /api/public/properties/[slug]/availability?year=2026&month=4
// Response:
{
  "blocked": ["2026-04-01", "2026-04-02", "2026-04-10"],
  "base_price": 100,
  "min_nights": 2
}
```

**Lógica de bloqueio:**
1. Buscar `property_id` pelo slug
2. Buscar reservas activas (status != 'cancelled') no mês/período
3. Buscar `availability` table se existir (datas marcadas como indisponíveis manualmente)
4. Retornar array de todas as datas bloqueadas

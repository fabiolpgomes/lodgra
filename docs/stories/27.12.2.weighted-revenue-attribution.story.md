# Story 27.12.2 — Weighted Revenue Attribution Model

**Epic:** 27 — Google Vacation Rentals Integration  
**Status:** Draft  
**Story ID:** 27.12.2  
**Created:** 2026-07-17  
**Type:** Backend / Analytics  
**Depends On:** Story 27.12.1 (Platform Sync Jobs)

---

## 📋 Story

**As a** property manager  
**I want** to understand revenue contribution by platform with accurate attribution  
**So that** I can make data-driven decisions about channel investment

---

## 🎯 Acceptance Criteria

### AC1: Revenue Attribution Methodology
- [ ] Define attribution model: first-click, last-click, linear, or position-based
- [ ] Document methodology in code comments
- [ ] Support different models (configurable)
- [ ] Validate against booking source data

### AC2: Multi-Touch Attribution
- [ ] Track customer journey: impression → click → booking
- [ ] Attribute revenue to all touchpoints (not just last click)
- [ ] Calculate contribution percentage per platform
- [ ] Handle direct bookings separately

### AC3: Currency Normalization
- [ ] Convert all platform revenue to base currency (USD)
- [ ] Apply historical exchange rates for accuracy
- [ ] Store original currency in audit trail
- [ ] **Success Metric:** Revenue totals within ±2% of actual

### AC4: Time Zone Alignment
- [ ] All metrics standardized to UTC
- [ ] Handle platform-specific timezone offsets
- [ ] Document timezone rules per platform
- [ ] Reconcile cross-timezone bookings

### AC5: Anomaly Detection
- [ ] Flag unusual revenue patterns
- [ ] Alert if platform revenue suddenly drops >20%
- [ ] Track year-over-year trends
- [ ] Suggest investigation areas

### AC6: Reporting Enhancements
- [ ] Revenue attribution report by platform
- [ ] Channel profitability analysis
- [ ] ROI calculation per channel
- [ ] **Success Metric:** Reports accurate to ±1%

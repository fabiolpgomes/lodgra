# Story 31.4: Branded Email Confirmations

**Epic:** Epic 31 — Multi-Tenant Infrastructure (Branded Direct Booking)  
**Status:** Draft  
**Priority:** High  
**Type:** Backend + Email Templates  
**Assigned to:** @dev  
**Complexity:** Medium (T-shirt: M)

---

## Story

**As** a Lodgra product owner,  
**I want** companies to receive booking confirmation emails with their own branding (logo, colors, custom message),  
**So that** customers feel they're booking with the company, not through a third-party platform.

---

## Context

Stories 31.2 and 31.3 customize the booking page visually. Story 31.4 extends that to email communications. When a customer books through `pousada.lodgra.io/booking`, they should receive a confirmation email branded as from "Pousada" with Pousada's logo and colors.

**Current State:**
- Booking confirmations sent via email service (likely SendGrid or similar)
- Emails use Lodgra branding
- No company-level customization

**Desired State:**
- Emails dynamically rendered with company branding
- Company logo in email header
- Company colors applied to email buttons/accents
- Company can customize confirmation message
- Custom from-address optional (e.g., noreply@pousada.lodgra.io)

## Dev Notes

### Technical Approach

**1. Database Migration**
```sql
CREATE TABLE organization_email_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID NOT NULL UNIQUE REFERENCES organizations(id) ON DELETE CASCADE,
  confirmation_subject TEXT DEFAULT 'Booking Confirmation',
  confirmation_message TEXT,
  from_email TEXT DEFAULT 'noreply@lodgra.io',
  from_name TEXT,
  reply_to_email TEXT,
  include_company_logo BOOLEAN DEFAULT true,
  footer_text TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE email_sent (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID REFERENCES organizations(id),
  booking_id UUID REFERENCES bookings(id),
  customer_email TEXT NOT NULL,
  template_type TEXT DEFAULT 'confirmation',
  sent_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

**2. Email Template Rendering**
- Use mjml library (MJML) for email markup
- Create template files: `src/lib/email/templates/booking-confirmation.mjml`
- Template supports Handlebars syntax for variables
- Render with company branding colors/logo

**3. Backend Logic**
```typescript
// Pseudocode
async function sendBookingConfirmation(booking, organization) {
  const template = await getEmailTemplate(organization.id);
  const branding = await getBranding(organization.id);
  
  const emailHtml = renderMJML('booking-confirmation.mjml', {
    ...booking,
    ...template,
    logo: branding.logo_url,
    primaryColor: branding.primary_color,
  });
  
  await sendEmail({
    from: template.from_email,
    to: booking.customer_email,
    subject: template.confirmation_subject,
    html: emailHtml,
  });
  
  await logEmailSent(organization.id, booking.id, booking.customer_email);
}
```

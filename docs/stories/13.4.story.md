# Story 13.4 — SEO Dinâmico para Páginas Públicas

## Objetivo
Melhorar a indexação Google e tráfego orgânico através de sitemap dinâmico, metadata enriquecido com Open Graph/Twitter Cards, e structured data (JSON-LD) nas páginas públicas de propriedades.

## Acceptance Criteria

- [x] Sitemap dinâmico inclui todas as `/p/[slug]` activas (query ao Supabase)
- [x] Sitemap inclui variantes i18n (`/pt/p/[slug]`, `/en-US/p/[slug]`, `/pt-BR/p/[slug]`)
- [x] `generateMetadata()` em `/p/[slug]` com Open Graph completo (title, description, images, type)
- [x] Twitter Card meta tags (summary_large_image)
- [x] JSON-LD structured data: `LodgingBusiness` schema em `/p/[slug]`
- [x] JSON-LD na landing page: `Organization` + `WebSite` schema
- [x] `hreflang` alternates para todas as locales em páginas públicas
- [x] Canonical URLs correctas por locale (evitar conteúdo duplicado)
- [x] `robots.ts` actualizado: permitir `/p/` para crawlers
- [x] Build PASS, Lint PASS

## Notas Técnicas

- Sitemap usa `createAdminClient()` para bypass RLS (precisa de ver todas as properties públicas)
- JSON-LD usa `LodgingBusiness` (schema.org — mais suportado que VacationRental)
- `alternates.languages` em metadata + sitemap para hreflang
- WebSite JSON-LD inclui SearchAction potentialAction
- robots.ts agora bloqueia `/dashboard`, `/auth/`, `/account`, `/settings`, `/monitoring`

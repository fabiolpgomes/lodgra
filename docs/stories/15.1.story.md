# Story 15.1: Schema Multi-Channel (Base de Dados)

## Status
Ready for Review

## Story

**As a** gestor de propriedades,  
**I want** o sistema a armazenar dados completos de reservas provenientes de múltiplos canais (Booking.com, Airbnb, canal directo),  
**so that** cada reserva tenha nome do hóspede, contacto, valor e status sem necessidade de input manual.

## Acceptance Criteria

1. Migração cria tabela `channels` com registos seed: "booking" e "direct"
2. Tabela `guests` criada com campos: `id`, `organization_id`, `name`, `email`, `phone`, `created_at`
3. Tabela `channel_listings` criada ligando `property_listings` ao ID externo do canal
4. Tabela `reservations` tem colunas novas: `raw_data` (jsonb), `channel_id` (fk → channels), `guest_name` (text), `guest_email` (text), `guest_phone` (text)
5. RLS policies aplicadas em `guests` e `channel_listings` (isolamento por `organization_id`)
6. Reservas existentes não são afectadas (novas colunas nullable ou com default)
7. Script de rollback documentado e testado

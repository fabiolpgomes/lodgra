## User Story

As a **Host**, I want **to monitor competitor prices for similar properties in my area** so that **I can understand market positioning and adjust my pricing competitively**.

---

## Description

Build a competitor price monitoring feature that tracks pricing for similar properties. Hosts manually add competitor URLs (Airbnb, Booking.com, VRBO) and receive alerts when prices change significantly. Includes market position analysis (higher/lower than market average) and benchmark reports.

**Why:** Hosts need market intelligence to price competitively without manually checking competitor sites daily.

## Risks

| Risk | Mitigation |
|------|-----------|
| Web scraping blocked by platforms | Use cached data, graceful degradation, document limitation |
| Rate limiting from competitor sites | Implement backoff strategy, distribute scraping across time |
| Incorrect price extraction | Manual validation, support for manual price entry |
| Competitor URL invalid/removed | Validate before storing, graceful error handling, retry logic |
| Privacy concerns with scraping | Clear TOS, document that data is public-facing, no personal data stored |

---

## Technical Notes

- Use Puppeteer or Cheerio for web scraping (evaluate for performance)
- Schedule scraper using node-schedule or similar
- Store price history in DB for trend analysis
- Market average: exclude outliers (top/bottom 10%), use median
- Confidence score: based on sample size (min 3 competitors for meaningful data)
- Alert deduplication: don't re-alert if price changes within same day
- Cache market analysis calculations (valid for 24 hours)
- CSV export: use `papaparse` library
- PDF export: use `jsPDF` library
- Mobile alerts: use toast notifications or bottom sheet

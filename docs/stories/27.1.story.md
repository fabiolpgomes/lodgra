## Objective

Add structured data (Schema.org markup) to public property pages (`/properties/[slug]`) to enable Google's Rich Results and SEO visibility. This is a zero-code-change integration that improves discoverability without modifying the page layout.

---

## Acceptance Criteria

- [x] **AC1:** LodgingBusiness + LocalBusiness schema injected into `<Head>` of `/properties/[slug]` pages
- [x] **AC2:** Schema includes required fields: name, description, address, phone, email, image, priceRange
- [x] **AC3:** AggregateRating included from OTA reviews (only if property has reviews, optional field)
- [x] **AC4:** OpeningHoursSpecification for booking availability (check-in 14:00, check-out 11:00 per default)
- [ ] **AC5:** Markup passes Google Rich Results Test (`https://search.google.com/test/rich-results`)

## Implementation Details

### Location
- **Primary file:** `/pages/properties/[slug].tsx` (add schema generation) OR create `/lib/schema-generators.ts` (new utility)
- **Output:** `<script type="application/ld+json">` in Next.js `<Head>` or via `next-seo` if available

### Data Source
- Property data: `properties` table (ID, name, description, slug, address, city, zipcode, phone, email)
- Images: `property_media` table (photo URLs, WebP format via Next.js Image optimization)
- Reviews: `property_reviews` table (rating, source, if available, aggregate per property)
- Pricing: `property_pricing` table or default pricing tier

### JSON-LD Schema Example
```json
{
  "@context": "https://schema.org",
  "@type": "LodgingBusiness",
  "name": "Cozy Beach Villa in Algarve",
  "description": "Stunning beachfront property with sea views...",
  "image": [
    "https://cdn.lodgra.app/properties/uuid/photo1.webp",
    "https://cdn.lodgra.app/properties/uuid/photo2.webp"
  ],
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "Rua da Praia 42",
    "addressLocality": "Lagos",
    "postalCode": "8600-001",
    "addressCountry": "PT"
  },
  "telephone": "+351 300 000 000",
  "email": "contact@beachvilla.pt",
  "priceRange": "€50-€200",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.7",
    "reviewCount": "23"
  },
  "openingHoursSpecification": {
    "@type": "OpeningHoursSpecification",
    "dayOfWeek": ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday"],
    "opens": "14:00",
    "closes": "11:00"
  }
}
```

### Implementation Approach
1. **Create schema generator utility** (`/lib/schema-generators.ts`):
   - Function `generateLodgingBusinessSchema(property, reviews)` that returns JSON-LD object
   - Handle edge cases: missing fields, no reviews, no images
   - Validate schema structure before returning

2. **Integrate into property page** (`/pages/properties/[slug].tsx`):
   - Server-side generation (Next.js getServerSideProps or async component)
   - Pass schema to `<Head>` via `next-seo` or Next.js native `<Head>` component
   - Ensure schema is always valid JSON

3. **Security & Privacy:**
   - NO PII in schema (email/phone should be minimal, check legal requirements)
   - Images must use signed CDN URLs if property is private
   - Validate no sensitive data leaks to public markup

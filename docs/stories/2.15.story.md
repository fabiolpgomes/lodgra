**Status:** Done

## 📖 Story

Como **administrador/gestor de propriedades**, quero visualizar cálculos corretos de:
1. **Previsão de Faturamento** — totalizações por moeda
2. **Receita do Mês** — com distribuição proporcional para reservas > 30 dias
3. **Lucro Real** — receita menos despesas

Para que **possa gerenciar financeiro com precisão**, entendendo a distribuição correta de receita entre meses.

---

## ✅ Acceptance Criteria

### AC1: Previsão de Faturamento (por moeda)
```gherkin
Given um sistema com reservas em múltiplas moedas (EUR, BRL, USD)
When o gestor visualiza "Previsão de Faturamento" no dashboard
Then vê o total de TODAS as reservas existentes agrupadas por moeda
And mostra formato: EUR €5.234,50 | BRL R$12.890,30 | USD $1.230,00
And total acumula até o dia atual de consulta
And atualiza em tempo real conforme novas reservas são criadas
```

### AC2: Receita do Mês (Reservas ≤ 30 dias)
```gherkin
Given uma reserva com duração ≤ 30 noites
When o sistema calcula "Receita do Mês"
Then a receita INTEGRAL é contabilizada no mês de check-in
Example: Reserva 5 noites no Maio = R$1.000 total
         Receita Maio = R$1.000
```

### AC3: Receita do Mês (Reservas > 30 dias) — Cálculo Proporcional
```gherkin
Given uma reserva com duração > 30 noites
When o sistema calcula "Receita do Mês"
Then aplica fórmula: Receita Mensal = (Total / Dias) × 30
And distribui saldo em meses seguintes usando mesma fórmula
And preenchimento segue exatamente os dias do mês

Example 1 (com 94 noites):
  Reserva: R$17.230,00 | 94 noites (02/05/2026 - 04/08/2026)
  
  Maio (29 dias):    Receita = 17.230 / 94 × 29  = R$5.318,08
  Saldo pós-Maio:    17.230 - 5.318,08             = R$11.911,92
  
  Junho (30 dias):   Receita = 11.911,92 / 65 × 30 = R$5.498,58
  Saldo pós-Junho:   11.911,92 - 5.498,58          = R$6.413,34
  
  Julho (30 dias):   Receita = 6.413,34 / 35 × 30  = R$5.496,29
  Saldo pós-Julho:   6.413,34 - 5.496,29           = R$917,05
  
  Agosto (4 dias):   Receita = 917,05 / 5 × 4      = R$733,64
  
  Verificação: 5.318,08 + 5.498,58 + 5.496,29 + 733,64 = R$17.046,59 (~R$17.230)
  (pequena diferença por arredondamento é aceitável)

Example 2 (com 64 noites):
  Reserva: €4.500,00 | 64 noites (10/06/2026 - 12/08/2026)
  
  Junho (20 dias):   Receita = 4.500 / 64 × 20     = €1.406,25
  Saldo pós-Junho:   4.500 - 1.406,25              = €3.093,75
  
  Julho (30 dias):   Receita = 3.093,75 / 44 × 30  = €2.107,95
  Saldo pós-Julho:   3.093,75 - 2.107,95           = €985,80
  
  Agosto (14 dias):  Receita = 985,80 / 14 × 14    = €985,80
```

### AC4: Distribuição de Receita por Mês (Tela)
```gherkin
Given o usuário acessa o dashboard em qualquer data
When visualiza "Receita do Mês"
Then vê colunas: [Mês | Valor Real | Valor Previsto]
And "Valor Real" = receita do mês corrente (cálculo proporcional)
And "Valor Previsto" = receita dos meses futuros (já calculada pela fórmula)
And mostra por moeda separadamente

Example visual:
┌─ RECEITA MAIO 2026 ─────────────────┐
│ EUR                                 │
│  Valor Real:      €5.234,50        │
│  Valor Previsto:  €3.450,20        │
│                                     │
│ BRL                                 │
│  Valor Real:      R$12.890,30      │
│  Valor Previsto:  R$8.234,10       │
└─────────────────────────────────────┘
```

### AC5: Lucro Real (Mês Atual)
```gherkin
Given "Receita do Mês" já calculada (AC3)
And "Despesas" registradas no sistema
When o gestor visualiza "Lucro Real"
Then mostra: Lucro = Receita (valor real) - Despesas
And agrupa por moeda
And mostra formato claro: "Lucro Real: EUR €1.234,50"
And suporta despesas: comissões, taxas de plataforma, outros custos

Example:
  Receita EUR (Maio):  €5.234,50
  Despesas:
    - Comissão Booking:     €347,50
    - Taxa Stripe:          €89,30
    - Manutenção:           €50,00
  Lucro Real = €5.234,50 - (€347,50 + €89,30 + €50,00) = €4.747,70
```

### AC6: Aplicação Consistente em Toda a Dashboard
```gherkin
Given qualquer seção do dashboard que exiba valores financeiros
When mostra "Receita" ou "Valor"
Then diferencia claramente entre:
  - [VALOR REAL] = receita de hoje/do mês corrente
  - [VALOR PREVISTO] = receita futura (próximos meses)
And usa cor/marcação visual consistente em toda a UI
And tooltips explicam "Baseado em fórmula proporcional para reservas > 30 dias"
```

### AC7: Relatórios & Exportação
```gherkin
Given dados de receita calculados corretamente
When usuário tenta exportar relatório (CSV/PDF)
Then exporta com colunas:
  [Data | Reserva ID | Check-in | Check-out | Duração | Moeda | Valor Total | Receita Mês | Saldo Previsto]
And valores batem com cálculos da tela
And permite filtro por moeda
And permite filtro por período
```

## 📝 Dev Notes

1. **Fórmula crítica:** `(Total / Dias) × 30` por mês
2. **Cuidado com Leap Years:** Fevereiro tem 28 dias (não-bissexto) ou 29 dias (bissexto). Usar `new Date(year, month + 1, 0).getDate()` para obter exatamente quantos dias tem o mês. Testar com reservas atravessando 29/2.
3. **Performance:** Cache cálculos por 1 hora (cache key: `revenue:${currency}:${month}`)
4. **Testes:** Incluir múltiplas moedas, limites de data, anos bissextos
5. **Integração:** Ensure despesas são importadas de tabela `expenses` (confirmar schema com @data-engineer)

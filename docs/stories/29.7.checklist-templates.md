# Story 29.7 — Checklist Templates: Manager Configuration

**Epic:** 29 — Cleaner Operations Portal  
**Status:** Ready  
**Prioridade:** P2 — Feature (customização)  
**Tipo:** Frontend + API  
**Agente Executor:** @dev (Dex)

## Story

**Como** manager,  
**Quero** criar templates de checklist personalizados,  
**Para que** cada propriedade tenha itens específicos.

---

## Acceptance Criteria

### AC1: Page `/[locale]/cleaning/manage/templates` com CRUD
- [ ] **GIVEN** manager authenticated
- **WHEN** acessa `/pt-BR/cleaning/manage/templates`
- **THEN** vê lista de templates + "Create New" button

### AC2: Form criar template (name, description, property, items)
- [ ] **WHEN** clica "Create New"
- **THEN** form abre com campos:
  - name (required, text)
  - description (optional, textarea)
  - property_id (optional, select — null = global org)
  - items list (initially empty, + button to add)

### AC3: Items: label, category, is_required, order_index
- [ ] **WHEN** adiciona item
- **THEN** form tem campos: label, category (select: quarto/banheiro/cozinha/geral), is_required checkbox
- [ ] **AND** item é adicionado à lista

### AC4: Reorder items (drag-drop)
- [ ] **GIVEN** 3+ items
- **WHEN** arrasta item para nova posição
- **THEN** order_index atualiza (salvo em DB)

### AC5: Salva como `cleaning_checklist_templates` + items
- [ ] **WHEN** clica "Salvar"
- **THEN** POST `/api/cleaning/templates`
- [ ] **AND** template criado com items

### AC6: Editar template (atualiza items)
- [ ] **GIVEN** template existente
- **WHEN** clica "Editar"
- **THEN** form pre-filled com items
- [ ] **AND** pode adicionar, remover, reordenar items
- [ ] **AND** PATCH `/api/cleaning/templates/[id]` ao salvar

### AC7: Deletar template
- [ ] **GIVEN** template sem tasks usando-o
- **WHEN** clica "Deletar"
- **THEN** popup confirm, após DELETE `/api/cleaning/templates/[id]`
- [ ] **AND** template removido (se nenhuma task ativa o usa)

### AC8: Template é reutilizável em múltiplas tasks
- [ ] **GIVEN** template criado
- **WHEN** manager cria nova task em 29.3
- **THEN** template aparece no select `checklist_template_id`

### AC9: Default template com 10 items padrão
- [ ] **GIVEN** organização nova sem templates
- **WHEN** manager tenta criar task
- **THEN** default template disponível (10 items padrão: trocar roupa, banheiro, etc.)

### AC10: Indicador se template está em uso
- [ ] **WHEN** lista templates
- **THEN** mostra "Used by 5 tasks" ou "Not in use"
- [ ] **AND** "Delete" bloqueado se em uso

## Integration Verification

### IV1: RLS garante org isolation
- [ ] **GIVEN** template de org-1
- **WHEN** manager de org-2 tenta acessar
- **THEN** 403

### IV2: Não quebra workflow de tasks
- [ ] **WHEN** edita template
- **THEN** tasks já criadas com esse template não afetadas (histórico)
- [ ] **BUT** novas tasks usam items atualizados

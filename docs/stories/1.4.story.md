**Epic:** Relatórios MVP para Lançamento
**Status:** Done

## Story

**Como** proprietário de imóveis,
**Quero** ver uma previsão de receitas futuras baseada nas reservas confirmadas,
**Para que** possa planear o meu fluxo de caixa e tomar decisões financeiras com antecedência.

---

## Contexto de Negócio

Reservas futuras já existem na base de dados com `status = 'confirmed'` e datas `check_in > hoje`. Actualmente os relatórios só mostram o passado. Este relatório não requer ML nem estimativas — usa apenas dados reais já confirmados. É um dos relatórios mais valorizados por investidores e proprietários.

---

## Critérios de Aceitação

- [x] **AC1** — Nova secção "Previsão" ou tab "Fluxo de Caixa" em `/reports` exibe receitas futuras confirmadas
- [x] **AC2** — A previsão está organizada em 3 horizontes:
  - **Próximos 30 dias**
  - **31–60 dias**

  - Receita prevista (total_amount das reservas confirmadas)

- [x] **AC6** — Os filtros de propriedade afectam a previsão (filtro de data NÃO se aplica — a previsão é sempre a partir de hoje)
- [x] **AC7** — Reservas passadas não confirmadas (status ≠ 'confirmed') não aparecem na previsão
- [x] **AC8** — Os dados podem ser exportados para Excel

### Query separada para reservas futuras

Em `reports/page.tsx`, adicionar **query adicional** paralela às existentes:

```typescript
// Query de reservas futuras (a partir de hoje)
const today = new Date().toISOString().split('T')[0]

let futureReservationsQuery = supabase
  .from('reservations')
  .select(`
    id,
    check_in,
    check_out,
    total_amount,
    currency,
    source,
    status,
    property_listings!inner(
      property_id,
      properties!inner(
        id,
        name
      )
    ),
    guests(
      first_name,
      last_name
    )
  `)
  .gte('check_in', today)
  .eq('status', 'confirmed')
  .order('check_in', { ascending: true })

if (propertyId) {
  futureReservationsQuery = futureReservationsQuery
    .eq('property_listings.property_id', propertyId)
}
if (userPropertyIds) {
  futureReservationsQuery = futureReservationsQuery
    .in('property_listings.property_id', userPropertyIds)
}

// Executar em paralelo com as outras queries
const [reservationsResult, expensesResult, futureReservationsResult] = await Promise.all([
  reservationsQuery,
  expensesQuery,
  futureReservationsQuery,
])
```

### Cálculo dos 3 horizontes

```typescript
const now = new Date()
const day30 = new Date(now); day30.setDate(now.getDate() + 30)
const day60 = new Date(now); day60.setDate(now.getDate() + 60)
const day90 = new Date(now); day90.setDate(now.getDate() + 90)

const futureReservations = futureReservationsResult.data || []

const horizon30 = futureReservations.filter(r => new Date(r.check_in) <= day30)
const horizon60 = futureReservations.filter(r =>
  new Date(r.check_in) > day30 && new Date(r.check_in) <= day60
)
const horizon90 = futureReservations.filter(r =>
  new Date(r.check_in) > day60 && new Date(r.check_in) <= day90
)

function summarize(reservations: typeof futureReservations) {
  return reservations.reduce((acc, r) => {
    const nights = Math.ceil(
      (new Date(r.check_out).getTime() - new Date(r.check_in).getTime()) / (1000 * 60 * 60 * 24)
    )
    acc.revenue += r.total_amount ? Number(r.total_amount) : 0
    acc.reservations += 1
    acc.nights += nights
    return acc
  }, { revenue: 0, reservations: 0, nights: 0 })
}
```

### Agrupamento por mês para listagem

```typescript
const futureByMonth = futureReservations.reduce((acc, r) => {
  const date = new Date(r.check_in)
  const monthKey = `${date.getFullYear()}-${String(date.getMonth() + 1).padStart(2, '0')}`
  const monthName = date.toLocaleDateString('pt-BR', { month: 'long', year: 'numeric' })

  if (!acc[monthKey]) acc[monthKey] = { month: monthName, reservations: [] }
  acc[monthKey].reservations.push(r)
  return acc
}, {})
```

# Story 36.12: Booking System Integration

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)

**Status:** Draft  
**Story Points:** 2  
**Priority:** HIGH  

---

## User Story

As a **Host**, I want **to synchronize my property prices across multiple booking platforms (Airbnb, Booking.com, VRBO)** so that **I can maintain consistent pricing and prevent overbooking**.

---

## Description

Build a booking system integration layer that syncs prices and availability to external platforms in real-time. Includes multi-platform support, channel conflict detection (prevent double bookings), and platform status dashboard. Tracks sync history and logs errors for troubleshooting.

**Why:** Hosts list properties on multiple platforms but need unified price management. Manual price updates across platforms are error-prone and time-consuming.

---

## Acceptance Criteria

### Platform Integration Setup

- [ ] Support Airbnb, Booking.com, VRBO integrations
- [ ] OAuth authentication flow for each platform
  - Airbnb: Use official Airbnb API
  - Booking.com: Use Partner API
  - VRBO: Use VRBO API
- [ ] Store platform credentials securely (encrypted in DB)
- [ ] Connection status indicator: Connected, Disconnected, Error
- [ ] Disconnect/reconnect functionality
- [ ] Display connected properties from each platform

### Real-Time Price Synchronization

- [ ] When price changes in Lodgra:
  - Extract changed dates
  - Push price update to each connected platform within 5 minutes
  - Return success/failure status
- [ ] Support different price types:
  - Base nightly rate
  - Weekend rates
  - Special dates/holidays
- [ ] Sync availability status: Available, Blocked, Unavailable
- [ ] Batch sync for bulk updates (multiple dates)
- [ ] Retry mechanism (exponential backoff) for failed syncs
- [ ] Log all sync attempts (success/failure)

### Multi-Property Price Management

- [ ] Dashboard showing all properties across platforms
- [ ] Matrix view: Properties × Platforms showing sync status
- [ ] Bulk sync button to force resync all properties
- [ ] Individual property sync history
- [ ] Resync specific date range or property

### Channel Conflict Detection

- [ ] Monitor for double bookings:
  - Same guest booking via multiple platforms
  - Overlapping booking dates on different platforms
- [ ] Alert when conflict detected: "Property booked on Airbnb (5-9) AND Booking.com (7-12)"
- [ ] Allow host to manually resolve conflicts
- [ ] Log conflict detection in history
- [ ] Automatic blocking of overlapping dates after host confirms conflict

### Platform Status Dashboard

- [ ] Show status per platform: Connected/Disconnected/Error
- [ ] Display last sync timestamp per platform
- [ ] Show pending syncs (number of changes waiting to sync)
- [ ] Error summary: Count of failed syncs, common error types
- [ ] Platform-specific stats:
  - Number of listings synced
  - Total properties under management
  - Last successful sync time
- [ ] Connection health indicator (green/yellow/red)

### Sync History & Error Logs

- [ ] Detailed sync history showing:
  - Timestamp
  - Platform
  - Property
  - Change type (price update, availability, conflict)
  - Status (success/failure)
  - Error message (if failed)
- [ ] Filter by: Date range, platform, property, status
- [ ] Export sync history as CSV
- [ ] Error log with troubleshooting info:
  - Error code from platform API
  - Recommended action
  - Manual retry button
- [ ] Retain history for 90 days

### Responsive Design

- [ ] Mobile (≤768px): Card-based platform status, scrollable history table
- [ ] Web (>768px): Matrix view for platforms, detailed sync history table
- [ ] Status indicators clearly visible (color-coded)
- [ ] All currency formatted as EUR
- [ ] Error messages actionable and user-friendly

## Risks

| Risk | Mitigation |
|------|-----------|
| Platform API rate limits | Implement request throttling, batch updates, queue system |
| OAuth token expiration | Auto-refresh tokens, graceful degradation if expired |
| Conflict detection misses overlaps | Comprehensive booking data validation, manual conflict resolution |
| Network timeouts during sync | Retry logic with exponential backoff, background job reliability |
| Inconsistent price data | Validation before sync, reconciliation job, audit trail |

---

## Technical Notes

- Use OAuth 2.0 for all platform authentications
- Store credentials encrypted using Node.js crypto
- Implement Bull queue for reliable job scheduling (price sync)
- Sync triggered by: Manual button, scheduled job (hourly), price change event
- Batch API calls to reduce rate limit impact
- Conflict detection: Compare booking calendar across platforms
- Error handling: Log detailed errors, categorize by type, suggest resolution
- Retry strategy: Exponential backoff (5s → 30s → 2min → 5min)
- Cache platform status for 5 minutes to reduce API calls
- CSV export: Use `papaparse` library
- Timezone: All dates/times in UTC, convert for display

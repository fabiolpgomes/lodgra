# Story 13.3 — Performance Audit & Optimization

## Objetivo
Realizar audit de performance completo e optimizar o bundle e loading para atingir Core Web Vitals verdes, melhorando a conversão nas páginas públicas de booking.

## Acceptance Criteria

- [x] Bundle analysis com `@next/bundle-analyzer` — relatório gerado
- [x] Identificar e eliminar dependências pesadas desnecessárias
- [x] Lazy loading de componentes pesados: FullCalendar, PDF generators, chart libraries
- [x] `next/dynamic` com `ssr: false` para componentes client-only pesados
- [x] Images nas páginas públicas optimizadas com `next/image` (sizes, priority, placeholder)
- [ ] Lighthouse score nas páginas públicas: Performance >= 90, Accessibility >= 90
- [ ] Core Web Vitals targets: LCP < 2.5s, CLS < 0.1, INP < 200ms
- [x] Script `npm run analyze` adicionado ao package.json
- [x] Build PASS, Lint PASS

## Notas Técnicas

- `ssr: false` não é permitido em Server Components (Next.js 15+) — solução: thin client wrappers em `src/components/lazy/`
- FullCalendar, chart.js, recharts, @react-pdf/renderer são os maiores bundles
- PDF download functions (OwnerReportPDF, FiscalReportPDF) convertidas para lazy dynamic import no call site
- Nenhum `<img>` raw encontrado — todas as imagens já usam `next/image` ou Supabase storage URLs

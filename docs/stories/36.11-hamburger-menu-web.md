# Story 36.11: Hamburger Menu (Web)

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.11  
**Status:** Draft  
**Story Points:** 3  
**Priority:** MEDIUM  

---

## User Story

As a **Host on Web**, I want **to access the main navigation via a hamburger menu** so that **I can navigate to other sections (Home, Announcements, Messages) without leaving calendar context**.

---

## Description

Build hamburger menu (≡ button) that opens overlay menu with main navigation options. Web-only feature.

**Why:** Web header needs menu access; hamburger is responsive pattern that works across screen sizes.

---

## Acceptance Criteria

### Hamburger Button

- [ ] Icon: ≡ (three horizontal lines)
- [ ] Position: Top-left corner, fixed position
- [ ] Visible only on web (>768px)
- [ ] Touch target: ≥44px height/width

### Menu Overlay

- [ ] On tap ≡: Overlay appears (semi-transparent dark background)
- [ ] Menu slides in from left or fades in
- [ ] Menu items:
  - Home
  - Calendários (current page, highlighted)
  - Anúncios
  - Mensagens
  - Configurações
  - Sair (logout)

### Close Menu

- [ ] Tap close button (X) in menu → closes
- [ ] Tap outside menu (overlay) → closes
- [ ] Tap menu item → navigates and closes

### Navigation

- [ ] Tap item → Navigate to page
- [ ] Active item highlighted (current page)
- [ ] Menu closes automatically after navigation

### Styling

- [ ] Overlay: Semi-transparent, dark background
- [ ] Menu: Slides from left or fades in
- [ ] Smooth animation (200-300ms)
- [ ] Body scroll locked when menu open

---

## Definition of Done

- [ ] Hamburger button visible on web
- [ ] Menu overlay appears/disappears
- [ ] Navigation works
- [ ] Close actions work (button, overlay, menu item)
- [ ] No scrolling behind overlay
- [ ] Smooth animation
- [ ] No TypeScript errors

## Risks

| Risk | Mitigation |
|------|-----------|
| Body scroll not locked | Use `document.body.style.overflow = 'hidden'` |
| Menu z-index conflicts | Use high z-index (1000+) |

---

## Technical Notes

- Use React state for menu open/close
- Overlay: `position: fixed, z-index: 1000`
- Lock body scroll when menu open
- Use CSS transitions for smooth animation

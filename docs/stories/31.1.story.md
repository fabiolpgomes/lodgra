# Story 31.1: Configure Wildcard DNS + Vercel for Multi-Tenant Subdomains

**Epic:** Epic 31 — Multi-Tenant Infrastructure (Branded Direct Booking)  
**Status:** Done  
**Priority:** High  
**Type:** Infrastructure / DevOps  
**Assigned to:** @devops  
**Complexity:** Medium (T-shirt: M)

---

## Story

**As** a Lodgra DevOps engineer,  
**I want** to configure wildcard DNS and Vercel to route all `*.lodgra.io` subdomains to the Lodgra app,  
**So that** companies can access their branded booking pages via `nomeempresa.lodgra.io/booking` instead of `lodgra.io?orgSlug=X`.

---

## Context

The Lodgra middleware **already detects subdomains** and sets the `x-org-slug` header. However, DNS and Vercel are not yet configured to accept wildcard subdomains. Once configured, the system will automatically route:

- `pousada.lodgra.io` → Same app, `x-org-slug: pousada`
- `hotel.lodgra.io` → Same app, `x-org-slug: hotel`
- `resort.lodgra.io` → Same app, `x-org-slug: resort`

This is a **prerequisite** for all Phase 2 customization stories (branding, email templates, etc.).

## Dev Notes

### Step-by-Step Implementation

#### 1. DNS Configuration (5-10 minutes)

**Goal:** Add wildcard record to domain registrar

**Registrars:** GoDaddy, Namecheap, Route53, Cloudflare, etc.

**Steps:**
1. Log into domain registrar (where `lodgra.io` is registered)
2. Go to DNS settings
3. **Add new DNS record:**
   - **Type:** CNAME or A record (check Vercel docs for current preference)
   - **Name:** `*` (or `*.lodgra.io` depending on registrar UI)
   - **Value:** Vercel edge URL (e.g., `cname.vercel-dns.com` or Vercel's IP)
   - **TTL:** 3600 (1 hour) for faster propagation testing, then increase to 86400

**Verification:**
```bash
# Wait 5-10 min for propagation
nslookup test.lodgra.io
# or
dig test.lodgra.io
# Should resolve to Vercel
```

**Note:** If current DNS uses Cloudflare, wildcard may already be partially configured. Check existing records first.

#### 2. Vercel Configuration (5 minutes)

**Goal:** Tell Vercel to accept `*.lodgra.io`

**Steps:**
1. Go to **Vercel Project Settings** → **Domains**
2. Click **Add Domain**
3. Enter: `*.lodgra.io`
4. Vercel will:
   - Auto-detect DNS records
   - Generate SSL certificate (wildcard)
   - Show status (may take 5-10 min)
5. Verify status shows **Valid Configuration**

**Expected SSL Certificate:**
```
Subject: CN=*.lodgra.io
Issuer: Let's Encrypt (or similar)
Validity: 90 days, auto-renews
```

#### 3. Test Routing (10 minutes)

**Local Testing:**

```bash
# Add test entry to /etc/hosts (macOS/Linux) or C:\Windows\System32\drivers\etc\hosts (Windows)
127.0.0.1 test.lodgra.io
127.0.0.1 example.lodgra.io

# Start dev server
npm run dev

# Test
curl -H "Host: test.lodgra.io" http://localhost:3000/booking
# Expected: HTML with x-org-slug header set
```

**Browser Testing:**

1. Once DNS propagates, open browser
2. Navigate to `test.lodgra.io/booking`
3. Check browser DevTools → Network → Response Headers
4. Verify `x-org-slug: test` is present
5. Page should load (assuming org exists in DB)

#### 4. Production Validation (5 minutes)

```bash
# After DNS propagates (5-30 min)
curl -v https://test.lodgra.io/booking 2>&1 | grep -E "SSL|x-org-slug|HTTP"

# Expected output:
# * SSL connection
# * subject: CN=*.lodgra.io
# HTTP/2 200
# x-org-slug: test
```

#### 5. Monitoring Setup (10 minutes)

**Vercel Logs:**
1. Vercel Dashboard → Logs
2. Filter by domain (should show requests)
3. Set up alert if DNS becomes invalid

**Browser Test:**
```javascript
// QA can use this to test any subdomain
async function testSubdomain(slug) {
  const response = await fetch(`https://${slug}.lodgra.io/booking`);
  const headers = response.headers;
  console.log(`Subdomain: ${slug}, Status: ${response.status}`);
  return response.ok;
}

testSubdomain('test');    // Should succeed
testSubdomain('invalid'); // Should show "Org not found" error
```

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| DNS propagation takes >1 hour | Delay testing | Plan 2-4 hours for validation, test in parallel regions |
| Wildcard conflicts with existing records | Services broken (email, api, etc.) | Check existing DNS first, document all records |
| SSL certificate takes time | HTTPS fails temporarily | Vercel usually provisions in 5 min, can be faster |
| Subdomain spoofing via header injection | Security vulnerability | Middleware only reads from `request.headers.get('host')`, cannot be spoofed via URL |
| Too many requests to invalid subdomains | Rate limit spam | Add validation: if org not found, return 404 quickly (don't process request) |

---

## Rollback Plan

If wildcard causes issues:

1. **DNS Rollback:** Remove wildcard record, restore to root domain only (5 min)
2. **Vercel Rollback:** Remove `*.lodgra.io` from domains, keep `lodgra.io` (5 min)
3. **Result:** System reverts to `lodgra.io/booking?orgSlug=X` (backward compatible)

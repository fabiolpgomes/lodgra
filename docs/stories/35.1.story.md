## 🔧 Dev Notes

### Current State (Diagnosis)
- `/booking` is pure SPA (Next.js Client-Side)
- No `<title>`, `<meta description>`, or robots.txt
- No JSON-LD schema
- No /llms.txt
- Domain `algarve-home-stay.lodgra.io` not indexed by Google, Perplexity, or ChatGPT

### Technology Stack
- Framework: Next.js (supports getServerSideProps, getStaticProps, getStaticPaths)
- Database: Supabase (real-time available)
- Deployment: Vercel (ISR-ready)
- Current data source: 9 properties (monthly pricing changes)

### Implementation Notes
- SSR already implemented in `/booking` page.tsx — verify it's working correctly
- ISR Strategy: **24h cache + on-demand revalidation** (optimal for price changes)
  - `revalidate = 86400` (24h fallback) prevents 1440 builds/day
  - `revalidateTag()` allows immediate updates when prices change (< 1s)
  - Balances performance + data freshness
- On-demand revalidation requires API integration:
  - Admin updates price → calls `/api/properties/:id/revalidate-price`
  - Endpoint revalidates cache immediately
  - Next request serves fresh HTML
- JSON-LD should include dynamic price data from Supabase (updated via revalidation)
- robots.txt and sitemap.xml can be static or dynamic (next-sitemap library recommended)

### Files to Create
- `/public/robots.txt`
- `/public/sitemap.xml` (or generated via build)
- `/public/llms.txt`
- `/src/lib/seo/schema.ts` (JSON-LD generators)
- `/src/app/api/properties/[id]/revalidate-price/route.ts` (NEW - on-demand revalidation endpoint)

### Files to Modify
- `src/app/p/[slug]/page.tsx` (update ISR: `revalidate = 86400`, add `revalidateTag()` support)
- `src/lib/seo/metaTags.ts` (dynamic meta tag generation - already exists)

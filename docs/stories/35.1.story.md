## 🔧 Dev Notes

### Current State (Diagnosis)
- `/booking` is pure SPA (Next.js Client-Side)
- No `<title>`, `<meta description>`, or robots.txt
- No JSON-LD schema
- No /llms.txt
- Domain `algarve-home-stay.lodgra.io` not indexed by Google, Perplexity, or ChatGPT

### Technology Stack
- Framework: Next.js (supports getServerSideProps, getStaticProps, getStaticPaths)
- Database: Supabase (real-time available)
- Deployment: Vercel (ISR-ready)
- Current data source: 9 properties (monthly pricing changes)

### Implementation Notes
- SSR/Hybrid is **critical**: without it, rastreadores see empty pages
- Hybrid recommended: Static pages + ISR for monthly availability updates
- JSON-LD should use dynamic data from Supabase (prices, availability)
- robots.txt and sitemap.xml can be static or dynamic (next-sitemap library recommended)

### Files to Create
- `/public/robots.txt`
- `/public/sitemap.xml` (or generated via build)
- `/public/llms.txt`
- `/src/lib/seo/schema.ts` (JSON-LD generators)
- `/src/lib/seo/metaTags.ts` (dynamic meta tag generation)

### Files to Modify
- `src/pages/booking.tsx` or `src/app/booking/page.tsx` (convert to SSR/SSG)
- `src/pages/p/[slug].tsx` or `src/app/p/[slug]/page.tsx` (convert to SSG)
- `next.config.js` (if ISR/caching config needed)
- `src/pages/_document.tsx` or `src/app/layout.tsx` (add meta tag injection)

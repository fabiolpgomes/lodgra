## Story Summary

Implement comprehensive price history tracking and analytics dashboard. Enable property managers to view historical price changes with audit trail, timeline visualization, price statistics (min/max/average), and export functionality. Provides visibility into pricing decisions and their business impact.

## Dev Notes

### Architecture

**Database Schema:**
- New table: `price_history` (property_id, price, date_applied, changed_by, change_reason, created_at)
- New table: `price_analytics` (property_id, period, min_price, max_price, avg_price, change_count, created_at)
- Audit triggers on `daily_prices` to log changes to `price_history`
- Denormalized analytics table refreshed daily via scheduled function

**API Endpoints:**
- `GET /api/properties/{id}/price-history` — fetch historical records with pagination
- `GET /api/properties/{id}/price-history/stats` — get price statistics (min/max/avg)
- `GET /api/properties/{id}/price-history/analytics` — analytics dashboard data
- `GET /api/properties/{id}/price-history/export` — export CSV
- `POST /api/properties/{id}/price-history/revert` — revert to previous price
- `DELETE /api/properties/{id}/price-history/{recordId}` — soft delete record (audit only)

**React Hooks & Utilities:**
- Create `usePriceHistory()` hook for fetching and managing history state
- Create `useAnalyticsCalculator()` hook for stats computation
- Utility: `calculatePriceStats()` — min, max, average, standard deviation
- Utility: `calculateRevenueImpact()` — estimate revenue change from price adjustments
- Utility: `exportToCsv()` — format history data and trigger download

**UI Components:**
- `PriceHistoryTimeline` — chronological view with card-based layout
- `PriceStatistics` — display min/max/avg in metric cards
- `PriceAnalyticsDashboard` — combined view with timeline + stats + export
- `PriceChangeNotification` — toast/badge when price changes detected
- `HistoryFilters` — date range picker and search controls
- `RevertModal` — confirm and revert to previous price

**Mobile Considerations:**
- Timeline view: vertically scrollable with touch-friendly cards
- Statistics: stacked metric display on small screens
- Export button: prominent, easy-to-tap action
- Filters: slide-out panel on mobile (drawer pattern)
- Responsive date picker: native input on mobile, calendar widget on desktop
- Performance: lazy-load history items (infinite scroll)

### Key Implementation Details

1. **Audit Trail:** Every price change is immutable; includes user_id, timestamp, reason/source
2. **Statistics Calculation:** Computed from `daily_prices` and cached in `price_analytics` table
3. **Revenue Impact:** Estimated by comparing current price to previous price × bookings in period
4. **Revert Logic:** Creates new price history record with `is_revert: true` flag (not destructive)
5. **Export Format:** CSV with columns: date_applied, price, changed_by, reason, revenue_impact
6. **Notifications:** Triggered when price changes > 10% or when manual overrides applied
7. **Performance:** Indexes on (property_id, created_at) for fast history queries
8. **Pagination:** Default 50 records per page, load-more for infinite scroll

## Testing Strategy

### Unit Tests
- **Statistics Calculation:** min, max, average, empty array, single value, large datasets
- **Revenue Impact:** price increase/decrease, no bookings, multiple bookings
- **CSV Export:** header row, data rows, special characters, currency formatting
- **Date Filtering:** exact date, date ranges, timezone handling

### Component Tests
- **PriceHistoryTimeline:** renders records, pagination, click to view details
- **PriceStatistics:** displays correct values, formatting, empty state
- **HistoryFilters:** date picker interaction, search input, apply filters
- **RevertModal:** show/hide, confirm/cancel, calls revert API
- **Mobile Responsiveness:** vertical scroll, touch interactions, layout stacking

### API Tests
- **GET /price-history:** returns paginated results, filters work, auth check
- **GET /price-history/stats:** returns accurate statistics, handles empty data
- **GET /price-history/analytics:** returns analytics data with time series
- **GET /price-history/export:** returns valid CSV, correct formatting
- **POST /price-history/revert:** creates new record, doesn't delete old, updates current
- **Error cases:** invalid property ID, unauthorized access, malformed requests

### Integration Tests
- Create price history via daily_prices updates, verify in history table
- Calculate statistics, verify accuracy against raw data
- Export history, verify CSV is valid and complete
- Revert price, verify audit trail shows both prices
- Filter by date range, verify only matching records returned
- Concurrent updates don't corrupt analytics cache

### E2E Tests
- Manager views price history for property
- Manager filters by date range and searches records
- Manager exports history to CSV
- Manager reverts to previous price
- Notifications triggered on significant price changes
- Mobile workflow: scroll history, export, revert all on phone

### Edge Cases
- Empty price history (first price set)
- Single record in history
- Very large history (10,000+ records) — performance test
- Same price set multiple times (no change)
- Very rapid price changes (within minutes)
- Timezone boundaries (UTC vs local)
- Leap year dates
- Daylight saving time transitions
- Concurrent revert attempts
- Export with missing data fields

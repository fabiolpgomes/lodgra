# Story 36.14: Mobile-First Calendar Layout (Airbnb Design Reference)

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.14  
**Status:** Ready for Review  
**Story Points:** 8  
**Priority:** HIGH  
**Scope:** Redesign calendar UI with mobile-first Airbnb-inspired layout

---

## User Story

As a **Mobile-First Host**, I want **to see my properties and calendars in a clean, card-based layout similar to Airbnb** so that **I can manage my pricing and reservations intuitively on my phone without overwhelm**.

---

## Description

Redesign the calendar interface following Airbnb's mobile-first design language:
- Property list as horizontal-scrollable cards with thumbnail + calendar preview
- Detailed calendar with large day cards showing prices
- Tab-based settings (Preços | Descontos | Disponibilidade)
- Clean whitespace, large typography, touch-friendly interactions

**Why:** Current design is web-first/desktop-oriented. Mobile users need visual hierarchy, large targets, and less information density.

---

## Scope

**IN (Included in this story):**
- Property list as vertical cards with calendar preview (dot pattern)
- Detailed calendar with large day cards showing prices
- Tab-based settings interface (Preços | Descontos | Disponibilidade)
- Mobile-first layout optimization (375px baseline)
- Swipe gestures for month navigation
- Responsive behavior up to web (>768px)
- Touch-friendly interactions (44×44px minimum tap targets)
- Visual design following Airbnb mobile language

**OUT (Explicitly excluded - future stories):**
- Desktop sidebar layout (→ Story 36.15)
- Drag-and-drop scheduling (→ Future enhancement)
- Multi-select days for bulk operations (→ Future enhancement)
- Email/push notifications (→ Separate epic)
- Deep linking and URL routing (→ Future feature)
- Full WCAG accessibility pass (→ Dedicated accessibility story)
- Advanced analytics on calendar (→ Separate feature)

## Acceptance Criteria

### Property List View (Mobile-First)
- [x] Display properties as vertical cards (375px width)
- [x] Card content: Property image (top), name, type, location
- [x] Calendar preview: Dot pattern showing availability (dots for occupied/available)
- [x] List scrollable, full-width on mobile
- [x] Tap card → Navigate to detailed calendar

### Detailed Calendar View
- [x] Header: [← Back] | Property name | [📅 Month picker] [⚙️ Settings]
- [x] Month/year selector for quick navigation
- [x] Calendar grid: 7×4 or 7×5 (day numbers large, readable)
- [x] Each day cell: Day number (large) + Price (EUR, bold)
- [x] Price color coding: Standard price (dark) | Weekend price (highlight)
- [x] Today indicator: Red circle on day number
- [x] Booked days: Gray background with guest name overlay
- [x] Tap day → Quick edit modal (from Story 36.3)
- [x] Swipe left/right for month navigation (mobile gesture)

### Settings Tabs (Mobile-First)
- [x] Tab navigation: Preços | Descontos | Disponibilidade
- [x] Active tab: Dark background, other tabs light gray
- [x] Tab content scrollable vertically

#### Preços Tab
- [x] Price boxes (cards):
  - "Preço básico" — R$ 149
  - "Preço de fim de semana" — R$ 157 (with remove option)
  - "Preço Inteligente" — Toggle on/off
- [x] Each price card: Title + value + action button/toggle
- [x] Large, tappable spacing

#### Descontos Tab
- [x] Discount cards:
  - "Por semana" (e.g., 0% for 7+ nights)
  - "Por mês" (e.g., 5% for 28+ nights)
  - "Hóspedes com avaliações excelentes" (15% for 4.8+ rating)
  - "Reservas de última hora"
  - "Reservas antecipadas"
- [x] Each card shows trigger condition + discount %
- [x] Tap to edit percentage

#### Disponibilidade Tab
- [x] Settings cards:
  - "Número mínimo de noites" (e.g., 90)
  - "Número máximo de noites" (e.g., 365)
  - "Tempo de antecedência" (e.g., "Mesmo dia")
  - "Aviso prévio" (e.g., 00:00)
  - "Tempo de preparação" (e.g., "Nenhum")
- [x] Large text, clear values

### Responsive Behavior
- [x] Mobile (≤768px): Full-screen card-based layout
- [x] Web (>768px): Center calendar, sidebars for settings
- [x] All text readable without pinch-zoom
- [x] All buttons/cards tap-friendly (min 44px height)

### Visual Design
- [x] Typography: Sans-serif, large headers (24-32px)
- [x] Spacing: Generous padding (16-24px between sections)
- [x] Colors: Minimal palette (white bg, gray accents, red highlights)
- [x] Cards: Subtle shadows, rounded corners (8-12px)
- [x] Icons: Simple, 24-32px size, high contrast

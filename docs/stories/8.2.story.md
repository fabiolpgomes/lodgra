## Story

**Como** visitante ou cliente existente,
**Quero** ver claramente os 3 planos disponíveis e poder subscrever o plano certo,
**Para que** eu entenda o valor de cada tier e não seja bloqueado por limite de plano sem saber como fazer upgrade.

---

## Contexto de Negócio

A Story 8.1 implementa o enforcement backend (limites + feature gates). Esta story garante que:
1. A landing page apresenta os 3 planos com preços e features
2. O checkout Stripe suporta 3 price IDs distintos
3. Quando um utilizador autenticado atinge um limite ou tenta aceder uma feature bloqueada, vê um prompt claro de upgrade (não um erro genérico)

### Prerequisito
Story 8.1 deve estar **Done** antes de implementar esta story.
Env vars Stripe devem estar configuradas: `STRIPE_PRICE_ID_STARTER_EUR`, `STRIPE_PRICE_ID_PROFESSIONAL_EUR`, `STRIPE_PRICE_ID_BUSINESS_EUR`.

## Dev Notes

### Checkout API — suporte a `plan`

Actualizar `POST /api/stripe/checkout`:
```ts
const { email, plan = 'starter' } = body
const PRICE_IDS: Record<string, string> = {
  starter:      process.env.STRIPE_PRICE_ID_STARTER_EUR!,
  professional: process.env.STRIPE_PRICE_ID_PROFESSIONAL_EUR!,
  business:     process.env.STRIPE_PRICE_ID_BUSINESS_EUR!,
}
const priceId = PRICE_IDS[plan]
if (!priceId) return NextResponse.json({ error: 'Plano inválido' }, { status: 400 })
```

### Upgrade Prompt — padrão de implementação

Para AC4 (criação de propriedade bloqueada):
```ts
// No componente de criação de propriedade, após fetch POST:
if (res.status === 403) {
  const data = await res.json()
  if (data.error === 'property_limit_reached') {
    toast.error(`Limite de ${data.limit} propriedades atingido. Faça upgrade do plano.`, {
      action: { label: 'Ver planos', onClick: () => router.push('/#pricing') }
    })
    return
  }
}
```

Para AC5 (feature gate em report/fiscal pages):
```ts
// Nas pages /owners/[id]/report e /owners/[id]/fiscal:
if (error === 'plan_upgrade_required') {

      <h3>Funcionalidade exclusiva do plano Professional</h3>
      <p>Faça upgrade para aceder a relatórios e compliance fiscal.</p>
      <Link href="/#pricing">Ver planos</Link>
    </div>
  )
}
```

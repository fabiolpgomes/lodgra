## Story

**Como** gestor de propriedades,
**Quero** ver e controlar quais propriedades têm página pública activa, e perceber quantas páginas o meu plano permite,
**Para que** possa partilhar URLs directamente com os meus hóspedes e saber quando preciso de fazer upgrade.

---

## Contexto de Negócio

Após o redesign (Story 14.2), cada propriedade activa com `is_public = true` tem uma página pública premium em `/p/[slug]`. O gestor precisa de:
1. Ver o URL público de cada propriedade e copiá-lo com um clique
2. Activar/desactivar a página pública de cada propriedade (toggle `is_public`)
3. Entender os limites do seu plano (X de Y páginas públicas usadas)
4. Ser incentivado a fazer upgrade quando atingir o limite

### Relação com Planos

O limite de páginas públicas = limite de propriedades activas (já implementado em Story 8.1):
- Starter: 3
- Professional: 10
- Business: ilimitado

Não há enforcement adicional — o toggle `is_public` é apenas de visibilidade, não cria nova propriedade.

### API — PATCH /api/properties/[id]

Verificar se existe `src/app/api/properties/[id]/route.ts`. Se não existir, criar.

```ts
// PATCH /api/properties/[id]
// Body: { is_public?: boolean }
const auth = await requireRole(['admin', 'gestor'])
// Verificar que property pertence à org do utilizador
const { data, error } = await supabase
  .from('properties')
  .update({ is_public: body.is_public })
  .eq('id', params.id)
  .eq('organization_id', auth.organizationId)
  .select('slug, is_public')
  .single()

const baseUrl = process.env.NEXT_PUBLIC_APP_URL ?? 'https://www.homestay.pt'
return NextResponse.json({
  is_public: data.is_public,
  public_url: data.is_public ? `${baseUrl}/p/${data.slug}` : null,
})
```

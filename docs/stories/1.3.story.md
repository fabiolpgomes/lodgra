## Dev Notes

### Campo `source` nas reservas

O campo `source` em `reservations` contém o identificador da plataforma conforme vem do sync iCal. Valores típicos:
- `"airbnb"`, `"booking"`, `"vrbo"`, `"direct"`, ou NULL

### Agregação por canal em `reports/page.tsx`

```typescript
// Agregar reservas por canal (source)
const revenueByChannel = reservations?.reduce((acc, r) => {
  const channel = r.source || 'direct'
  const normalizedChannel = normalizeChannelName(channel)

  if (!acc[normalizedChannel]) {
    acc[normalizedChannel] = {
      name: normalizedChannel,
      revenue: 0,
      reservations: 0,
      nights: 0,
      currency: r.currency || 'EUR'
    }
  }

  acc[normalizedChannel].revenue += r.total_amount ? Number(r.total_amount) : 0
  acc[normalizedChannel].reservations += 1

  const nights = Math.ceil(
    (new Date(r.check_out).getTime() - new Date(r.check_in).getTime()) / (1000 * 60 * 60 * 24)
  )
  acc[normalizedChannel].nights += nights

  return acc
}, {} as Record<string, ChannelStat>)
```

### Função de normalização de nomes

```typescript
// src/lib/utils/channels.ts (novo ficheiro)
export function normalizeChannelName(source: string): string {
  const map: Record<string, string> = {
    airbnb: 'Airbnb',
    booking: 'Booking.com',
    'booking.com': 'Booking.com',
    vrbo: 'VRBO',
    homeaway: 'HomeAway',
    expedia: 'Expedia',
    direct: 'Directo',
    manual: 'Manual',
  }
  return map[source.toLowerCase()] || source
}
```

Layout sugerido:

- Para cada canal: nome, receita, reservas, ADR, barra de % dependência
- Barra de dependência colorida: >50% vermelha (risco de concentração), 25–50% amarela, <25% verde

### Nota sobre multi-currency

Se houver reservas em EUR e BRL de diferentes canais, a percentagem de dependência deve ser calculada por moeda separadamente, ou o componente deve filtrar por moeda primária.

## Story

**Como** produto Lodgra em fase de estabilização,  
**Quero** focar a landing page pública exclusivamente no mercado Brasil (PT-BR),  
**Para que** usuários vejam uma experiência clara e focada, sem confusão entre múltiplos mercados, enquanto mantenho a funcionalidade multi-moeda e multi-país do sistema para operações existentes (Portugal em EUR).

---

## Contexto de Negócio

O Lodgra opera em 3 mercados (Brasil, Portugal, EUA) mas está em fase de consolidação no Brasil. A landing page pública atual exibe seletores de idioma para PT-BR, EN-US e ES, criando confusão para usuários brasileiros e impossibilitando validação clara do produto em um único mercado.

**Situação atual:**
- Landing page apresenta 3 mercados: PT (Portugal), BR (Brasil), US (EUA)
- Usuários podem mudar entre locales via dropdown na Navbar
- Conteúdo em 3 idiomas renderizado dinamicamente

**Objetivo:**
- Remover UI de seletores de idioma (EN/ES) da landing page pública
- Manter landing page em PT-BR exclusivamente
- **Preservar integralmente:**
  - Backend multi-moeda (EUR, BRL, USD, etc.)
  - Operações em Portugal (lodgra/p) — EUR continua funcional
  - Checkout flow com Stripe
  - Administração de propriedades por país

**Benefícios:**
- ✅ UX clara e focada no Brasil
- ✅ Possibilita testes precisos com usuários BR
- ✅ Prepara para expansão sequencial (PT → ES/EN) após validação
- ✅ Zero débito técnico — mudança apenas de UI, nenhuma alteração lógica

**Critical Blocker:**
- ❌ **INFRASTRUCTURE ROUTING COLLAPSE** (Not Story 2.5 specific - affects ALL app pages)
  - **Root cause:** Next.js 16.2.6 failing to compile ANY page routes at app level
  - **Symptoms:** 
    - Dev server: `curl http://localhost:3000/landing` → 404 "This page could not be found"
    - Prod build: Only 3 routes in manifest: /_not-found, /api/features/check, /api/properties/check-limit
    - Build log: "Generating static pages using 5 workers (0/4)...✓ (4/4)" — only 4 pages total generated (errors + API)
    - No page routes in .next/server/app/ except _not-found and _global-error
  
  - **Testing completed (all failed):**
    - ❌ Removed proxy.ts entirely → still 404
    - ❌ Removed generateStaticParams from root pages → still 404
    - ❌ Converted middleware.ts → proxy.ts (per Next.js 16 spec) → still 404
    - ❌ Switched Turbopack → Webpack → still 404
    - ❌ Clean .next cache → still 404
    - ✓ Build compiles successfully (no errors reported)
    - ✓ TypeScript validation passes
    - ✓ Lint passes
  
  - **Evidence pointing to infrastructure issue:**
    - Problem exists with AND without proxy (not proxy-caused)
    - Problem exists in dev AND prod (not env-specific)  
    - Problem affects ALL pages, not just /landing (app-wide collapse)
    - 72+ page files exist in src/app/, but only 0 compiled to .next/server/app/
    - Manifest shows "ƒ Proxy (Middleware)" but no actual pages
    - This blocks ALL story development until fixed (not Story 2.5 issue, but infrastructure issue)
  
  - **Suspected root cause** (ranked by probability):
    1. **Sentry wrapper interference** - withSentryConfig may be breaking route discovery in Next.js 16
    2. **Next.js 16 + complex app structure conflict** - possible edge case with 70+ routes
    3. **Turbopack routing bug** - known issues with dynamic routing in early versions

**Recommendations for Approval:**
1. **ESCALATE TO INFRASTRUCTURE** — This is NOT a story-level issue. App-wide routing collapse blocks ALL development.
2. **Investigate Sentry wrapper** — Primary suspect: `withSentryConfig()` may interfere with Next.js 16 route discovery
   - Test: Temporarily disable Sentry wrapper in next.config.js and rebuild
   - If routes appear → Sentry configuration needs update for Next.js 16
3. **If Sentry is culprit:**
   - Update @sentry/nextjs to latest (check Next.js 16 compatibility matrix)
   - Or adjust withSentryConfig options (autoInstrumentServerFunctions, etc.)
4. **Fallback options if Sentry is blocking:**
   - Downgrade Next.js from 16.2.6 → 15.x until Next.js 16 + Sentry compatibility confirmed
   - Or disable Sentry instrumentation for development/testing

**Status:** ✅ **INFRASTRUCTURE FIXED** — Root cause found: stale `./app/` directory at project root was overriding `./src/app/`.

**Resolution:** Deleted `./app/` (contained outdated API route copies). Now 97 routes compile correctly.

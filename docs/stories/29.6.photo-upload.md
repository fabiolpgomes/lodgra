## Story

**Quero** fotografar a propriedade como prova,

---

## Acceptance Criteria

### AC1: Page `/[locale]/cleaning/tasks/[taskId]/photos` com upload zone
- [ ] **GIVEN** task carregada
- **WHEN** navega para photos tab
- **THEN** vê upload area (drag-drop ou file picker)

### AC2: Drag-drop ou file picker para múltiplas imagens
- [ ] **WHEN** arrasta ficheiros para upload area
- **THEN** imagens são selecionadas
- [ ] **OR** clica "Select Files" e pick múltiplas

### AC3: Preview antes de upload
- [ ] **WHEN** imagens selecionadas
- **THEN** mostra thumbnails (200x200px) antes de upload

### AC4: Auto-compressão (max 2MB, JPEG 80% quality)
- [ ] **GIVEN** imagem 10MB PNG
- **WHEN** upload
- **THEN** comprimido para ~1-2MB JPEG

### AC5: Upload para Supabase Storage
- [ ] **WHEN** clica "Upload"
- **THEN** POST `/api/cleaning/[id]/photos`
- [ ] **AND** ficheiro armazenado em `cleaning-photos/{org_id}/{task_id}/{filename}`

### AC6: Filename: timestamp + random hash
- [ ] **THEN** filename like `1609459200_abc123def456.jpg`
- [ ] **AND** colisão impossível (random suffix)

### AC7: Metadata armazenado (uploaded_at, uploader_id, task_id)
- [ ] **WHEN** foto uploadada
- **THEN** row em cleaning_photos com: id, task_id, file_path, uploaded_at, uploader_id

### AC8: Galeria mostra fotos com delete option
- [ ] **WHEN** fotos uploadadas
- **THEN** grid view com thumbails
- [ ] **AND** delete button com confirmação

### AC9: Progress bar durante upload
- [ ] **WHEN** upload em progresso
- **THEN** progress bar mostra percentage (0-100%)

### AC10: Error handling (network fail → retry logic)
- [ ] **GIVEN** network interruption
- **WHEN** upload falha
- **THEN** mostra erro + "Retry" button
- [ ] **AND** max 3 retries automáticas

### AC11: Não permite upload de ficheiros inválidos
- [ ] **WHEN** tenta upload de .exe ou .zip
- **THEN** erro: "Apenas imagens permitidas (JPG, PNG, WebP)"

### AC12: RLS verificado (cleaner vê apenas suas fotos)
- [ ] **GIVEN** cleaner-A faz upload
- **WHEN** cleaner-B tenta acessar ficheiro
- **THEN** 403 Forbidden (Storage RLS policy)

## Integration Verification

### IV1: Storage bucket RLS isolado
- [ ] **WHEN** foto salva
- **THEN** path `cleaning-photos/{org_id}/{task_id}/` isolado por RLS policy

### IV2: API POST `/api/cleaning/[id]/photos`
- [ ] **WHEN** endpoint chamado
- **THEN** valida auth, salva metadata, retorna URL público

### IV3: Metadata persistido em `cleaning_photos` table
- [ ] **WHEN** upload completo
- **THEN** row inserido com metadata correto

## Story

**Como** gestor,  
**Quero** enviar um link de acesso via WhatsApp ao cleaner,  
**Para que** ele aceda ao portal sem precisar de password ou conta no sistema principal.

---

## Contexto de Negócio

Os cleaners não são utilizadores do sistema de gestão. Precisam de um acesso simples, mobile-first, sem fricção de registo. O fluxo é: gestor cria cleaner no sistema → sistema envia link WhatsApp → cleaner clica → acede ao seu painel.

### O que já existe
- RBAC com `guest_type` (staff, owner)
- Tabela `cleaner_access_tokens` (story 29.1)
- WhatsApp Cloud API (story 30.1)

### O que falta
- Novo `guest_type = 'cleaner'` em `user_profiles`
- Endpoint `/api/cleaners/send-access-link` — gera token e envia WhatsApp
- Endpoint `/api/cleaners/auth` — valida token, cria sessão
- Middleware de protecção da rota `/cleaner/`
- Formulário de cadastro de cleaner no painel do gestor

## Fluxo de Autenticação

```
Gestor → Cria cleaner → Clica "Enviar link"
  → POST /api/cleaners/send-access-link
  → Gera token (32 bytes hex, expires 24h)
  → Envia WhatsApp: "Olá [nome]! Acesse suas tarefas: [link]"
  
Cleaner → Clica link → GET /cleaner/auth?token=XXXXX
  → POST /api/cleaners/auth { token }
  → Valida token (existe, não expirado, não revogado)
  → Cria sessão JWT com cleaner_id + organization_id
  → Redirect → /cleaner/dashboard
```

---

## Riscos & Mitigação

**Token brute-force** — Tokens curtos podem ser adivinhados. **Mitigação:** 32 bytes hex = 256 bits de entropia; rate limit no endpoint de auth (5 tentativas/min por IP).

**Link partilhado** — Cleaner partilha link com terceiros. **Mitigação:** Sessão limitada a 8h; token de uso único configurável por organização.

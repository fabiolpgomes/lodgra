# Story 31.2: Dynamic Company Branding (Logos, Colors, Favicons)

**Epic:** Epic 31 — Multi-Tenant Infrastructure (Branded Direct Booking)  
**Status:** Draft  
**Priority:** High  
**Type:** Backend + Frontend  
**Assigned to:** @dev  
**Complexity:** Medium (T-shirt: M)

---

## Story

**As** a Lodgra product owner,  
**I want** each company to customize their booking page branding (logo, colors, favicon),  
**So that** the booking page reflects each company's brand identity and improves conversion through brand recognition.

---

## Context

With wildcard DNS and Vercel configured (Story 31.1), subdomains now route to the app. However, all companies see the generic Lodgra branding. Phase 2 introduces company-level customization, starting with visual branding (logos, colors, favicons).

**Current State:**
- Middleware extracts org_id from subdomain
- Booking page reads x-org-slug header
- Database supports organization_id isolation
- No branding customization yet

**Desired State:**
- Companies can upload logo and favicon
- Companies can customize primary brand colors
- Booking page dynamically applies company branding
- Email confirmations use company branding

---

## Acceptance Criteria

### AC1: Database Schema for Company Branding
**Given:** An organization exists in Supabase  
**When:** Admin uploads branding data  
**Then:**
- [ ] `organization_branding` table created with fields:
  - `id` (UUID primary key)
  - `organization_id` (FK to organizations)
  - `logo_url` (text, nullable — uploaded to Vercel Blob Storage)
  - `favicon_url` (text, nullable)
  - `primary_color` (text, hex color code, default: Lodgra blue #1E40AF)
  - `secondary_color` (text, hex, default: Lodgra gray #6B7280)
  - `accent_color` (text, hex, default: Lodgra gold #FFC000)
  - `created_at` (timestamp)
  - `updated_at` (timestamp)
- [ ] RLS (Row Level Security) enforced: users can only see branding for their organization
- [ ] Unique constraint on organization_id (one branding config per org)

### AC2: File Upload Endpoint
**Given:** Admin is logged in with org ownership  
**When:** POST request to `/api/organizations/[orgId]/branding/upload`  
**Then:**
- [ ] Accepts file uploads (logo PNG/JPG max 2MB, favicon ICO/PNG max 100KB)
- [ ] Uploads files to Vercel Blob Storage with org-namespaced paths
- [ ] Returns signed URLs for frontend to consume
- [ ] Returns 401 if user doesn't own the organization
- [ ] Returns 400 if file exceeds size limits or wrong format

### AC3: Branding Colors API Endpoint
**Given:** Admin wants to update brand colors  
**When:** PATCH request to `/api/organizations/[orgId]/branding/colors`  
**Then:**
- [ ] Accepts JSON: `{ primary_color, secondary_color, accent_color }`
- [ ] Validates hex color format (e.g., #FFFFFF)
- [ ] Updates database and returns updated branding object
- [ ] Returns 401 if user doesn't own organization
- [ ] Returns 400 if invalid color format

### AC4: Branding Fetch API
**Given:** Booking page is loading for organization  
**When:** GET request to `/api/organizations/[orgSlug]/branding`  
**Then:**
- [ ] Returns branding object for the organization (or defaults if not set)
- [ ] Response includes: logo_url, favicon_url, primary_color, secondary_color, accent_color
- [ ] Works for non-authenticated requests (public read)
- [ ] Returns 404 if organization not found

### AC5: Booking Page Dynamic Branding
**Given:** Company branding is stored in database  
**When:** Booking page loads (e.g., `pousada.lodgra.io/booking`)  
**Then:**
- [ ] Fetches branding from `/api/organizations/[orgSlug]/branding`
- [ ] Applies dynamic styles:
  - Primary color used for CTAs, buttons, headers
  - Secondary color used for card borders, dividers
  - Accent color used for highlights (price badges, badges)
- [ ] Renders company logo in header (if provided)
- [ ] Sets favicon to company favicon (if provided)
- [ ] Falls back to Lodgra defaults if branding not configured
- [ ] No visual regression if branding missing

### AC6: Favicon Dynamic Application
**Given:** Company has uploaded favicon  
**When:** Browser loads `pousada.lodgra.io/booking`  
**Then:**
- [ ] Dynamic favicon endpoint serves company favicon: `/api/favicon?orgSlug=[slug]`
- [ ] Favicon is cached for 24 hours (Cache-Control: public, max-age=86400)
- [ ] Falls back to Lodgra favicon if company favicon not set
- [ ] Works across all pages (not just /booking)

### AC7: Admin Dashboard for Branding Management
**Given:** Admin navigates to organization settings  
**When:** Admin visits `/settings/organizations/[orgId]/branding`  
**Then:**
- [ ] Form displays current branding (logo, favicon, colors)
- [ ] Color picker UI for primary/secondary/accent colors
- [ ] Logo upload input (drag-and-drop supported)
- [ ] Favicon upload input
- [ ] "Save" button updates branding in database
- [ ] "Preview" button shows how booking page will look
- [ ] Success toast on save
- [ ] Error handling if upload fails

### AC8: Security & Data Isolation
- [ ] RLS policies prevent cross-org branding data leakage
- [ ] File uploads validated (no script injection via SVG, etc.)
- [ ] File size limits enforced
- [ ] Authenticated endpoints require org ownership

## Dev Notes

### Technical Approach

**1. Database Migration (Supabase)**
```sql
CREATE TABLE organization_branding (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID NOT NULL UNIQUE REFERENCES organizations(id) ON DELETE CASCADE,
  logo_url TEXT,
  favicon_url TEXT,
  primary_color TEXT DEFAULT '#1E40AF',
  secondary_color TEXT DEFAULT '#6B7280',
  accent_color TEXT DEFAULT '#FFC000',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

ALTER TABLE organization_branding ENABLE ROW LEVEL SECURITY;

CREATE POLICY org_branding_select ON organization_branding
  FOR SELECT USING (
    auth.uid() IN (
      SELECT user_id FROM organization_members 
      WHERE organization_id = organization_branding.organization_id
    )
  );

CREATE POLICY org_branding_update ON organization_branding
  FOR UPDATE USING (
    auth.uid() IN (
      SELECT user_id FROM organization_members 
      WHERE organization_id = organization_branding.organization_id 
      AND role = 'admin'
    )
  );
```

**2. File Upload Strategy**
- Use Vercel Blob Storage for logo/favicon
- Path format: `orgs/{orgId}/logo` and `orgs/{orgId}/favicon`
- Return signed URLs from upload endpoint
- Cache headers: 1 year for immutable assets

**3. Frontend Implementation**
- Create `<BrandingProvider>` context to pass branding to components
- Update `BookingPageClient` to use branding colors (Tailwind CSS variables)
- Dynamic favicon: use HTML `<link rel="icon">` meta tag in RSC

**4. Color Application**
- Use CSS custom properties (--color-primary, --color-secondary, --color-accent)
- Apply in booking page layout
- Update button, badge, and header styles

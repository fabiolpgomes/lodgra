## Story

**Como** gestor,  
**Quero** ver as fotos carregadas pelo cleaner em tempo real sem delay ou polling,  
**Para que** possa monitorar o progresso da limpeza instantaneamente.

**Como** utilizador iOS,  
**Quero** fazer upload de fotos em formato nativo HEIC,  
**Para que** não seja necessário converter manualmente no dispositivo.

**Como** utilizador do portal,  
**Quero** que a galeria de fotos carregue rapidamente com imagens otimizadas,  
**Para que** a experiência de visualização seja fluida e responsiva.

---

## Contexto de Negócio

Story 29.5 implementou upload de fotos com polling (5s) e suporte limitado a formatos. Esta story melhora três aspetos críticos identificados na QA:

1. **Realtime Updates** — Substituir polling com subscriptions via Supabase Realtime para updates instantâneos
2. **iOS Compatibility** — Adicionar suporte HEIC nativo para utilizadores iOS sem conversão manual
3. **Performance** — Otimizar LCP substituindo `<img>` com Next.js `<Image />` component

## Riscos & Mitigação

**Realtime Connection Instability** — Supabase Realtime may have connection issues on poor networks. **Mitigação:** Implement exponential backoff for reconnection; fallback to polling if realtime unavailable.

**HEIC Library Size** — `heic2any` library adds bundle size. **Mitigação:** Use dynamic import with fallback to file rejection for unsupported browsers.

**Image Component Configuration** — Incorrect Next.js Image props can break layout. **Mitigação:** Use fixed width/height or `fill` with container; test across screen sizes.

## Dev Notes

**Realtime Implementation:**
- Use `supabase.channel('realtime:cleaning_photos').on('*', handler)` pattern
- Filter by `task_id` on client-side to avoid processing unrelated photos
- Consider RLS policies to prevent unauthorized subscriptions

**HEIC Conversion:**
- Import heic2any dynamically: `const heic2any = (await import('heic2any')).default`
- Call before compressImage: `const jpeg = await heic2any({ blob: heicFile })`
- Chain with existing Canvas compression for consistency

**Image Component:**
- For thumbnails: Use `fill` with container `position: relative`
- For previews: Use explicit width/height (e.g., 96px × 96px)
- Signed URLs: Verify Next.js Image Optimization doesn't override external URL

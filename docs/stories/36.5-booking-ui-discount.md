# Story 36.5: Booking UI — Show Discount Breakdown

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.5  
**Status:** Draft  
**Story Points:** 5  
**Priority:** HIGH  

---

## User Story

As a **Guest**, I want **to see the discount applied to my reservation** so that **I understand the price I'm paying and trust the booking**.

---

## Description

Integrate discount calculation into booking flow. After guest selects dates, call `POST /api/properties/:id/calculate-price` endpoint (Story 36.4) and display breakdown showing base price, discount, and final total.

**Why:** Transparency builds trust and encourages longer stays through discount visibility.

---

## Acceptance Criteria

### Booking Flow Integration

- [ ] After guest selects check-in and check-out dates:
  - Call `POST /api/properties/:id/calculate-price` with dates
  - Display loading state while fetching

### Price Breakdown Display

- [ ] Show breakdown section:
  ```
  Preço base:        €1.043,00
  Desconto (21%):    -€219,00
  ─────────────────────────────
  Total:             €824,00
  ```
- [ ] If discount applied:
  - Show discount label: "Desconto por semana" or "Desconto por mês"
- [ ] If no discount:
  - Show base total only

### Real-Time Updates

- [ ] Update breakdown when guest changes dates
- [ ] Debounce API calls (max 1 request per 500ms)
- [ ] Show spinner while recalculating

### Error Handling

- [ ] If API error → Show error message, use last valid price
- [ ] Invalid dates → Show validation error

### Store Pricing Data

- [ ] Store pricing breakdown in booking/order object
- [ ] Include in order creation payload:
  - `base_total`
  - `discount_type`
  - `discount_percentage`
  - `final_total`

## Risks

| Risk | Mitigation |
|------|-----------|
| Excessive API calls if debouncing fails | Implement debouncing (500ms) |
| Stale pricing data | Always call API on date change |
| Mobile overflow (long numbers) | Use responsive layout, abbreviate currency |

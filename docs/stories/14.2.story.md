**Epic:** 14 — Property Public Experience
**Status:** 🔵 Ready for Dev
**Prioridade:** P1 — Core do epic, impacto directo na conversão
**Tipo:** Frontend (redesign completo)
**Agente:** @dev

---

## Story

**Como** visitante de uma página pública de propriedade (`/p/[slug]`),
**Quero** uma experiência visual moderna e imersiva (galeria full-screen, booking widget claro, mapa e comodidades detalhadas),
**Para que** me sinta confiante para reservar directamente sem recorrer a uma OTA.

---

## Contexto de Negócio

A página `/p/[slug]` actual é funcional mas minimalista. Plataformas líderes como Holidu e Airbnb demonstram que a experiência visual do imóvel é o principal driver de conversão. O redesign deve:

1. **Galeria imersiva** — hero full-width com lightbox, grid Masonry ou layout Airbnb
2. **Booking widget sticky** — preço, calendário inline, CTA no desktop; bottom sheet no mobile
3. **Conteúdo estruturado** — descrição expandível, comodidades por categoria, localização
4. **SEO** — estrutura semântica, JSON-LD já implementado (manter)
5. **Performance** — imagens optimizadas (já usa `property_images` + `image_variants`), lazy loading

### Referências de Design

- **Holidu:** layout 2 colunas (conteúdo | booking), galeria grid 1+4, badge "Reserva directa"
- **Airbnb:** hero panorâmico, floating booking card, estrelas/reviews
- **Identity Home Stay:** tons brand (azul-petróleo), accent coral, tipografia Inter

### Booking Widget — interacção com AvailabilityCalendar

O componente `AvailabilityCalendar` existente (`src/components/public/AvailabilityCalendar.tsx`) deve ser reutilizado/adaptado:
- Desktop: embutido no card booking (sem border extra)
- Mobile: dentro de um Sheet/Modal (usar Radix Dialog ou solução nativa)

### Mapa Estático

Usar OpenStreetMap Staticmap (gratuito, sem API key):
```
https://staticmap.openstreetmap.de/staticmap.php?center={lat},{lng}&zoom=14&size=600x300&markers={lat},{lng},lightblue
```
Se `property.latitude` e `property.longitude` existirem na tabela. Se não existirem, mostrar placeholder com endereço textual.

> Verificar se colunas `latitude`/`longitude` existem em `properties`. Se não, não bloquear — mostrar só endereço.

### Header Scroll Behaviour

```tsx
// PropertyPageHeader.tsx (client component)
const [scrolled, setScrolled] = useState(false)
useEffect(() => {
  const onScroll = () => setScrolled(window.scrollY > 80)
  window.addEventListener('scroll', onScroll, { passive: true })
  return () => window.removeEventListener('scroll', onScroll)
}, [])
// className: scrolled ? 'bg-white shadow-sm' : 'bg-transparent'
```

# Story 36.9b: Revenue Forecasting UI & Export

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.9b  
**Status:** Done  
**Story Points:** 3  
**Priority:** HIGH  

---

## User Story

As a **Host**, I want **to see and interact with my revenue forecasts through an intuitive dashboard** so that **I can understand projected revenue visually and export forecasts for financial planning**.

---

## Description

Build the frontend UI layer for revenue forecasting (Story 36.9 backend complete). Displays forecast cards, interactive charts with confidence intervals, detailed breakdowns, and PDF/CSV export. Mobile-first responsive design with touch-friendly interactions.

**Why:** Backend forecasting algorithms are ready; hosts need visual interface to access and use predictions.

---

## Acceptance Criteria

### Forecast Cards Display (3 Cards: 30/60/90 Days)

- [ ] ForecastCards component displays three cards side-by-side (web) or stacked (mobile)
- [ ] Each card shows: Period label (30/60/90 days) + Projected revenue (EUR) + % change indicator
- [ ] Color coding for % change: Green (positive) | Red (negative) | Gray (neutral)
- [ ] Confidence score badge on each card (e.g., "High (95%)", "Medium (70%)", "Low (50%)")
- [ ] Tap card → Opens ForecastDetailsModal with full breakdown
- [ ] Last updated timestamp: "Updated 2 hours ago" with refresh button
- [ ] Loading state with skeleton loaders during data fetch
- [ ] Error state with retry button if fetch fails
- [ ] Summary line: "Projected total revenue over next 90 days: €12,450"

### Interactive Chart Component

- [ ] ForecastChart displays line chart with Recharts library
- [ ] Chart shows: Forecast line (solid) + Confidence bands (±15%, shaded area)
- [ ] X-axis: Date range (adjustable: 30/60/90 days)
- [ ] Y-axis: EUR amounts with proper scaling
- [ ] Legend: "Forecast" (blue) | "Confidence Range" (light blue) | "Historical Actuals" (gray)
- [ ] Tooltip on hover: Date + Projected amount + Confidence level
- [ ] Touch-friendly on mobile: Swipe to pan, pinch to zoom (optional advanced feature)
- [ ] Responsive sizing: Full width on mobile, max 900px on web
- [ ] Export button on chart: "Download chart as image"
- [ ] No grid lines distraction on mobile; subtle grid on web

### Statistics Cards Section

- [ ] StatisticsCards component (below chart)
- [ ] Shows 4 key metrics:
  1. **Average Daily Rate (ADR)** - Projected avg revenue per night (€)
  2. **Confidence Score** - Percentage with explanation (e.g., "High: Based on 35 bookings")
  3. **Occupancy Rate Forecast** - Projected occupancy % for next 30 days
  4. **Trend Indicator** - "↑ Upward", "↓ Downward", "→ Stable" with trend % change
- [ ] Each stat card shows: Label + Value + Explanation tooltip
- [ ] Icon/badge for quick visual reference
- [ ] Mobile: Cards stack vertically, web: 2x2 grid layout

### Forecast Details Modal

- [ ] Tap any forecast card → ForecastDetailsModal opens
- [ ] Modal displays:
  1. **Period breakdown**: Daily forecast values table for selected period
  2. **Assumptions section**: Seasonal factor, occupancy rate, base price used
  3. **Comparison chart**: Forecast vs historical actual chart (if data available)
  4. **Forecast methodology**: "Based on 90-day historical analysis + seasonal adjustments"
  5. **Major factors influencing forecast**: Bullet list (e.g., "Summer peak season +30%", "Holiday block-out -10%")
  6. **Data quality warning**: "Low confidence: only 5 bookings in similar season" (if applicable)
- [ ] Close button (X) or tap outside to dismiss
- [ ] Scroll-able content on mobile
- [ ] Desktop: Modal centered, max 800px width

### PDF Export Functionality

- [ ] Export button in top-right of dashboard: "Download PDF"
- [ ] PDF includes:
  1. Header: Property name + date range + "Revenue Forecast Report"
  2. Three forecast cards summary (30/60/90 days)
  3. Forecast chart embedded as image
  4. Statistics section with all 4 metrics
  5. Detailed daily breakdown table
  6. Methodology explanation
  7. Footer: Generated date, property ID, confidence disclaimer
- [ ] PDF is branded with property logo (if available)
- [ ] File naming: `{PropertyName}_Forecast_{StartDate}_{EndDate}.pdf`
- [ ] Loading indicator during PDF generation
- [ ] "PDF downloaded successfully" toast notification
- [ ] Handle error gracefully: "Could not generate PDF. Try again."
- [ ] Mobile: PDF generates same content but optimized for A4 aspect ratio

### CSV Export Functionality

- [ ] "Export as CSV" option (next to PDF button or in menu)
- [ ] CSV includes:
  1. Property name, ID, date range in header rows
  2. Daily forecast data: Date | Projected Revenue | Confidence Level | Occupancy %
  3. Summary section below data
- [ ] Proper CSV formatting: Correct escaping, UTF-8 encoding
- [ ] File naming: `{PropertyName}_Forecast_Daily_{StartDate}_{EndDate}.csv`
- [ ] "CSV downloaded successfully" toast notification
- [ ] Support opening in Excel/Google Sheets without formatting issues

### Component Structure

```

├── ForecastingDashboard.tsx       (Main orchestrator)
├── ForecastCards.tsx               (3-card display with tap handlers)
├── ForecastCard.tsx                (Single card component)
├── ForecastChart.tsx               (Recharts line chart)
├── StatisticsCards.tsx             (4-metric grid)
├── StatisticCard.tsx               (Single stat card)
├── ForecastDetailsModal.tsx        (Detailed breakdown modal)
└── __tests__/
    ├── ForecastingDashboard.test.tsx
    ├── ForecastCards.test.tsx
    ├── ForecastChart.test.tsx
    ├── StatisticsCards.test.tsx
    └── ForecastDetailsModal.test.tsx

src/hooks/
├── useForecastData.ts              (API data fetching + caching)
└── __tests__/useForecastData.test.ts

src/lib/
├── export/
│   ├── pdfExport.ts                (jsPDF + html2canvas)
│   ├── csvExport.ts                (papaparse)
│   └── __tests__/
│       ├── pdfExport.test.ts
│       └── csvExport.test.ts
└── formatting/
    └── currencyFormatter.ts        (EUR formatting)

src/types/
└── forecasting.ts                 (TypeScript interfaces)
```

### Data Flow

```
useForecastData hook
  ├─ Fetch from GET /api/properties/[id]/analytics/forecasting
  ├─ Cache for 24 hours (localStorage + memory)
  └─ Return: { forecast30, forecast60, forecast90, loading, error, refresh }

ForecastingDashboard
  ├─ useRevenueForecasting hook
  └─ Render:
      ├─ ForecastCards (tap → ForecastDetailsModal)
      ├─ ForecastChart (line chart + bands)
      ├─ StatisticsCards (4 metrics)
      ├─ Export buttons (PDF, CSV)
      └─ Error boundary fallback
```

### Export Libraries

- **PDF**: `jsPDF` + `html2canvas` (render chart as image, embed in PDF)
- **CSV**: `papaparse` (format and generate CSV blob)
- Both libraries already in project dependencies

### Responsive Breakpoints

- Mobile: 0–767px (single column, full width)
- Tablet: 768–1023px (hybrid layout, cards may be 2-column)
- Desktop: 1024px+ (full layout with sidebar, 3-column grid)

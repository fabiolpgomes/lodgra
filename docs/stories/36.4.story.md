## Story

**Como** sistema de gestão de preços de alojamento local,  
**Quero** calcular o preço final de uma reserva aplicando dinamicamente as regras de desconto (duração, fins de semana, sobreposições),  
**Para que** os anfitriões possam oferecer preços competitivos e o sistema mostre transparência total do cálculo de tarifa.

---

## Contexto

O cálculo de preço de uma reserva envolve múltiplas camadas:
1. **Preço base:** tarifa de noite fornecida pela propriedade
2. **Multiplicador de fim de semana:** aumento opcional para sexta-feira e sábado (Story 36.1)
3. **Descontos por duração:** desconto de 7 noites (semana) e 28 noites (mês)
4. **Sobreposição de preços diários:** override de preço específico por data (Story 36.3)
5. **Validação de noites mínimas:** verificação se a duração atende requisitos

O Story 36.1 fornece dados de schema (base price, weekend multiplier). O Story 36.3 fornece sobreposições diárias. O Story 36.2 fornece acesso ao contexto global de preços.

Esta story implementa a **lógica pura de cálculo** — um utilitário sem dependências externas que retorna:
- **total:** preço final da reserva
- **avgNightly:** tarifa média por noite (para exibição em cards)
- **breakdown:** detalhe de cada componente aplicado (transparência)

## Critérios de Aceitação

- [x] **AC1** — Função `calculateBookingPrice()` retorna { total, avgNightly, breakdown }
  - total: número (soma de todas as noites ajustadas)
  - avgNightly: número (total / número de noites, 2 decimais)
  - breakdown: array de objetos com { component, value, reason }

- [x] **AC2** — Cálculo de preço base
  - Baseia-se em `nightly_rate` da propriedade (Story 36.1)
  - Multiplicado por número de noites
  - Ex: 7 noites × 100€ = 700€

- [x] **AC3** — Aplicação de multiplicador de fim de semana
  - Se `weekend_multiplier` está configurado (ex: 1.2)
  - Detecta sextas e sábados no intervalo de datas
  - Aplica multiplicador a essas noites específicas
  - Ex: sexta e sábado com 1.2x → 100€ × 1.2 = 120€/noite

- [x] **AC4** — Aplicação de descontos por duração
  - **Desconto de 7 noites:** se >= 7 noites, aplicar percentagem de desconto (ex: 5%)
  - **Desconto de 28 noites:** se >= 28 noites, aplicar percentagem maior (ex: 10%)
  - Ambos os descontos não se acumulam — aplicar o maior
  - Validar: se propriedade exige min_nights, rejeitar se duração < min_nights

- [x] **AC5** — Aplicação de sobreposições de preço diário
  - Dados vêm de Story 36.3 (DailyPrice com override)
  - Se `override` existe para uma data, substitui o preço calculado para essa noite
  - Não se acumula com desconto — é valor final para essa noite

- [x] **AC6** — Validação de requisitos de noites mínimas
  - Propriedade pode ter `min_nights` (ex: 3)
  - Se duração < min_nights, retornar erro { error: "Duration too short", required: 3 }
  - Precedência: validação antes de cálculo

- [x] **AC7** — Suporte para stacking de descontos e sobreposições
  - Se uma noite tem OVERRIDE, ela usa override (não desconto de duração)
  - Se nenhum override, calcula base + weekend_multiplier + desconto de duração
  - Ordem: base → weekend → duração desconto → override (se existe)

- [x] **AC8** — Tratamento de nulos e valores indefinidos
  - weekend_multiplier: null/undefined = 1.0 (sem multiplicação)
  - daily_prices (overrides): null/undefined = ignorar, calcular normal
  - min_nights: null/undefined = sem validação
  - Retornar valores válidos mesmo com dados parciais

- [x] **AC9** — Suite de testes (8+ casos)
  - Cálculo de preço base simples (sem multiplicadores)
  - Aplicação de multiplicador de fim de semana
  - Desconto de 7 noites

  - Sobreposição de preço diário substitui desconto
  - Validação de noites mínimas
  - Stacking de desconto + sobreposição

```typescript
// src/utils/pricing-calculator.ts

export interface DailyPrice {
  date: string;              // ISO 8601: YYYY-MM-DD
  basePrice: number;
  weekendPrice?: number;
  override?: number | null;  // null = sem override
}

export interface PricingConfig {
  checkInDate: string;       // ISO 8601: YYYY-MM-DD
  checkOutDate: string;
  nightlyRate: number;
  weekendMultiplier?: number; // ex: 1.2 (20% aumento) | null
  minNights?: number;        // validação de duração mínima
  sevenNightDiscount?: number; // ex: 0.05 (5% desconto) | null
  twentyEightNightDiscount?: number; // ex: 0.10 (10%) | null
  dailyPrices?: DailyPrice[]; // sobreposições por data (Story 36.3)
}

export interface PricingResult {
  total: number;             // preço final total
  avgNightly: number;        // média por noite (2 decimais)
  breakdown: BreakdownItem[];
  error?: {
    code: string;
    message: string;
    required?: number;
  };
}

export interface BreakdownItem {
  component: 'base' | 'weekend' | 'discount_7night' | 'discount_28night' | 'override';
  nightCount: number;
  ratePerNight: number;
  value: number;
  reason: string;
}

export class PricingCalculator {
  /**
   * Calcula o preço total de uma reserva aplicando todas as regras de desconto
   * @param config Configuração de preços e descontos
   * @returns Resultado com total, média e breakdown detalhado
   */
  static calculateBookingPrice(config: PricingConfig): PricingResult {
    // implementação aqui
  }

  /**
   * Helper: calcula se uma data é fim de semana (sexta ou sábado)
   */
  private static isWeekend(dateStr: string): boolean {
    const date = new Date(dateStr);
    const day = date.getDay();
    return day === 5 || day === 6; // 5=sexta, 6=sábado
  }

  /**
   * Helper: calcula número de noites entre check-in e check-out
   */
  private static calculateNights(checkIn: string, checkOut: string): number {
    const start = new Date(checkIn);
    const end = new Date(checkOut);
    return Math.ceil((end.getTime() - start.getTime()) / (1000 * 60 * 60 * 24));
  }

  /**
   * Helper: obtém preço para uma data específica (override ou base)
   */
  private static getPriceForDate(
    date: string,
    baseRate: number,
    dailyPrices?: DailyPrice[],
    weekendMultiplier?: number
  ): number {
    // se existe override para essa data, retornar
    const override = dailyPrices?.find(p => p.date === date)?.override;
    if (override !== null && override !== undefined) {
      return override;
    }

    // senão, aplicar multiplicador de fim de semana se aplicável
    if (weekendMultiplier && this.isWeekend(date)) {
      return baseRate * weekendMultiplier;
    }

    // senão, retornar base rate
    return baseRate;
  }
}
```

### Algoritmo de Cálculo

```
1. VALIDAÇÃO
   - Se duração < min_nights → retornar erro
   - Se checkOut <= checkIn → retornar erro

2. INICIALIZAÇÃO
   - nights = calculateNights(checkIn, checkOut)
   - dailyPricesMap = Map<date, override>
   - breakdown = []
   - total = 0

3. ITERAÇÃO POR DIA
   Para cada noite entre checkIn e checkOut:
     - date = checkIn + i
     - priceForNight = getPriceForDate(date, baseRate, dailyPrices, weekendMultiplier)
     - total += priceForNight
     - Se override existe para data:
         breakdown.push({ component: 'override', value: priceForNight, reason: 'Daily price override' })
     - Senão se weekend:
         breakdown.push({ component: 'weekend', value: (baseRate * weekendMultiplier - baseRate), reason: 'Weekend rate' })
     - Senão:
         breakdown.push({ component: 'base', value: baseRate, reason: 'Base nightly rate' })

4. APLICAÇÃO DE DESCONTO POR DURAÇÃO
   - Não aplicar a noites com override
   - Se nights >= 28: aplicar desconto 28 noites
   - Senão se nights >= 7: aplicar desconto 7 noites
   - breakdown.push({ component: 'discount_X', value: discountAmount, reason: '...' })
   - total -= discountAmount

5. RETORNO
   - avgNightly = total / nights (2 decimais)
   - return { total, avgNightly, breakdown }
```

## Testes (Critérios de Aceitação)

### Teste 1: Preço Base Simples
- Input: 3 noites, 100€/noite, sem multiplicadores
- Expectativa: total = 300€, avgNightly = 100.00€
- Validação: breakdown contém 1 item (base, 3 noites, 300€)

### Teste 2: Multiplicador de Fim de Semana
- Input: 3 noites (sexta 10, sábado 11, domingo 12), 100€, 1.2x weekend
- Expectativa: total = 320€ (120 + 120 + 100), avgNightly = 106.67€
- Validação: 2 itens de weekend, 1 de base em breakdown

### Teste 3: Desconto de 7 Noites
- Input: 7 noites, 100€, 5% desconto 7-night
- Expectativa: total = 665€, avgNightly = 95.00€
- Validação: breakdown contém 2 itens (base 700€, desconto -35€)

### Teste 4: Desconto de 28 Noites
- Input: 28 noites, 100€, 5% desconto 7-night + 10% desconto 28-night
- Expectativa: aplicar 10% (maior), não ambos → total = 2520€ (2800 - 280)
- Validação: breakdown contém desconto 28-night, não 7-night

### Teste 5: Sobreposição Substitui Desconto
- Input: 7 noites, 100€, 5% desconto, mas noite 1 = 150€ override
- Expectativa: total = 720€ (150 + 600 - 30) = noite 1 não participa do desconto
- Validação: breakdown mostra override, desconto aplicado a 6 noites apenas

### Teste 6: Validação Noites Mínimas
- Input: 2 noites, min_nights = 3
- Expectativa: retorna { error: { code: 'MIN_NIGHTS_NOT_MET', required: 3 } }
- Validação: sem cálculo de total (null ou undefined)

### Teste 7: Descontos Não Acumulam
- Input: 28 noites com ambos 7% e 10% descontos configurados
- Expectativa: aplicar 10%, não soma 17%
- Validação: desconto em breakdown = 10% somente

### Teste 8: Nulos Tratados
- Input: weekend_multiplier = null, sevenNightDiscount = undefined, dailyPrices = undefined
- Expectativa: total = 300€ (3 × 100), sem erros, sem multiplicadores
- Validação: breakdown contém apenas item base

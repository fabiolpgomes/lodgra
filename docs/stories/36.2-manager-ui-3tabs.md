# Story 36.2: Manager UI — 3-Tab Settings Panel

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.2  
**Status:** Draft  
**Story Points:** 5  
**Priority:** HIGH  

---

## User Story

As a **Host**, I want **to configure pricing, discounts, and availability in a settings panel** so that **I can easily adjust my property's rental rules**.

---

## Description

Build a mobile-responsive settings panel with 3 tabs (Preços | Descontos | Disponibilidade). Users access this via ⚙️ icon on calendar view. Form inputs save to API endpoints from Story 36.1.

**Why:** Hosts need intuitive UI to manage pricing without coding.

---

## Acceptance Criteria

### Tab 1: Preços

- [ ] Display "Preço básico" input (EUR, required)
- [ ] Display "Preço de fim de semana" input (EUR, optional)
- [ ] Show "Remover" link next to weekend price (if set)
- [ ] Toggle switch for "Preço Inteligente" (disabled state, label: "Seus preços serão ajustados automaticamente...")
- [ ] Section "Taxas" with → chevron (expandable, future feature)
- [ ] "Salvar" button posts to `POST /api/properties/:id/prices`
- [ ] Show success message on save
- [ ] Show error message if validation fails

### Tab 2: Descontos

- [ ] Card "Por semana":
  - Label: "Para 7 noites ou mais"
  - Input: Percentage (0-100)
  - Display calculated average: "A média semanal é de €XXX.XX"
  - Calculation: `(base_price × 7) × (1 - percentage/100)`

- [ ] Card "Por mês":
  - Label: "Para 28 noites ou mais"
  - Input: Percentage (0-100)
  - Display calculated average: "A média mensal é de €XXX.XX"
  - Calculation: `(base_price × 28) × (1 - percentage/100)`

- [ ] Disabled cards (future):
  - "Hóspedes com avaliações excelentes"
  - "Reservas de última hora"
  - "Reservas antecipadas"

- [ ] "Salvar" button posts to `POST /api/properties/:id/discounts`
- [ ] Show success/error messages

### Tab 3: Disponibilidade

- [ ] Input "Número mínimo de noites" (default 1)
- [ ] Input "Número máximo de noites" (default 365)
- [ ] Select "Tempo de antecedência" (0=same day, 1-90 days, default 0)
- [ ] Time picker "Aviso prévio para o mesmo dia" (HH:MM, default 00:00)
- [ ] Select "Tempo de preparação" (0-30 days, default 0)
- [ ] "Salvar" button posts to `PUT /api/properties/:id/availability`
- [ ] Show success/error messages

### Form Validation

- [ ] Client-side: Required fields, numeric ranges, time format
- [ ] Server-side: Same validation on API (Story 36.1)
- [ ] Show validation errors inline

### Responsive Design

- [ ] Mobile (≤768px): Full-screen modal or bottom sheet
- [ ] Web (>768px): Side panel (integrate with Story 36.5 layout)

## Risks

| Risk | Mitigation |
|------|-----------|
| Form validation out of sync with API | Write shared validation schema (zod) |
| Mobile keyboard covers inputs | Use scrolling or bottom sheet |
| Floating point precision (prices) | Use DECIMAL in DB, format on UI |

---

## Technical Notes

- Use React Hook Form for form management
- Use `zod` for validation schema (shared with backend)
- Average calculations: display only, don't store
- Form state management: local React state + React Query for API calls
- Toast notifications for success/error

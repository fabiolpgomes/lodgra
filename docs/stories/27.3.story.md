## Objective

Surface OTA reviews in Google My Business listing by extracting top-rated reviews from the property_reviews table and including them in the Google Vacation Rentals feed as a reviews section. This enables Google to display aggregate ratings and review information in search results.

---

## Acceptance Criteria

- [ ] **AC1:** Extract reviews from `property_reviews` table with rating >= 4.0
- [ ] **AC2:** Filter reviews by source: Booking.com, Airbnb, Vrbo only
- [ ] **AC3:** Extend 27.2 feed to include `<reviews>` section with review metadata
- [ ] **AC4:** Calculate and display average rating in feed (simple average across all reviews per property, not weighted)
- [ ] **AC5:** Include review text and author information (if available)
- [ ] **AC6:** Feed updates incrementally when new reviews are ingested
- [ ] **AC7:** No breaking changes to existing feed endpoint (AC1-AC7 from 27.2 remain passing)

## Implementation Details

### Location
- **Primary file:** `/src/lib/feeds/google-feed-generator.ts` (extend existing)
- **Query file:** `/src/lib/feeds/review-aggregator.ts` (new — review extraction logic)

- Reviews: `property_reviews` table (existing from Epic 25)

## Dev Notes

- **Previous Story Insights**: Story 27.2 established the feed structure and pagination. Reuse existing `generateGoogleVacationRentalsFeed()` function and extend with review section.
- **Data Models**: Reviews stored in `property_reviews` table (Epic 25.1). Schema includes: `id`, `property_id`, `rating`, `source`, `text`, `author`, `date`, `created_at`. [Source: Epic 25.1 migration]
- **Component Reuse**: PropertyReviewScore component (25.3) already aggregates reviews by OTA source. Reuse aggregation logic from this component.
- **Performance Consideration**: Review aggregation may impact query performance for large property sets. Consider database view or materialized view for `property_review_aggregates`.
- **Testing Strategy**: Unit tests should cover: multiple reviews per property, filtering by rating/source, empty reviews, average calculation, incremental updates.
- **AC6 Implementation Note**: `updated_since` parameter filters by property `updated_at` timestamp (inherited from Story 27.2). Reviews are freshly fetched for each property returned, so new reviews appear on the next feed generation for that property. Real-time review sync from OTA sources is handled by a separate sync process (out of scope per Story Notes).

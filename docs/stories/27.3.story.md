## 📝 Dev Notes

### Architecture

**Review Aggregation Flow:**
```
Daily Cron (02:00 UTC)
  ├─ Booking.com API → Pull reviews
  ├─ Airbnb API → Pull reviews
  ├─ Google My Business API → Pull reviews
  ├─ Merge & deduplicate in property_reviews table
  └─ Update aggregateRating in LodgingBusinessSchema
```

### Database Schema (property_reviews)

Already exists from Epic 25 (OTA Reviews). Add fields if needed:
```sql
property_reviews (
  id UUID PRIMARY KEY,
  property_id UUID,
  reviewer_name VARCHAR,
  rating DECIMAL(2,1),
  source VARCHAR, -- 'booking', 'airbnb', 'google'
  comment TEXT,
  review_date DATE,
  is_featured BOOLEAN,
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  last_synced TIMESTAMP -- Track last sync time per review
)
```

### API Integrations

#### Booking.com API
- Endpoint: `https://secure-supply-xml.booking.com/hotels/xml/reviews`
- Auth: Hotel ID + API key (provided by Booking)
- Rate limit: 100 req/min
- Review fields: `reviewer_name`, `review_text`, `rating`, `review_date`

#### Airbnb API
- Endpoint: Need to use Airbnb public API or web scraping (if available)
- Auth: Access token (OAuth 2.0)
- Rate limit: 200 req/hour
- Rating scale: 1-5 (map to Booking's 1-10 scale)

#### Google My Business API
- Endpoint: `https://mybusiness.googleapis.com/v4/accounts/{accountId}/locations/{locationId}/reviews`
- Auth: OAuth 2.0 service account
- Rate limit: 1000 req/min
- Review fields: `reviewer.displayName`, `starRating`, `reviewReply`, `createTime`

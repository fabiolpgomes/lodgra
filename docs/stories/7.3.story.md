## Story

**Como** gestor de propriedades,
**Quero** visualizar e gerir reservas num calendário interactivo com drag-drop,
**Para que** eu possa mover datas de reservas e criar novas reservas directamente no calendário de forma intuitiva.

---

## Contexto de Negócio

A gestão de reservas actualmente é feita via tabela/lista. Um calendário visual permite:
1. Ver ocupação de cada propriedade num relance
2. Mover uma reserva para outras datas arrastando (drag-drop)
3. Criar nova reserva clicando num período livre
4. Detectar overlaps visualmente e prevenir conflitos

Esta é uma das features mais pedidas por property managers.

---

## Critérios de Aceitação

- [ ] **AC1** — Existe página `/calendar` acessível a admin, manager e viewer
- [ ] **AC2** — Calendário mostra reservas confirmadas de todas as propriedades da org (ou filtradas por propriedade)
- [ ] **AC3** — Cada reserva aparece como bloco colorido com: nome do hóspede, datas, propriedade
- [ ] **AC4** — Drag-drop: arrastar reserva para novas datas actualiza `check_in`/`check_out` na BD via PATCH
- [ ] **AC5** — Ao soltar reserva em datas ocupadas (overlap) → mostrar erro e reverter visualmente
- [ ] **AC6** — Clicar em período livre → abre modal "Nova Reserva" pré-preenchido com datas
- [ ] **AC7** — Filtro por propriedade (dropdown) — mostra todas ou filtra por propriedade seleccionada
- [ ] **AC8** — Viewer não pode arrastar (read-only) — apenas admin e manager podem modificar
- [ ] **AC9** — Navegação entre meses (anterior/próximo) e botão "Hoje"

### API

**GET `/api/calendar/reservations?from=YYYY-MM-DD&to=YYYY-MM-DD&property_id=optional`**
```json
[
  {
    "id": "res-001",
    "title": "João Silva — Casa Alfama",
    "start": "2026-03-10",
    "end": "2026-03-15",
    "color": "#3b82f6",
    "extendedProps": {
      "guest_name": "João Silva",
      "property_name": "Casa Alfama",
      "property_id": "prop-001",
      "status": "confirmed"
    }
  }
]
```

**PATCH `/api/calendar/reservations/[id]`**
```json
{
  "check_in": "2026-03-12",
  "check_out": "2026-03-17"
}
```
- Verificar overlap antes de guardar
- Retornar 409 se overlap detectado

### Overlap Detection (API)
```sql
SELECT id FROM reservations
WHERE property_listing_id IN (
  SELECT id FROM property_listings WHERE property_id = $propertyId
)
AND status = 'confirmed'
AND id != $reservationId
AND check_in < $newCheckOut
AND check_out > $newCheckIn
LIMIT 1
```

### Componente Principal

```tsx
// src/components/calendar/ReservationCalendar.tsx
import FullCalendar from '@fullcalendar/react'
import dayGridPlugin from '@fullcalendar/daygrid'
import interactionPlugin from '@fullcalendar/interaction'

export function ReservationCalendar({ role }) {
  const [events, setEvents] = useState([])

  const handleEventDrop = async ({ event, revert }) => {
    if (role === 'viewer') { revert(); return }
    const result = await patchReservation(event.id, { check_in: ..., check_out: ... })
    if (!result.ok) { revert(); toast.error('Datas ocupadas') }
  }

  const handleDateSelect = ({ startStr, endStr }) => {
    if (role === 'viewer') return
    openNewReservationModal({ check_in: startStr, check_out: endStr })
  }

  return (
    <FullCalendar
      plugins={[dayGridPlugin, interactionPlugin]}
      initialView="dayGridMonth"
      editable={role !== 'viewer'}
      selectable={role !== 'viewer'}
      events={events}
      eventDrop={handleEventDrop}
      select={handleDateSelect}
      locale="pt"
    />
  )
}
```

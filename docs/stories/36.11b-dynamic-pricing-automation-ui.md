# Story 36.11b: Dynamic Pricing Automation — UI & Automation (Deferred)

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.11b  
**Status:** InReview  
**Story Points:** 5  
**Priority:** HIGH  
**Scope:** Complete 36.11a with UI components, automation scheduling, audit trail

---

## User Story

As a **Host**, I want **to create and manage pricing rules with a visual editor, preview changes before applying, and see a history of all pricing changes** so that **I can understand how my prices are changing and have confidence in automation**.

---

## Description

Complete the dynamic pricing automation feature deferred from 36.11a. Implements visual rule builder, dry-run preview mode, scheduled automation, and audit trail logging. Includes all UI components and background job scheduling.

**Why:** Users need visual feedback and control before applying automated pricing changes.

---

## Acceptance Criteria

### Rules Editor UI (Complete)

- [x] Advanced rule builder with condition/action UI
- [x] Pre-built rule templates (occupancy tiers, booking pace, seasonal)
- [x] Visual rule priority reordering (drag-drop)
- [x] Rule enable/disable toggle
- [x] Delete rule with confirmation

### Dry-Run Preview

- [x] Toggle "Preview pricing changes" mode
- [x] Display current vs simulated prices side-by-side
- [x] Show EUR and % difference per night
- [x] Calculate total impact on next 30 days revenue
- [x] Accept/reject preview changes
- [x] Discard to revert to current prices

### Holiday & Event Detection

- [x] Configure holiday list (auto-detect major holidays)
- [x] Add/edit custom holidays for local celebrations
- [x] Show holidays on calendar
- [x] Rules trigger ±3 days around holidays

### Automation Scheduling

- [x] Configure daily automation run time (default 00:00 UTC)
- [x] Manual trigger button "Apply rules now"
- [x] Pause automation for date range
- [x] Show next scheduled run time
- [x] Background job execution with retry logic

### Audit Trail & Logging

- [x] Log all price changes (manual + automated)
- [x] Log view with filters (date range, rule, type)
- [x] Export audit log as CSV
- [x] Retain history for 1 year
- [x] Show triggered rule name in log entry

### Responsive Design

- [x] Mobile: Card-based rules editor, stacked layout
- [x] Web: Form-based editor, side-by-side preview
- [x] All prices formatted as EUR
- [x] Touch-friendly controls on mobile

## Risks

| Risk | Mitigation |
|------|-----------|
| Timezone confusion in scheduling | Force UTC internally, convert for display |
| Dry-run inaccuracy | Test against actual rule engine, edge cases |
| Audit trail data size | Rotate logs by year, archive old entries |
| Job scheduling failures | Retry with exponential backoff, alert on failure |

---

## Technical Notes

- UI components use React hooks for state management
- Dry-run: Calculate for next 30 days without persisting
- Schedule: Use node-cron or bull queue for reliability
- Audit trail: Immutable append-only log (no edits/deletes)
- Holiday detection: iCalendar format for flexibility

# Story 36.11b: Dynamic Pricing Automation — UI & Automation (Deferred)

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)  
**Story ID:** 36.11b  
**Status:** Planned (Deferred from 36.11a)  
**Story Points:** 5  
**Priority:** HIGH  
**Scope:** Complete 36.11a with UI components, automation scheduling, audit trail

---

## User Story

As a **Host**, I want **to create and manage pricing rules with a visual editor, preview changes before applying, and see a history of all pricing changes** so that **I can understand how my prices are changing and have confidence in automation**.

---

## Description

Complete the dynamic pricing automation feature deferred from 36.11a. Implements visual rule builder, dry-run preview mode, scheduled automation, and audit trail logging. Includes all UI components and background job scheduling.

**Why:** Users need visual feedback and control before applying automated pricing changes.

---

## Acceptance Criteria

### Rules Editor UI (Complete)

- [ ] Advanced rule builder with condition/action UI
- [ ] Pre-built rule templates (occupancy tiers, booking pace, seasonal)
- [ ] Visual rule priority reordering (drag-drop)
- [ ] Rule enable/disable toggle
- [ ] Delete rule with confirmation

### Dry-Run Preview

- [ ] Toggle "Preview pricing changes" mode
- [ ] Display current vs simulated prices side-by-side
- [ ] Show EUR and % difference per night
- [ ] Calculate total impact on next 30 days revenue
- [ ] Accept/reject preview changes
- [ ] Discard to revert to current prices

### Holiday & Event Detection

- [ ] Configure holiday list (auto-detect major holidays)
- [ ] Add/edit custom holidays for local celebrations
- [ ] Show holidays on calendar
- [ ] Rules trigger ±3 days around holidays

### Automation Scheduling

- [ ] Configure daily automation run time (default 00:00 UTC)
- [ ] Manual trigger button "Apply rules now"
- [ ] Pause automation for date range
- [ ] Show next scheduled run time
- [ ] Background job execution with retry logic

### Audit Trail & Logging

- [ ] Log all price changes (manual + automated)
- [ ] Log view with filters (date range, rule, type)
- [ ] Export audit log as CSV
- [ ] Retain history for 1 year
- [ ] Show triggered rule name in log entry

### Responsive Design

- [ ] Mobile: Card-based rules editor, stacked layout
- [ ] Web: Form-based editor, side-by-side preview
- [ ] All prices formatted as EUR
- [ ] Touch-friendly controls on mobile

## Risks

| Risk | Mitigation |
|------|-----------|
| Timezone confusion in scheduling | Force UTC internally, convert for display |
| Dry-run inaccuracy | Test against actual rule engine, edge cases |
| Audit trail data size | Rotate logs by year, archive old entries |
| Job scheduling failures | Retry with exponential backoff, alert on failure |

---

## Technical Notes

- UI components use React hooks for state management
- Dry-run: Calculate for next 30 days without persisting
- Schedule: Use node-cron or bull queue for reliability
- Audit trail: Immutable append-only log (no edits/deletes)
- Holiday detection: iCalendar format for flexibility

# Story i18n.4: Locale Selector & User Preferences

**Epic:** i18n & Global Expansion  
**Status:** ⏸️ Deferred (Architectural Redesign Required)  
**Priority:** P1  
**Tipo:** Feature / UX  
**Agente:** @dev + @ux-design-expert

> **Note (2026-04-05):** This story was implemented but reverted due to architectural issues with aggressive i18n routing. Will be re-implemented after i18n redesign (root-first pattern). See `docs/REVERT_DECISION_2026_04_05.md`.

---

## Story

**Como** utilizador multilíngue,  
**Quero** poder mudar de idioma/moeda facilmente no header,  
**Para que** possa alternar entre PT/BR/EN sem refazer login.

---

## Context

Depende de: **Story i18n.1** (infrastructure), **i18n.2** (translations), **i18n.3** (currencies)

Sistema já terá routing por locale e formatação funcionando. Agora adicionar:
1. Componente visual para selector de idioma (flag icons + dropdown)
2. Persistência da preferência do utilizador
3. Sincronização com localStorage/DB
4. Sem perder estado da página (não faz hard refresh)

---

## Acceptance Criteria

### AC1: Locale Selector Component
- [ ] Header tem botão com flag do idioma atual
- [ ] Click abre dropdown com 3 opções (PT, PT-BR, EN-US)
- [ ] Cada opção mostra: flag icon + language name
- [ ] Mobile-friendly: full-width selector ou bottom sheet
- [ ] Desktop-friendly: compact dropdown

### AC2: User Preference Persistence
- [ ] Preferência salva em `user_preferences.preferred_locale`
- [ ] Fallback: Accept-Language header
- [ ] Fallback 2: localStorage
- [ ] Ao login: restaurar preference do profile

### AC3: Seamless Language Switching
- [ ] Clicar em "English" → redirect a `/en-US{currentPath}`
- [ ] Sem logout, sem perda de dados
- [ ] URL muda, página recarrega em nova lingua
- [ ] Charts/relatórios refazem com locale novo
- [ ] Toast confirma: "Idioma alterado para English"

### AC4: Mobile Optimization
- [ ] Flag icons responsive (24px → 32px)
- [ ] Touch target mínimo 44px
- [ ] Bottom sheet em mobile se necessário
- [ ] Landscape orientation suportada

### AC5: Accessibility
- [ ] ARIA labels em cada opção de idioma
- [ ] Keyboard navigation (Tab, Enter, Escape)
- [ ] Screen reader anouncement na mudança
- [ ] Color contrast 4.5:1 nos botões

### AC6: Testing & Integration
- [ ] Change locale en dashboard → componentes traduzem
- [ ] Change locale em calendar → months/days atualizam
- [ ] Refresh página → locale persistido
- [ ] Multi-browser: Chrome, Firefox, Safari

## Story

**Como** potencial hóspede,
**Quero** aceder a uma página pública da propriedade com fotos, descrição e disponibilidade,
**Para que** possa decidir reservar sem precisar de conta no sistema.

---

## Contexto de Negócio

O objectivo estratégico é reduzir dependência do Airbnb/Booking, gerando reservas directas. Para isso, cada propriedade precisa de uma URL pública, indexável pelo Google, com informação completa e botão de reserva.

Esta story é o pré-requisito para todas as outras do Epic 9 (calendário, preços, checkout).

---

## Critérios de Aceitação

- [ ] **AC1** — Rota `/p/[slug]` acessível sem autenticação (middleware deve excluir esta rota do check de sessão)
- [ ] **AC2** — Slug gerado automaticamente a partir do nome da propriedade (ex: "T1 Sesimbra" → `t1-sesimbra`); admin pode personalizar via settings
- [ ] **AC3** — Página mostra: nome, descrição, localização, capacidade (máx. hóspedes), comodidades, fotos (URLs da coluna `photos` ou placeholder)
- [ ] **AC4** — Botão "Verificar Disponibilidade / Reservar" leva ao componente de calendário (Story 9.2)
- [ ] **AC5** — Página tem meta tags SEO: `<title>`, `<meta name="description">`, Open Graph (og:title, og:image, og:description)
- [ ] **AC6** — Design mobile-first, responsivo, com branding neutro (sem referências ao dashboard interno)
- [ ] **AC7** — Se slug não existir → 404 correcto
- [ ] **AC8** — Campo `slug` adicionado à tabela `properties` com constraint UNIQUE e índice

## Dev Notes

### Schema — Migration
```sql
-- supabase/migrations/20260320_01_property_slug.sql
ALTER TABLE properties
  ADD COLUMN IF NOT EXISTS slug TEXT UNIQUE,
  ADD COLUMN IF NOT EXISTS description TEXT,
  ADD COLUMN IF NOT EXISTS photos TEXT[] DEFAULT '{}',
  ADD COLUMN IF NOT EXISTS amenities TEXT[] DEFAULT '{}',
  ADD COLUMN IF NOT EXISTS max_guests INTEGER DEFAULT 2,
  ADD COLUMN IF NOT EXISTS is_public BOOLEAN DEFAULT false;

CREATE INDEX IF NOT EXISTS idx_properties_slug ON properties(slug);

-- Função para gerar slug a partir do nome
CREATE OR REPLACE FUNCTION generate_slug(name TEXT) RETURNS TEXT AS $$
BEGIN
  RETURN lower(regexp_replace(unaccent(trim(name)), '[^a-z0-9]+', '-', 'g'));
END;
$$ LANGUAGE plpgsql IMMUTABLE;
```

**Epic:** 36 (Dynamic Pricing + Calendar Redesign)

**Status:** Ready for Review  
**Story Points:** 5  
**Priority:** HIGH  

---

## User Story

As a **Host**, I want **to automatically adjust my prices based on occupancy and demand** so that **I can optimize revenue without manually adjusting prices**.

---

## Description

Build an automated pricing engine that applies pricing rules based on occupancy rates, booking pace, and demand signals. Includes a rules engine (if-then logic), minimum/maximum price guardrails, holiday/event detection, and dry-run mode for testing. All automated changes are logged in an audit trail.

**Why:** Hosts need automation to dynamically price without manual intervention, maximizing revenue during peak demand and filling vacancy during low season.

---

## Acceptance Criteria

### Rules Engine

- [ ] Create rules editor with if-then logic builder
- [ ] Rule conditions:
  - Occupancy rate (>=50%, >=70%, >=80%, >=90%)
  - Booking pace (days until full, advance bookings)
  - Seasonal indicator (peak/shoulder/low season)
  - Day-of-week (weekday vs weekend)
  - Days until check-in (1-7, 8-14, 15-30, 30+ days)
- [ ] Rule actions:
  - Increase price by X% or €X amount
  - Decrease price by X% or €X amount
  - Set price to specific value
- [ ] Support multiple rules with priority order
- [ ] Example rules (pre-built templates):
  - "If occupancy >= 80%, increase by 15%"
  - "If occupancy <= 30%, decrease by 20%"
  - "If booking within 7 days, increase by 10%"
- [ ] Enable/disable individual rules without deleting

### Price Guardrails

- [ ] Set minimum price (currency input, required)
- [ ] Set maximum price (currency input, required)
- [ ] Ensure calculated price never violates guardrails
- [ ] Display warning if rule would exceed guardrails
- [ ] Log guardrail violations in audit trail

### Holiday & Event Detection

- [ ] Automatic detection of major holidays (configurable list)
- [ ] Manual holiday/event creation for local celebrations
- [ ] Rules trigger based on holiday proximity (±3 days)
- [ ] Example: "Increase 20% during New Year week"
- [ ] Display holidays on calendar

### Dry-Run Mode

- [ ] Toggle "Preview pricing changes" before applying
- [ ] Dry-run shows:
  - Current prices (actual)
  - Simulated prices (after rules)
  - Difference per night (EUR and %)
  - Total impact on next 30 days revenue
- [ ] Compare before/after calendar view
- [ ] Accept changes → apply to calendar
- [ ] Discard → revert to current prices

### Audit Trail & Logging

- [ ] Log every price change (manual + automated)
- [ ] Log entry includes:
  - Timestamp
  - Triggered by (rule name or manual)
  - Old price → New price
  - Reason (occupancy, holiday, etc.)
  - User action (accepted/rejected dry-run)
- [ ] Display audit log in settings
- [ ] Filter by date range, rule, or type (manual/auto)
- [ ] Export audit log as CSV
- [ ] Retain audit history for 1 year

### Automation Schedule

- [ ] Run automation daily at specified time (default 00:00 UTC)
- [ ] Manual trigger button "Apply pricing rules now"
- [ ] Pause automation for date range (e.g., during promotions)
- [ ] Schedule custom rules to activate/deactivate on specific dates
- [ ] Show next scheduled run time

### Responsive Design

- [ ] Mobile (≤768px): Rules editor card-based, stacked layout
- [ ] Web (>768px): Form-based rules editor, side-by-side comparison
- [ ] Dry-run preview display clearly on both layouts
- [ ] All currency formatted as EUR

## Risks

| Risk | Mitigation |
|------|-----------|
| Rules cause unexpected price changes | Dry-run mode mandatory, comprehensive rule validation |
| Guardrail violations go unnoticed | Visual warnings, audit logging, automatic enforcement |
| Performance impact with many rules | Optimize rule evaluation, cache compiled rules |
| Timezone confusion for scheduling | Force UTC, display local time with conversion |
| Users create conflicting rules | Priority order prevents conflicts, validation warnings |

---

## Technical Notes

- Rules engine: Parse rule conditions into evaluator, use expression parser
- Price calculation: Apply rules sequentially by priority, enforce guardrails after each rule
- Holiday database: Use iCalendar format, support custom holidays
- Schedule: Use node-cron or bull queue for reliable job execution
- Audit trail: Immutable log entries, no editing (only soft-delete with reason)
- Performance: Pre-compile rules on save, cache occupancy calculations
- Timezone handling: Store all times in UTC, convert to user's timezone for display
- Dry-run: Calculate for next 30 days, don't persist predictions
- Guardrail validation: Enforce at rule save time and at execution time

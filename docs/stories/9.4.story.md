## Story

**Como** potencial hóspede,
**Quero** reservar uma propriedade directamente no site, introduzindo os meus dados e pagando com cartão,
**Para que** possa confirmar a estadia sem ir ao Airbnb ou Booking.

---

## Contexto de Negócio

Este é o núcleo do booking engine: o momento em que o hóspede passa de visitante a cliente pagante. O fluxo deve ser simples (3 passos máximo), seguro (Stripe Checkout), e gerar automaticamente uma reserva no sistema com status `pending_payment` → `confirmed` após webhook Stripe.

O pagamento total é cobrado no acto da reserva (não pré-autorização).

---

## Critérios de Aceitação

- [ ] **AC1** — Rota `/p/[slug]/checkout` recebe `checkin`, `checkout`, `guests` como query params; se inválidos → redireciona para `/p/[slug]`
- [ ] **AC2** — Passo 1 (Resumo): mostra datas, noites, hóspedes, preço total (calculado via Story 9.3); botão "Continuar"
- [ ] **AC3** — Passo 2 (Dados do hóspede): nome completo, email, telefone, país; validação cliente + servidor
- [ ] **AC4** — Passo 3 (Pagamento): botão "Pagar [X€]" cria Stripe Checkout Session e redireciona para Stripe
- [ ] **AC5** — Endpoint `POST /api/public/bookings` cria reserva com `status = 'pending_payment'` e retorna `stripe_checkout_url`
- [ ] **AC6** — Após pagamento bem-sucedido, Stripe redireciona para `/p/[slug]/booking-confirmed?session_id=xxx`
- [ ] **AC7** — Página de confirmação mostra resumo da reserva e instrução de contacto
- [ ] **AC8** — Se hóspede cancelar no Stripe → redireciona para `/p/[slug]` (sem criar reserva permanente)
- [ ] **AC9** — Reservas `pending_payment` com mais de 30 minutos sem confirmação são automaticamente canceladas (cron existente ou novo)
- [ ] **AC10** — Campos do hóspede criados/actualizados na tabela `guests` (reutilizar estrutura existente)

## Dev Notes

### Fluxo técnico
```
/p/[slug]/checkout?checkin=X&checkout=Y&guests=N
  → POST /api/public/bookings
      → cria guest (ou upsert por email)
      → cria reservation (status=pending_payment)
      → cria Stripe Checkout Session
      → retorna { checkout_url }
  → redirect para Stripe
  → Stripe webhook (Story 9.5) confirma pagamento
  → redirect para /p/[slug]/booking-confirmed
```

### Stripe Checkout Session
```typescript
const session = await stripe.checkout.sessions.create({
  payment_method_types: ['card'],
  mode: 'payment',
  line_items: [{
    price_data: {
      currency: 'eur',
      unit_amount: Math.round(totalAmount * 100),  // em cêntimos
      product_data: {
        name: `${property.name} — ${nights} noites`,
        description: `${checkin} → ${checkout}`,
      },
    },
    quantity: 1,
  }],
  success_url: `${process.env.NEXT_PUBLIC_APP_URL}/p/${slug}/booking-confirmed?session_id={CHECKOUT_SESSION_ID}`,
  cancel_url: `${process.env.NEXT_PUBLIC_APP_URL}/p/${slug}`,
  metadata: {
    reservation_id: reservation.id,
  },
})
```

## Story

**Como** gestor de propriedades,
**Quero** gerar um relatório mensal ou anual por proprietário mostrando receitas, despesas e valor a transferir,
**Para que** eu possa enviar ao proprietário um resumo transparente e profissional das suas propriedades.

---

## Contexto de Negócio

Os gestores de propriedades precisam de comunicar regularmente com os proprietários dos imóveis. Actualmente não existe nenhuma forma de gerar um relatório por proprietário. O gestor tem de calcular manualmente e preparar um Excel ou PDF.

Com esta feature, o gestor pode:
1. Ir a `/owners/[id]` → clicar "Ver Relatório"
2. Seleccionar período (mês/trimestre/ano)
3. Ver e exportar PDF com P&L por propriedade do proprietário

### Export PDF
Usar `window.print()` com CSS `@media print` — simples, sem dependências externas:
```tsx

<Button onClick={() => window.print()}>Exportar PDF</Button>

// CSS
@media print {
  .no-print { display: none; }
  .report-container { width: 100%; }
}
```

### Dados para cálculo

```typescript
// Query principal
const { data: properties } = await supabase
  .from('properties')
  .select('id, name, management_percentage')
  .eq('owner_id', ownerId)
  .eq('organization_id', organizationId)

// Reservas no período (via property_listings)
const { data: reservations } = await supabase
  .from('reservations')
  .select('total_amount, check_in, check_out, property_listings(property_id)')
  .gte('check_in', from)
  .lte('check_in', to)
  .in('property_listing_id', listingIds)

// Despesas no período
const { data: expenses } = await supabase
  .from('expenses')
  .select('amount, property_id')
  .gte('expense_date', from)
  .lte('expense_date', to)
  .in('property_id', propertyIds)
```

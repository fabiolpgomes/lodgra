## Pre-Launch Phase (Week Before)

### Code & Testing

- [ ] **Lint & Build**
  - [ ] `npm run lint` — PASS

  - [ ] All warnings resolved or documented

- [ ] **Manual Testing**
  - [ ] Test full user flow: signup → create property → set pricing → publish → book
  - [ ] Multi-currency: EUR + BRL flows tested
  - [ ] Mobile responsiveness verified (landing page, checkout, dashboard)
  - [ ] Stripe checkout flow tested (test card: 4242 4242 4242 4242)
  - [ ] Gmail OAuth flow tested (with test Gmail account)
  - [ ] Settings/integrations UI functional

- [ ] **Edge Cases**
  - [ ] Expired session behavior
  - [ ] Network timeout handling
  - [ ] Invalid input validation (dates, prices, email)
  - [ ] Empty state screens (no properties, no bookings, no reports)
  - [ ] Permission boundaries (viewer vs manager vs admin)

- [ ] **Database**
  - [ ] All migrations applied (`supabase migration up`)
  - [ ] RLS policies verified for each table
  - [ ] Org isolation confirmed in 5+ tables
  - [ ] Test data fixtures loaded (if needed)

## Known Limitations v1.1

Document here any known issues / planned future work:

- [ ] **Story 3.1 (Email Parsing)** — Implementation complete, awaiting QA manual testing with real emails
- [ ] **2FA** — Noted as future feature in Business plan; not implemented
- [ ] **API Access** — Noted as future feature; public REST API not yet available
- [ ] **Bulk Operations** — Properties cannot be bulk imported; manual one-by-one only
- [ ] **Custom Pricing** — Pricing tiers are fixed (Starter/Prof/Business); no per-property custom pricing
- [ ] **Multi-Language** — Only Portuguese (PT) supported; no i18n yet
- [ ] **Mobile App** — Web-only; no native iOS/Android app

## Rollback Procedure

**If critical issue found, execute rollback:**

1. **Immediate (within 5 min)**
   - [ ] Vercel: Revert to previous deployment (use Vercel UI)
   - [ ] Supabase: Revert to latest backup (if schema change)
   - [ ] Verify previous version accessible

2. **Communication**
   - [ ] Announce incident to users
   - [ ] Provide ETA for fix

3. **Root Cause**
   - [ ] Document what went wrong
   - [ ] Identify fix
   - [ ] Test fix locally before re-deploy

4. **Re-Deploy**
   - [ ] Run through smoke tests again
   - [ ] Deploy fixed version
   - [ ] Announce resolution

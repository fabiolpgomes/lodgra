# v1.1 Monitoring Template — Guia de Uso

**Arquivo:** `v1.1-MONITORING-TEMPLATE.csv`
**Período:** 24h pós-launch (23 Mar 10:00 → 24 Mar 10:00)
**Frequência:** A cada 2 horas

---

## Como Usar

1. **Abra o arquivo CSV:**
   - No Excel: File → Open → v1.1-MONITORING-TEMPLATE.csv
   - No Google Sheets: File → Open → Upload → v1.1-MONITORING-TEMPLATE.csv
   - Ou copie/cole o conteúdo em uma planilha

2. **A cada 2 horas:**
   - Preencha uma linha com os valores atuais
   - Adicione notas se algo anormal ocorrer
   - Salve o arquivo

3. **Se encontrar um alerta:**
   - Marque em vermelho ou "⚠️"
   - Anote o problema na coluna "Notas"
   - Investigue (veja "Onde Obter Cada Métrica" abaixo)

## Onde Obter Cada Métrica

### **Erros (%)**
**Fonte:** Sentry.io ou Vercel Logs

- Acesse: https://vercel.com/fabiolpgomes-projects/home-stay/logs
- Procure por `error` ou `[ERROR]`
- Conte erros vs. total de requisições
- **Target:** < 1%
- **Fórmula:** (Total erros / Total requisições) × 100

**Exemplo:**
```
Se houver 5 erros em 1000 requisições = 0.5% ✅
Se houver 50 erros em 1000 requisições = 5% ⚠️
```

---

### **Signups (#)**
**Fonte:** Supabase Dashboard ou Database

```sql
-- Execute em Supabase SQL Editor:
SELECT COUNT(*) as signups_today
FROM user_profiles
WHERE created_at >= now() - interval '24 hours'
  AND created_at >= '2026-03-23 10:00:00';
```

Ou vá para: https://app.supabase.com → Browse Data → user_profiles → Filter by created_at

---

### **Bookings (#)**
**Fonte:** Supabase Dashboard

```sql
-- Execute em Supabase SQL Editor:
SELECT COUNT(*) as bookings_today
FROM reservations
WHERE created_at >= now() - interval '24 hours'
  AND created_at >= '2026-03-23 10:00:00'
  AND source = 'direct';  -- bookings diretos (não importados)
```

---

### **Email Delivery (%)**
**Fonte:** Resend Dashboard

1. Vá para: https://resend.com/emails
2. Filtro por data: "Hoje"
3. Veja:
   - Total enviados
   - Delivered (verde)
   - Bounced (vermelho)
   - Spam (amarelo)

**Fórmula:**
```
(Delivered / Total Enviados) × 100
Target: > 95%
```

**Exemplo:**
```
100 enviados, 98 delivered = 98% ✅
100 enviados, 85 delivered = 85% ⚠️
```

---

### **Page Load - LCP (ms)**
**Fonte:** Chrome DevTools ou Lighthouse

1. Abra https://www.homestay.pt em Chrome
2. Pressione `F12` → Lighthouse tab
3. Click "Analyze page load"
4. Veja **LCP (Largest Contentful Paint)**

**Target:** < 2500ms (2.5s)
**Bom:** < 1500ms
**Ruim:** > 3000ms

**Ou manualmente:**
1. F12 → Network tab
2. Recarregue a página (Ctrl+R)
3. Veja o tempo total na parte inferior

---

### **Stripe Webhooks (OK/Total)**
**Fonte:** Stripe Dashboard

1. Vá para: https://dashboard.stripe.com/events
2. Filtre por "type" = `charge.succeeded` e `invoice.payment_succeeded`
3. Conte:
   - **OK:** Status "succeeded" (verde)
   - **Total:** Todos os eventos desse tipo
   - **Failed:** Status "failed" (vermelho)

**Exemplo:**
```
20 webhooks processados com sucesso / 21 total = 95% (1 falhou)
Valor esperado: 20/20 ou 21/21 = 100%
```

---

### **Site Status**
**Fonte:** Browser simples

Valores possíveis:
- `✅ UP` — Site carrega, HTTP 200
- `⚠️ SLOW` — Carrega mas lento (>5s)
- `🔴 DOWN` — Erro 502/503/504

**Como testar:**
```bash
curl -I https://www.homestay.pt
# Esperado: HTTP/2 200
```

## Coluna "Notas"

Anote:
- Qualquer evento incomum
- Mudanças significativas nas métricas
- Problemas encontrados
- Ações tomadas

**Exemplos:**
```
"5 erros de timeout em Stripe, mas resolvidos em 10 min"
"Spike de signups às 14:00 (marketing email?)"
"Page load spike às 18:00 — investigar database queries"
"Tudo normal, trending up"
```

## Status Codes / Interpretação

| Métrica | Alerta Verde ✅ | Alerta Amarelo ⚠️ | Alerta Vermelho 🔴 |
|---------|-----------------|------------------|-------------------|
| **Erros** | < 1% | 1-5% | > 5% |
| **Signups** | > 5 | 1-5 | 0 (check why) |
| **Bookings** | > 1 | 0 (early) | — |
| **Email Delivery** | > 95% | 80-95% | < 80% |
| **LCP** | < 1500ms | 1500-2500ms | > 2500ms |
| **Webhooks** | 100% | 90-99% | < 90% |
| **Site** | ✅ UP | ⚠️ SLOW | 🔴 DOWN |


## Ações por Alerta

### Se Erros > 5%
1. Vá para Vercel Logs
2. Procure por `[ERROR]` e `[Exception]`
3. Identifique o padrão (qual endpoint? qual erro?)
4. Se crítico: escalade para Dev team

### Se Email Delivery < 80%
1. Vá para Resend Dashboard
2. Veja qual tipo de email está falhando
3. Check: template válido? Endereço correto? Domain authenticated?
4. Se >20% bounces: escalade para Resend support

### Se Page Load > 3s
1. F12 → Network tab → reload
2. Identifique qual recurso está lento
3. Se é API: check database slow queries
4. Se é JS: check bundle size

### Se Site Down
1. Check Vercel deployment status
2. Check database (Supabase) connection
3. If deployed recently: rollback
4. Contact DevOps/Tech Lead immediately

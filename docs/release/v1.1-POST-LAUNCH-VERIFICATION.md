# v1.1 Post-Launch Verification Checklist

**Purpose:** Validate system health and user experience in the weeks following production launch
**Audience:** QA, Support, Product, Tech Lead
**Timeline:** Day 1, Day 3, Day 7, Day 30

---

## Day 1 (Launch Day + 24h)

### System Health

- [ ] **Error Rate**
  - Check: Sentry / Vercel logs
  - Target: <1% error rate
  - Red flag: >10 errors / hour
  - Action: If high: See troubleshooting guide

- [ ] **Performance**
  - Check: Vercel Analytics (if configured)
  - Target: LCP <2.5s, FID <100ms, CLS <0.1
  - Red flag: Page load >5s
  - Action: Investigate slow queries / cold starts

- [ ] **Database**
  - Check: Supabase → Logs
  - Target: No connection timeouts
  - Red flag: Connection pool exhausted, locked tables
  - Action: Restart connection pool if needed

- [ ] **Third-Party Services**
  - [ ] Stripe: Webhook events processing (check Stripe logs)
  - [ ] Resend: Emails delivering (>95% delivery rate)
  - [ ] Supabase: No alerts in status
  - [ ] Vercel: No build queue backlog

### User Flow Testing

**Sign Up → Create Property → Publish → Book**

- [ ] **Desktop Flow**
  - [ ] Landing page loads
  - [ ] Sign up accepts valid email
  - [ ] Email confirmation works
  - [ ] Dashboard loads after login
  - [ ] Create property flow completes
  - [ ] Property appears in list
  - [ ] Publish property (enable public booking)
  - [ ] Public URL accessible
  - [ ] Calendar shows availability
  - [ ] Booking flow initiated
  - [ ] Stripe checkout appears
  - [ ] Test payment accepted (4242...)
  - [ ] Confirmation email received
  - [ ] Booking appears in dashboard

- [ ] **Mobile Flow** (iOS Safari + Chrome Android)
  - [ ] Landing page responsive
  - [ ] Forms usable on mobile
  - [ ] Touch targets (buttons) appropriately sized
  - [ ] No horizontal scroll
  - [ ] Checkout functional on mobile

- [ ] **Edge Cases**
  - [ ] Invalid date range (checkout < checkin): Error shown
  - [ ] Expired session: Redirect to login
  - [ ] Browser back button: No data loss
  - [ ] Rapid successive clicks: No double-booking
  - [ ] Offline → Online: Page reloads gracefully

### Data Integrity Checks

```sql
-- Run these queries in Supabase SQL Editor

-- 1. Verify all signups have organizations
SELECT COUNT(*) as users_without_org
FROM user_profiles
WHERE organization_id IS NULL;
-- Expected: 0

-- 2. Verify org isolation (pick random org)
SELECT * FROM organizations LIMIT 1;
SELECT * FROM properties WHERE organization_id = '[org-from-above]';
-- Expected: Properties belong only to that org

-- 3. Verify booking data integrity
SELECT COUNT(*) as bookings_total FROM reservations;
SELECT COUNT(DISTINCT organization_id) as orgs_with_bookings FROM reservations;
-- Expected: bookings > 0, orgs > 0

-- 4. Verify RLS policies working
-- Run as different user, verify data isolation
SELECT * FROM properties WHERE organization_id != current_user_org_id;
-- Expected: Returns 0 rows (RLS enforced)

-- 5. Check for orphaned data
SELECT COUNT(*) as orphaned_properties
FROM properties
WHERE organization_id NOT IN (SELECT id FROM organizations);
-- Expected: 0
```

**Actions:**
- [ ] All checks pass
- [ ] If any checks fail: Investigate & hotfix
- [ ] Document any data anomalies

## Critical Defects Found? (Any Time)

**If you discover a critical bug:**

1. **Immediate (0-5 min)**
   - [ ] Assess impact: How many users affected?
   - [ ] Is system down? Can users still book/pay?
   - [ ] Create incident ticket

2. **Urgent (0-30 min)**
   - [ ] Post status: Announce issue & ETA
   - [ ] Engage team: Hotfix required?
   - [ ] Workaround: Anything support can tell users?

3. **Fix (30 min - hours)**
   - [ ] Root cause analysis
   - [ ] Write fix locally
   - [ ] Test thoroughly
   - [ ] Deploy to production

4. **Follow-up (after fix)**
   - [ ] Post-mortem: Why wasn't this caught?
   - [ ] Prevention: How to avoid in future?
   - [ ] Document: Add to troubleshooting guide

## Appendix: Quick Diagnostics

### Is the system down?

```bash
# Test public URL
curl -I https://homestay.example.com
# Expected: 200 or 301 (not 502/503/504)

# Check Vercel status
# Go to: https://vercel.com/status
# Expected: All systems operational

# Check Supabase status
# Go to: https://status.supabase.com
# Expected: All systems operational
```

### Are bookings being created?

```sql
SELECT COUNT(*) as total_bookings FROM reservations;
SELECT MAX(created_at) as latest_booking FROM reservations;
-- Expected: count > 0, latest_booking = today or recent
```

### Why aren't emails sending?

```sql
-- Check recent email logs (if captured)
SELECT * FROM email_logs ORDER BY created_at DESC LIMIT 10;
```

Or check Resend Dashboard → Emails → Filter by status

### Is RLS working?

Log in as user from Org A:
```sql
SELECT * FROM properties;
-- Expected: Returns ONLY properties.organization_id = Org A
```

Switch to user from Org B: Should see different properties.
